# Genorah v4.5.2 — Remove dashboard Quick Actions queue

Patch on v4.5.1. Removes the dashboard's **Quick Actions** queue buttons.

## Why
The Quick Action buttons only wrote passive *reminder* files to `.planning/genorah/.action-queue/` — they never ran anything (Claude Code can't invoke its own slash commands) and never cleared themselves, so they accumulated as stale "queued action" reminders. Net clutter for no real capability.

## Removed
- The **Quick Actions** panel (the `Queue /gen:audit` / `status` / `self-audit` buttons) and the queued-action ⚡ banner from the dashboard.
- Their now-dead client JS and CSS (`queueAction`, the `.action-btn` listener, the banner render + styles).
- The stale documentation for the panel in `commands/dashboard.md`; the dashboard is now described as read-only.

## Left in place (dormant, harmless)
- The server `POST /api/action/:cmd` route, `handleAction`, and the snapshot's `action_queue` field remain (nothing in the UI triggers them now). `buildViewModel`'s `queue` field and its unit test are unchanged. These can be fully stripped in a later cleanup if desired.

## Verification
Running dashboard reload-verified: the Quick Actions panel and banner are gone, panels read Waves → Sections → Gate Hotspots → Visual QA → Context → Decisions → Knowledge Graph, **0 console errors**. Full suite **249/249, clean exit**; mirror parity verified; self-audit BLOCK=0.

**Full delta:** `v4.5.1..v4.5.2`.

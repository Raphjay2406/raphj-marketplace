# v3.5.4 – v3.18 Deep Analysis

**Full scope analysis produced during v3.5.3 → v3.5.4 planning.**

This document captures the analysis shared at the start of Phase 1 implementation. It covers v3.5.4 Execution Closure through v3.18 Advanced Visual Refinement with per-version scope, implementation specifics, dependencies, edge cases, testing strategy, and open questions.

See the commit history for the conversational analysis; this file preserves the canonical plan.

## Phase structure

- **Phase 1 — Execution Closure (v3.5.4 – v3.5.7):** close the spec↔runtime gap, seed data foundations, production-grade reliability + observability, E2E test harness + tutorial.
- **Phase 2 — New Axes (v3.6 – v3.15):** backend + ops, mobile depth, AI-native + personalization, content depth, integration ecosystem (with HubSpot / n8n / Supabase added).
- **Phase 3 — Depth & Polish (v3.10+):** i18n, advanced a11y, legal/ethics, plugin self-improvement, new archetypes, testing infrastructure for generated projects, security hardening, advanced visual refinement.

## Phase 1 summary (see commits for detailed implementation)

- **v3.5.4 — Execution Closure:** 11 runtime scripts, 4 hook upgrades (post-tool-use ledger emissions, pre-compact survivor protocol, session-start drift alerts + compaction resume, session-end postship nudge), Meshy MCP scaffold, test harness.
- **v3.5.5 — Data Foundation:** 25 archetype testable-markers, 25 weights, 25 lighting rigs, 25 material sets, 25 HDRi maps, 25 Recraft substyles, golden set seed (25 entries), reference library seed (50 entries), budget-tier baselines.
- **v3.5.6 — Reliability + Observability:** wave failure recovery, graceful degrade for probe hangs / MCP crashes / dev-server restarts / NSFW blocks / low judge confidence / disk exhaustion, dashboard v2 (11 tabs), cost transparency, error taxonomy + recovery guide.
- **v3.5.7 — Test Harness + Tutorial:** E2E canary pipeline, plugin health dashboard, interactive `/gen:tutorial`, glossary, FAQ, troubleshooting playbook, migration guide.

## Phase 2 summary

- **v3.6 — Backend + Ops:** RSC patterns, API routes, DB schema gen, env var scheme, deploy-preview automation, canary/rolling releases, telemetry/RUM, background jobs.
- **v3.7 — Mobile Depth:** adaptive icons/splash/store graphics per platform, mobile quality-gate variant, DNA→Material/Cupertino bridge, simulator + real-device testing, QR preview.
- **v3.8 — AI-Native + Personalization:** voice UI patterns, WebXR AR, real-time personalization (per-user DNA adjustments), AI chat depth, WebAuthn/Passkeys, active UX adjustment.
- **v3.9 — Content Depth:** long-form pipeline, React Email templates, video script gen, interactive content, social asset variants.

- **v3.11 Advanced A11y** — eye-tracking sim, low-bandwidth mode, voice-control audit, tremor-friendly, battery-aware.

- **v3.13 Plugin Self-Improvement** — opt-in telemetry, skill usage analytics, auto-deprecation, cross-user health dashboard.
- **v3.14 New Archetypes + Creative Depth** — 8+ new archetypes (museum, academic, gaming UI, medical, civic, financial, portfolio, hand-drawn), archetype mixing protocol.
- **v3.16 Testing Infrastructure for Generated Projects** — E2E test gen from PLAN.md journey, visual regression baselines, a11y suites, perf-budget CI wiring, journey templates, API integration scaffolds.
- **v3.17 Security Hardening** — prompt-injection defense, 2026 PII regex, font license tracking, SBOM, API key rotation, content moderation, CSP generator.

## Key risks (ranked)

1. Data curation bandwidth (v3.5.5) — bottleneck on human curator time
2. External API costs (v3.6, v3.8) — AI/deploy costs can surprise
3. Cross-platform parity (v3.7) — native platform quirks
4. Third-party API churn (v3.15) — integrations age
5. Plugin telemetry privacy posture (v3.13) — needs careful framing

## Recommended sequencing

Ship rc per week in Phase 1. Phase 2 parallel after v3.5.7 stable. Phase 3 incremental by user demand.

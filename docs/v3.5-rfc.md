# Genorah v3.5 RFC — Asset Forge + Quality Spine + Context Fabric

**Status:** Draft
**Target release:** v3.5.0 → v3.5.3 (4 patch cycles)
**Hard prereq:** v3.4.2 "Enforcement Closure" must ship first.

This RFC merges three design tracks proposed during the v3.4.1 post-ship audit into one coherent release plan.

## Vision

Genorah v3.5 closes three simultaneous gaps:

1. **Asset pipeline is shallow.** v3.4.x generates code to SOTD quality but its *media* pipeline is limited to raster image prompts, SVG extrusion, and a few procedural primitives. A Brutalist hero with a stock photo still feels generic.
2. **Quality gate is aesthetic-heavy.** 85% of the 234-pt gate measures craft; ~15% measures accessibility; ~0% measures usability, cognitive load, conversion intent, or synthetic-user task completion. "SOTD-Ready" today means "looks like SOTD," not "works like SOTD."
3. **Retention is amnesiac.** Every agent spawn is cold. CONTEXT.md preserves DNA tokens but nothing about what agent did last, why it chose that, what worked. Judge drifts between runs. Reference library doesn't learn. Cross-project lessons evaporate.

v3.5 addresses all three with a single architecture because the three are coupled: a deeper pipeline creates more artifacts, which demands retention; empirical calibration of gates produces numbers, which demand persistence; an asset forge with DNA governance must store manifest state that survives sessions.

### Asset sub-gate (cascades into Integration Quality)

| Check | Test | On fail |
|---|---|---|
| Manifest entry | sha256 + seed + model + archetype present | Category × 0.6 |
| DNA coverage | ≥ 60% dominant colors ∈ DNA (ΔE2000 ≤ 8) | −6 |
| Archetype material | ∈ archetype `preferred_materials` | −6 |
| Forbidden-pattern scan | No archetype-forbidden techniques | −8 |
| Style coherence | Pairwise SSIM ≥ 0.45 across kind/beat | −4 |
| License clear | No `license: unknown` | Block commit |
| Cache reproducibility | Same key → bit-identical OR seed+prompt+model recorded | −3 |

## Axis 2: Quality Spine (quality-gate-v3)

Two-axis 354-pt gate. A section must clear **both** axes for tier ≥ Strong.

```
Axis 1: DESIGN CRAFT      234 pts  (existing v2, with v3.4.2 measurement rules)
Axis 2: UX INTEGRITY      120 pts  (new in v3.5)
```

### UX Integrity sub-gates (120 pts)

| Sub-gate | Pts | Machine checks |
|---|---|---|
| **UX Heuristics** (Nielsen 10) | 20 | Status visibility, match real-world, user control, consistency, error prevention, recognition vs recall, flexibility, aesthetic-minimalist, error recovery, help discoverable |
| **Interaction Fidelity** | 20 | 44×44 touch targets, focus ring ≥ 3:1, keyboard reaches all, hover ≠ default, form autocomplete, escape-closes-modal, primary-action-above-fold at 1280×720 |
| **Cognitive Load** | 20 | Element count per viewport ≤ beat max, reading grade ∈ archetype band, choices ≤ 7 (Hick), heading hierarchy strict, CTA ≤ 2 per viewport, whitespace ratio ∈ band |
| **Conversion Integrity** | 20 | Primary CTA above fold at 1280 AND 375px, trust signal ≤ 2 scrolls, friction audit, CTA copy specific, social proof attributed, pricing transparent |
| **Visual Craft (Quantified)** | 20 | Baseline-grid ≥ 90%, type-scale ratio 1.125–1.333, whitespace band, color harmony (max 3 off-DNA hues), Gestalt proximity |
| **Synthetic Usability** | 20 | 6-persona task runs (see v3.5.1) |

### Tier scale (both axes required)

| Tier | Design | UX | Budget mode alignment |
|---|---|---|---|
| Reject | < 140 | < 70 | — |
| Baseline | 140–169 | 70–84 | lean |
| Strong | 170–199 | 85–99 | standard |
| SOTD | 200–219 | 100–109 | standard |
| Honoree | 220–234 | 110–119 | max |
| SOTM | 235+ | 120 | max |

## Axis 3: Context Fabric (v3.5.3)

### 8-Layer retention

| Layer | Scope | Persistence | Artifact |
|---|---|---|---|
| L1 Agent scratchpad | single spawn | ephemeral | in-prompt `<scratch>` |
| L2 Agent episodic | per agent kind | per-project | `.planning/genorah/agent-memory/{agent}.ndjson` |
| L3 Section memory | per section | per-project | `sections/{id}/MEMORY.md` |
| L4 Project ledger | whole project | per-project | `.planning/genorah/journal.ndjson` (append-only) |
| L5 Semantic index | L3+L4 embeddings | per-project | `.planning/genorah/index/{vectors,meta}.db` (sqlite-vec) |
| L6 Cross-project KB | lessons, patterns | user-global | Obsidian vault (enriched) |
| L7 Calibration store | judge drift, thresholds | user-global | `~/.claude/genorah/calibration.db` |
| L8 User auto-memory | user preferences | user-global | existing `~/.claude` (present) |

### Key mechanisms

- **Ledger entry schema:** `{ts, actor, kind, subject, payload, refs[]}` — one NDJSON line per significant event.
- **Compaction-Survivor Protocol:** `pre-compact.mjs` packs top-10 ledger entries + open decisions + active critiques + next-action into `compaction-summary.md` (≤ 3000 tokens); read first on resume.
- **Agent episodic memory:** specialists get top-3 nearest-neighbor past entries in spawn prompt via L5.
- **Decision Graph:** typed JSON, queryable edges (`impacts[]`, `supersedes[]`, `evidence[ledger_refs]`).
- **Context Diff on Resume:** `session-start.mjs` emits "what changed since last session" block.
- **Injection-Aware Skill Dedup:** skills not re-injected during same section work unless DNA or prompt signals change.
- **Reference Library Feedback:** variants clearing 95th percentile proposed to RAG; weekly batch review.
- **Cross-Project Lesson Extraction:** `session-end.mjs` on completion prompts 3 lessons → vault L6.
- **Calibration Persistence:** golden-set re-scores + judge κ history + UX-floor empirics persist user-global in SQLite.

## Risks & Mitigations

| Risk | Mitigation |
|---|---|
| API cost surprise | `/gen:assets init` hard cap + explicit approval; per-project caps in `.claude/genorah.local.md` |
| Model drift | Version pinned in manifest; regen requires explicit confirmation |
| Non-determinism | Seed pinning + cache; style coherence measured via SSIM |
| UX gate blocks bold archetypes | Archetype-weighted thresholds (Brutalist UX 65; Editorial 80) |
| Token cost of new loops | Budget mode (`lean|standard|max`) gates variant N, persona count, critic cycles |
| Context Fabric disk growth | Ledger rotates at 50MB → compact monthly to semantic summary |
| sqlite-vec install friction | Optional; degrade to BM25 over text ledger |
| Research program blocks release | Tracks run in parallel with shipping; defaults ship "provisional", recalibration upgrades in-place |
| Cross-project KB leakage | Lessons opt-in per project (default OFF for client work) |

## Shipping Dependencies

- **v3.4.2 Enforcement Closure** — non-negotiable prereq. Without Tier 1 audit items closed, new gates inherit advisory status.
- **Golden set (50 panel-scored sections)** — longest-lead item; start immediately, blocks R1/R5 and judge calibration.
- **Ledger schema lock** — every Fabric mechanism depends on it; schema churn is the most expensive mistake available.

# Genorah v4.5.1 — Full-Page Knowledge Graph

Patch on v4.5.0. Adds a dedicated full-page view for the dashboard's Knowledge Graph, plus a graceful fallback when the interactive viz is too large to generate.

## Full-page graph view
- **New `/graph` page** — a dedicated full-viewport graph explorer: a slim themed header with the live `nodes · edges · updated` summary (from `/api/state`) and a **← Dashboard** link, with the graphify visualization filling the rest of the screen.
- **Panel link** — the dashboard's Knowledge Graph panel gains a **"⤢ Full page"** button that opens `/graph` in a new tab. The compact in-panel preview stays as-is.

## Graceful viz fallback
- graphify **skips `graph.html`** for graphs larger than ~5,000 nodes (reachable on real repos — including this one), so `graph.json` can exist without an HTML visualization. Previously the embedded iframe rendered a bare "not found" on a white pane. Now **`/api/graph` serves a themed fallback** that explains the situation and shows the exact command to generate a viz (`graphify tree --output graphify-out/graph.html`) — fixing both the panel and the full-page view.

## Notes
- Zero-dependency, XSS-safe (`textContent` only; no new `innerHTML` on data). Server side-effects remain behind the main-guard.

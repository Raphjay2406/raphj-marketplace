# Genorah v4.4.0 — "Living Dashboard"

Minor release. **Phase 2 of the dashboard redesign** — the `localhost:4455` cockpit is rebuilt from a flat, all-monospace panel stack into a DNA-themed, animated, Awwwards-direction surface, all on the same zero-dependency Node stack. Genorah's own cockpit now looks like something Genorah would pass through its quality gate.

## The redesign

- **Wears the project's Design DNA.** On every update the dashboard applies the project's DNA color tokens (`bg`, `surface`, `text`, `primary`, `accent`, `glow`, …) as CSS variables, so each project's cockpit is skinned in its own palette — with a graceful default-dark theme when tokens are absent.
- **Instant-read stat band.** A new header band shows totals at a glance — sections, verified, floor-pass, floor-fail, average score — with count-up animation on change.
- **Bento layout + real type system.** A composed responsive grid (1→2→3 columns across 375/768/1280/1440) replaces the vertical stack; a refined sans/mono type system with tabular numerals replaces the uniform monospace; layered surfaces, hairline borders, and a DNA-tinted glow add depth.
- **Tier-colored section cards.** Each section's score is shown in its quality-gate named tier (Reject → SOTM-Ready) with tier-keyed color, above the Verification Spine FLOOR verdict row (badge + failing-check chips + advisory Ceiling) from Phase 1.
- **Wave status from real data.** Wave pills now derive pending/building/qa/done/failed from each wave's actual section verdicts and scores (sections gained an additive `wave` field parsed from `PLAN.md`), falling back to scraping `MASTER-PLAN.md` when sections lack a wave.
- **Motion, accessibly.** Staggered panel entrance, number count-ups, growing hotspot bars, and a live status pulse — **all gated by `prefers-reduced-motion`**.

## Architecture

- **New pure module `scripts/dashboard/view-model.mjs`** (`buildViewModel` + `scoreTier`/`summarize`/`hotspotBars`/`themeFromTokens`/`relativeTime`/`deriveWaves`) holds every presentation derivation. It has no DOM, no I/O, and no clock dependency, so it is fully `node:test`-covered (15 tests) — and the browser imports **the same module** from a new path-locked server route (`GET /scripts/dashboard/*.mjs`) instead of duplicating the logic.
- **Server change is additive-only**: sections gain an optional `wave` field; the Phase-1 `snapshot()` contract is otherwise frozen and its test passes untouched. Side-effects remain behind the main-guard (no import-hang regression).
- Rendering stays vanilla and **XSS-safe** (`textContent`/element construction; `encodeURIComponent` on URLs; `innerHTML` only on a static constant). Zero dependencies, no build step, no web-font fetch (offline-first).

## Fixes / polish

- The knowledge-graph iframe is now **lazy-loaded** only when a graph exists, and an empty favicon link is added — eliminating two stray `404`s (most visibly the `/api/graph` 404 every "no graph yet" project used to log).
- Non-finite scores can never leak `NaN` into the UI (defense-in-depth in the pure view-model).

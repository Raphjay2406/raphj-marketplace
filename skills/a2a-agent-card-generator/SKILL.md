# A2A Agent Card Generator

## Layer 1 — Decision Guidance

Invoke when creating or updating a Genorah agent (director or worker). Every agent MUST publish an A2A v0.3 compliant card derived from its frontmatter.

## Layer 2 — Example

Frontmatter → card via `buildAgentCard()`:

```ts
import { buildAgentCard } from "@genorah/protocol";
const card = buildAgentCard({
  name: "creative-director",
  id: "genorah/creative-director",
  version: "4.0.0",
  channel: "stable",
  tier: "director",
  description: "Owns taste and archetype personality",
  capabilities: [{ id: "review-plan", input: "PlanInput", output: "PlanReview" }]
});
```

Automatically generated by `scripts/generate-agent-cards.mjs` into `.claude-plugin/generated/agent-cards.json`.

## Layer 3 — Integration

- Reads agent frontmatter (name, id, version, channel, tier, description, capabilities, tools)
- Fills A2A v0.3 defaults (auth, streaming flags, MCP compat)
- Served by embedded HTTP daemon at `/.well-known/agent.json`
- Pipeline stage: post-scaffold, runs on `/gen:self-audit`

## Layer 4 — Anti-Patterns

- Hand-editing `.claude-plugin/generated/agent-cards.json` (regenerated on each scaffold)
- Forgetting `channel` field (required A2A v0.3)
- Using uppercase or dots in `id` (regex rejects)

## Layer 1: Decision Guidance

### When to Generate DNA

DNA is a **prerequisite** for all building. Generate it after brainstorm, before section planning.

- The creative-director agent generates DNA during `/gen:start-project`
- Archetype selection populates DNA defaults; user customizes from there
- DNA is written to `.planning/genorah/DESIGN-DNA.md` in the target project
- DNA is **immutable** during a build session -- changes require a new session

### Decision Tree: Token Selection

1. **Start from archetype locked palette** -- every archetype pre-defines all 12 color tokens, approved fonts, signature element, and motion defaults
2. **User overrides** -- user can adjust any token within archetype constraints
3. **Tension override** -- user can break ONE archetype constraint with documented rationale (see `design-archetypes` skill for escape hatch protocol)
4. **Font selection** -- choose from archetype-approved list (5-8 fonts per archetype). Display font MUST be distinctive -- Inter, Roboto, Arial, system-ui are forbidden as display fonts
5. **Signature element** -- combine archetype signature with user input. Must be specific and implementable, not vague

### When to Reference DNA

- **Section builders**: read DNA before writing any code. DNA snapshot is pre-extracted into spawn prompts by the build-orchestrator
- **Quality reviewer**: checks that built output uses DNA tokens (not arbitrary hex values)
- **Anti-slop gate**: scores DNA compliance across colors, typography, spacing, motion
- **Design scaffold (Wave 0)**: generates `globals.css` and Tailwind `@theme` block directly from DNA tokens

### When NOT to Use

- For archetype selection itself -- use `design-archetypes` skill
- For emotional beat definitions -- use `emotional-arc` skill (DNA contains the arc template, but beat constraints live in that skill)
- For anti-slop scoring criteria -- use `anti-slop-gate` skill (DNA is what gets scored, not the scoring system)

### Pipeline Connection

- **Generated by:** creative-director agent during `/gen:start-project` (after brainstorm)
- **Consumed by:** section-builder agents (every build), quality-reviewer (every verify), design-system-scaffold skill (Wave 0)
- **Locked at:** Start of execution. Immutable until explicit user reset

## Typography System

### Font Stack

| Role | Font Family | Weight Range | Google Fonts Import |
|------|------------|-------------|-------------------|
| Display | [e.g., Clash Display] | 500-700 | `family=Clash+Display:wght@500;600;700` |
| Body | [e.g., DM Sans] | 400-500 | `family=DM+Sans:wght@400;500` |
| Mono | [e.g., JetBrains Mono] | 400-500 | `family=JetBrains+Mono:wght@400;500` |

### Type Scale (8 levels)

| Level | Size (clamp) | Weight | Letter-spacing | Line-height | Usage |
|-------|-------------|--------|---------------|-------------|-------|
| hero | `clamp(3rem, 8vw, 6rem)` | 700 | -0.04em | 0.85 | Main hero headline |
| h1 | `clamp(2.25rem, 5vw, 3.75rem)` | 600 | -0.03em | 0.9 | Section headlines |
| h2 | `clamp(1.5rem, 3vw, 2.25rem)` | 600 | -0.02em | 0.95 | Sub-section heads |
| h3 | `clamp(1.25rem, 2vw, 1.5rem)` | 500 | -0.01em | 1.1 | Card/feature titles |
| h4 | `clamp(1.125rem, 1.5vw, 1.25rem)` | 500 | -0.005em | 1.2 | Small headings |
| body-large | `clamp(1.125rem, 1.2vw, 1.25rem)` | 400 | normal | 1.6 | Intro paragraphs |
| body | `1rem` | 400 | normal | 1.65 | Standard body text |
| small | `0.875rem` | 400 | 0.01em | 1.5 | Captions, metadata |

All heading sizes use `clamp()` for fluid responsive scaling. Body and small are fixed.

### Typography Rules
- Headlines: always font-display, always negative tracking
- Body: always font-body, max-w-[65ch] for readability
- At least 3 font weights visible per page
- One typographic surprise per page: [describe -- e.g., gradient text, variable font animation, oversized numeral]

## Spacing System

### 5-Level Scale

| Level | Token | Rem Value | Tailwind | Usage |
|-------|-------|----------|---------|-------|
| xs | `--space-xs` | 0.5rem | `p-2`, `gap-2` | Tight coupling (icon + label) |
| sm | `--space-sm` | 1rem | `p-4`, `gap-4` | Related elements |
| md | `--space-md` | 2rem | `p-8`, `gap-8` | Content blocks within sections |
| lg | `--space-lg` | 4rem | `py-16`, `gap-16` | Between content groups |
| xl | `--space-xl` | 6rem-8rem | `py-24 md:py-32` | Between major sections |

### Section Padding (Vertical Rhythm)

```
Section spacing: py-24 md:py-32 lg:py-40 (--space-xl)
Block spacing:   space-y-12 md:space-y-16 (between content blocks)
Element spacing: space-y-4 md:space-y-6 (between related elements)
Container:       px-6 md:px-12 lg:px-20 (horizontal padding)

## Motion Language

### Motion Tokens (8 tokens)

| Token Name | Duration | Easing (cubic-bezier) | Direction | Trigger |
|-----------|----------|----------------------|-----------|---------|
| entrance | 400ms | cubic-bezier(0.16, 1, 0.3, 1) | from-bottom | Scroll into view |
| exit | 300ms | cubic-bezier(0.4, 0, 1, 1) | to-top | Scroll out of view |
| hover | 200ms | cubic-bezier(0.3, 1.2, 0.2, 1) | scale-up | Mouse enter |
| scroll-reveal | 600ms | cubic-bezier(0.16, 1, 0.3, 1) | from-bottom | Viewport intersection |
| page-transition | 500ms | cubic-bezier(0.4, 0, 0.2, 1) | crossfade | Route change |
| micro-interaction | 150ms | cubic-bezier(0.3, 1.2, 0.2, 1) | varies | Click/tap/toggle |
| loading | 1000ms | cubic-bezier(0.4, 0, 0.2, 1) | pulse | Data fetching |
| attention | 800ms | cubic-bezier(0.16, 1, 0.3, 1) | scale-pulse | Highlight important |

### CSS Custom Properties

Motion tokens are CSS custom properties (NOT in `@theme` namespace) because they are consumed
by JS animation libraries (GSAP, motion/react), not Tailwind utilities.

```css

  /* Exit */
  --motion-exit-duration: 300ms;
  --motion-exit-easing: cubic-bezier(0.4, 0, 1, 1);

  /* Hover */

  --motion-hover-easing: cubic-bezier(0.3, 1.2, 0.2, 1);

## Layer 3: Integration Context

### DNA -> Archetype

DNA defaults come from the selected archetype's locked palette, approved fonts, and signature element. The archetype skill defines the constraints; DNA is the instantiation of those constraints for a specific project.

```
Archetype "Brutalist" defines:
  locked palette -> fills DNA color tokens
  approved fonts -> fills DNA font stack
  signature element -> fills DNA signature
  tension techniques -> fills DNA tension plan
  arc template -> fills DNA emotional arc
```

### DNA -> Section Builder

Builders receive a DNA snapshot in their spawn prompt. They reference tokens by CSS variable name, never by raw value.

```
Builder reads: --color-primary: #ff6f3c
Builder writes: className="text-primary"     (correct)
Builder writes: className="text-[#ff6f3c]"   (VIOLATION)
```

### DNA -> Anti-Slop Gate

The gate checks that built output uses DNA tokens. Scoring criteria include:

| Gate Check | What It Verifies | Penalty for Violation |
|-----------|-----------------|---------------------|
| Color compliance | All colors reference DNA tokens | -1 per arbitrary hex |
| Font compliance | Only DNA fonts used | -5 for Inter/Roboto as display |
| Spacing compliance | DNA spacing scale followed | -1 per hardcoded value |
| Signature presence | Signature element appears | -3 if missing entirely |
| Motion compliance | DNA motion tokens used | -1 per flat 300ms ease-in-out |

### DNA -> Emotional Arc

The arc template in DNA defines the default beat sequence. Each beat maps to specific section types. The emotional-arc skill defines constraints per beat; DNA provides the project-specific sequence.

### DNA -> Tailwind v4

The `@theme` block in DNA IS the Tailwind configuration. Exact mapping:

```
DNA token "primary: #ff6f3c"
  -> @theme { --color-primary: #ff6f3c; }
  -> Tailwind generates: bg-primary, text-primary, border-primary, ring-primary
  -> Builder writes: className="bg-primary text-text border-border"
```

Key rules:
- `--color-*: initial` in `@theme` resets Tailwind default colors -- the project owns its full palette
- Font families use `--font-display`, `--font-body`, `--font-mono` in `@theme`
- Spacing tokens are NOT in `@theme` (Tailwind's default scale is used with DNA-defined section/block/element patterns)

### DNA -> Motion Libraries

Motion tokens are CSS custom properties consumed by GSAP and motion/react:

```tsx
// GSAP consumption
gsap.from(element, {
  y: 30,
  opacity: 0,
  duration: getComputedStyle(document.documentElement)
    .getPropertyValue('--motion-entrance-duration')
    .replace('ms', '') / 1000,
  ease: 'power3.out', // Maps from --motion-entrance-easing
});

// motion/react consumption
<motion.div
  initial={{ y: 30, opacity: 0 }}
  whileInView={{ y: 0, opacity: 1 }}
  transition={{
    duration: 0.4, // from --motion-entrance-duration
    ease: [0.16, 1, 0.3, 1], // from --motion-entrance-easing
  }}
/>
```

Motion tokens provide the **values**; animation libraries provide the **implementation**. This separation lets DNA define motion personality while builders choose the right tool.

## Layer 4: Anti-Patterns

### Anti-Pattern: Arbitrary Hex Values

**What goes wrong:** Using `#ff6600` or `text-[#ff6600]` instead of `var(--color-primary)` or `text-primary`. Breaks the token system entirely -- impossible to theme, impossible to audit, impossible to maintain consistency across sections built by different agents.
**Instead:** Always reference DNA CSS variables via Tailwind utilities (`bg-primary`, `text-accent`) or CSS variables (`var(--color-primary)`). Zero raw hex values in component code.

### Anti-Pattern: System Font as Display

**What goes wrong:** Using Inter, Roboto, Arial, or system-ui for headings. This is the single strongest "AI slop" signal. Every AI-generated site uses Inter. Visitors recognize it instantly and disengage.
**Instead:** Use the archetype-approved display font from DNA. Every archetype defines 5-8 distinctive fonts. The display font is the fastest way to signal design intentionality.

### Anti-Pattern: Flat Motion Language

**What goes wrong:** Same `300ms ease-in-out` on every animation. No motion personality. The site feels mechanical and lifeless -- like a PowerPoint presentation with "Appear" on every slide.
**Instead:** Use DNA motion tokens. Each token has a specific duration, easing curve, and direction. `entrance` feels different from `hover` which feels different from `micro-interaction`. The variation IS the personality.

### Anti-Pattern: Ignoring Expressive Tokens

**What goes wrong:** Only using the 8 semantic tokens (bg, surface, text, border, primary, secondary, accent, muted). Pages look safe, professional, and completely generic. No creative energy.
**Instead:** Use the 4 expressive tokens (glow, tension, highlight, signature) for creative moments. `--color-glow` for box-shadows on hero CTAs. `--color-tension` for the one rule-breaking moment. `--color-signature` for the signature element. These tokens encode creative intent.

### Anti-Pattern: Hardcoded Spacing

**What goes wrong:** Using `p-4 mt-8 gap-3` based on what "looks right" per element. Inconsistent rhythm across sections. Different builders pick different values. The page has no visual rhythm.
**Instead:** Reference the DNA spacing scale. Section gaps use `--space-xl`. Block gaps use `--space-lg`. Element gaps use `--space-md` or `--space-sm`. The 5-level scale creates consistent rhythm that visitors feel even if they cannot articulate it.

### Anti-Pattern: DNA Drift Mid-Session

**What goes wrong:** Over a long build session, builders gradually deviate from DNA. Colors shift slightly. Spacing loosens. The signature element disappears from later sections. By the end, early and late sections look like different projects.
**Instead:** DNA is immutable during a build session. The build-orchestrator includes a DNA snapshot in every spawn prompt. The quality reviewer checks DNA compliance after every wave. Canary checks detect drift. If drift is detected, the session must reset to the last clean wave.

## Machine-Readable Constraints

| Parameter | Required | Format | Enforcement |
|-----------|----------|--------|-------------|
| Color tokens | 12 (8 semantic + 4 expressive) | CSS variable + hex value | HARD -- reject if any missing |
| Font stack | 3 roles (display, body, mono) | Font family + weight range | HARD -- reject if display is system font |
| Type scale | 8 levels | clamp() + weight + spacing + height | HARD -- reject if levels missing |

| Forbidden patterns | 1+ entries | pattern name + reason | HARD -- reject if empty |
| Tension plan | 1-3 techniques | technique + target beat + approach | HARD -- reject if zero |

| Tailwind @theme | Complete block | `@theme { --color-*: initial; ... }` | HARD -- reject if @theme missing |

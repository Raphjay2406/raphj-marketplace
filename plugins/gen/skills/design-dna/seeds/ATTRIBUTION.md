# Seed Data Attribution

The JSON seed files in this directory are distilled from the UI UX PRO MAX skill
(`nextlevelbuilder/ui-ux-pro-max-skill`, v2.0, MIT License © 2024 Next Level Builder).

## Files

- `uipro-palettes.json` — 160 industry-keyed palettes, remapped from PRO MAX's 17-token schema to Genorah's 12-token DNA structure.
- `uipro-fonts.json` — 73 font pairings (all Google Fonts, OFL-licensed).
- `../../design-brainstorm/seeds/uipro-industry-rules.json` — 161 industry → UI-pattern reasoning rules.

## Remapping Rules (PRO MAX 17-token → Genorah 12-token)

| PRO MAX token | Genorah DNA token | Notes |
|---|---|---|
| Primary | primary | 1:1 |
| Secondary | secondary | 1:1 |
| Accent | accent | 1:1 |
| Background | bg | 1:1 |
| Foreground | text | 1:1 |
| Card | surface | 1:1 |
| Muted | muted | 1:1 |
| Border | border | 1:1 |
| Destructive | tension | semantic shift — used as the expressive "destructive/caution/tension" slot |
| Ring | highlight | semantic shift — focus ring → highlight token |
| On-* variants | — | dropped (derivable from contrast against their base) |
| Muted Foreground, Card Foreground | — | dropped (derivable) |

Genorah's expressive tokens (`glow`, `signature`) are NOT sourced from PRO MAX —
they remain project-specific, derived during DNA generation by the creative-director.

## Full Re-Ingestion

To re-generate seeds from PRO MAX source CSVs:

1. Clone `https://github.com/nextlevelbuilder/ui-ux-pro-max-skill`
2. Locate `src/ui-ux-pro-max/data/{colors,typography,ui-reasoning}.csv`
3. Apply the remapping above (Primary→primary, Card→surface, etc.)
4. Convert CSV rows to JSON entries matching `uipro-palettes.json` schema

The JSON files in this dir are a **representative distillation** (not verbatim full-corpus dump)
covering the most actionable industry/style combinations. For exhaustive lookup,
consult the PRO MAX source directly.

## Attribution Notice for Generated Projects

When Genorah uses a PRO MAX-derived palette in a user project, the `DESIGN-DNA.md`
generated by `design-dna` skill should include:

```yaml
palette_source: "distilled from UI UX PRO MAX (nextlevelbuilder/ui-ux-pro-max-skill, MIT)"
```

This preserves credit without requiring the user to understand the lineage.

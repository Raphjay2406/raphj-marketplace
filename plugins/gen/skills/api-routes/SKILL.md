# API Routes

Template library + generator. `/gen:api <template>` scaffolds route file + test + docs entry.

## Layer 1 — When to use

Any backend-facing endpoint generated by Genorah. Manual hand-edit allowed after scaffold; templates are starting points.

## Layer 2 — Templates

### auth-email-password
Sign-in + sign-up + password-reset. Zod validated. bcrypt via Argon2id. Session via jose (JWT) or next-auth v5.

### auth-magic-link
Email-based OTP + session. Resend or SES transport.

### auth-oauth-callback
Callback handler for Google / GitHub / Apple OAuth. State token verification.

### crud-list
Paginated list with cursor + filter + sort. Zod query schema. Cache 60s.

### crud-detail
Single-record fetch by ID. 404 on missing. Zod param schema.

### crud-mutation
Create / update / delete with Zod body. 201 / 200 / 204 responses.

### webhook-stripe
Stripe webhook. Signature verification via `Stripe.webhooks.constructEvent`. Idempotency via event.id stored in DB.

### webhook-shopify
HMAC verification via `shop + topic + hmac` header.

### webhook-hubspot
OAuth signature v3. Payload includes subscription type.

### webhook-generic
Generic HMAC-signed webhook with replay protection.

### upload-signed-url
Generates presigned URL (S3 / Vercel Blob / Supabase Storage / R2). Zod filename + mime validation.

### rate-limited
Wraps any handler with Vercel Edge Config or Upstash Ratelimit. Token bucket or sliding window.

### ai-stream
Streaming AI response via Vercel AI SDK v6. `StreamingTextResponse`. Optional tool-use.

## Layer 3 — Zod validation pattern

```ts
// app/api/tiers/route.ts
import { z } from 'zod';

const QuerySchema = z.object({
  limit: z.coerce.number().int().min(1).max(100).default(20),
  cursor: z.string().optional(),
});

export async function GET(req: Request) {
  const url = new URL(req.url);
  const parsed = QuerySchema.safeParse(Object.fromEntries(url.searchParams));
  if (!parsed.success) {
    return Response.json({ error: parsed.error.format() }, { status: 400 });
  }
  const { limit, cursor } = parsed.data;
  // ...
}
```

## Layer 4 — Rate limiting

Upstash Ratelimit pattern (recommended; Vercel Edge Config fallback):

```ts
import { Ratelimit } from '@upstash/ratelimit';
import { Redis } from '@upstash/redis';

const ratelimit = new Ratelimit({
  redis: Redis.fromEnv(),
  limiter: Ratelimit.slidingWindow(10, '10 s'),
  analytics: true,
});

export async function POST(req: Request) {
  const ip = req.headers.get('x-forwarded-for') ?? 'anon';
  const { success, limit, remaining } = await ratelimit.limit(ip);
  if (!success) {
    return Response.json({ error: 'rate limit' }, { status: 429, headers: {
      'X-RateLimit-Limit': String(limit),
      'X-RateLimit-Remaining': String(remaining),
    }});
  }
  // ...
}
```

## Layer 5 — OpenAPI generation

Zod schemas feed `@asteasolutions/zod-to-openapi` → emits OpenAPI 3.1 spec at `docs/api/openapi.json`.

`/gen:api docs` rebuilds spec from all routes.

## Layer 6 — Integration

- PLAN.md `api` section lists required endpoints; `/gen:api generate` scaffolds from plan
- Test file generated alongside each route (supertest + vitest)
- Env vars auto-added to `.env.example` per template needs

## Layer 7 — Anti-patterns

- ❌ Handwritten validation instead of Zod — inconsistent errors
- ❌ Rate limit missing on auth endpoints — credential stuffing surface
- ❌ Webhook without signature verification — anyone can spoof
- ❌ 2xx response to unverified webhook (forces retry) — use 401
- ❌ PII in error messages — sanitize

# Genorah v4.5.0 — "Dashboard Drill-Down"

Minor release. **Phase 3 of 3 — completes the dashboard redesign.** The `localhost:4455` cockpit becomes a place you can *investigate* from, not just watch: drill into any section, inspect screenshots full-size, and filter the grid — on the same zero-dependency Node stack.

## Interaction

- **Section detail drawer.** Click (or keyboard-activate) any section card to slide open a detail drawer showing that section's beat + wave, the **full** Verification Spine verdict — **every failing check with its `detail` text** (the grid only shows check names) plus the advisory ceiling — and the section's `SUMMARY.md` + `PLAN.md`. Detail is fetched on demand from a new **`GET /api/section/<name>`** endpoint (a path-locked reader), so the live SSE stream stays lean.
- **Screenshot lightbox.** Click a Visual QA thumbnail to open it full-size, with `←`/`→` to step across the captured breakpoints and `Esc` to close.
- **Section filtering.** Filter the grid by floor verdict (All / Pass / Fail / Unverified) and by a free-text name match; filters apply instantly and persist across live updates.

## Accessibility

Cards and thumbnails are now real controls (`role="button"`, keyboard-activatable with `Enter`/`Space`, visible focus rings). The drawer and lightbox are `role="dialog"` + `aria-modal` with `Esc` to close, a `Tab` focus trap, and focus restored to the triggering element on close. All new motion (drawer slide, lightbox fade) is `prefers-reduced-motion`-gated.

## Architecture

- Pure logic stays in the unit-tested `scripts/dashboard/view-model.mjs`: `filterSections(sections,{q,verdict})` (case-insensitive name + verdict facet, non-mutating) and `buildSectionDetail(raw)` (formats the endpoint payload, **keeping the full `{check,detail}`** failures, parsing beat/wave, mapping pass/fail/none).
- The server gains one lazy, **path-locked** endpoint (`readSectionDetail` + route, traversal-guarded, decode-once with a `400` on malformed percent-encoding). The SSE `snapshot()` contract is **unchanged** — Phase 1 + Phase 2 tests pass untouched; drill-down data never bloats the stream.
- Rendering stays vanilla and **XSS-safe** (`textContent`/element construction; `encodeURIComponent` on the `/api/section` and `/api/screenshot` URLs). The drawer guards against a stale-response race; the lightbox re-clamps its index if the capture set shrinks. Side-effects remain behind the main-guard (no import-hang).

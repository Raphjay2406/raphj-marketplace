# Genorah v4.3.1 — "Dashboard Signals"

Patch on v4.3.0. Phase 1 of the dashboard redesign — the live `localhost:4455` cockpit now reflects the verification pipeline instead of drifting behind it.

## Dashboard signal foundation
- **Section verdicts** — each section card now shows its Verification Spine **FLOOR pass/fail**, the exact failing checks (build/console/overflow/axe/perf/assets/interactions/motion), and the advisory **Ceiling** score, read from each section's `VERDICT.json` (reuses the spine's `readVerdict`). Sections without a verdict render "not verified."
- **Gate Hotspots** — a ranked bar panel of which floor checks fail most across all sections, so you see where the build is weakest at a glance.
- **Visual QA** — a 4-breakpoint screenshot grid from `audit/` (the server's `/api/screenshot/` route, finally used).
- **Real project header** — the hardcoded "Genorah Dashboard" title is replaced with the actual project name + archetype + goal from `PROJECT.md`/`DESIGN-DNA.md`.
- New pure `scripts/dashboard/signals.mjs` (`parseProjectMeta` / `computeHotspots` / `listAuditShots`), unit-tested; all rendering stays vanilla, zero-dependency, and XSS-safe.
- `commands/dashboard.md` corrected — its documented panels now match what actually renders.

## Fix
- The dashboard server's `fs.watch` + `setInterval` are now started only when the server actually runs (behind the main-guard), not on import — importing the module for tests no longer leaks a live timer/watcher that hung the test runner. Full suite completes 220/220 with a clean exit.

## Notes
- Phase 1 of 3: this is the *signal foundation*. The visual redesign (Phase 2) and interaction/drill-down (Phase 3) follow. No Obsidian, graphify, or pipeline behavior changed — the dashboard read-only-consumes artifacts the pipeline already writes.

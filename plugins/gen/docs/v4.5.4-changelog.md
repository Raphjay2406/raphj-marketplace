# Genorah v4.5.4 ‚Äî Bundled MCP servers actually load

Patch on v4.5.3. **Fixes a latent bug: none of the plugin's bundled MCP servers ever loaded.**

## The bug
Claude Code's plugin loader reads a plugin's MCP config from the plugin **root** `.mcp.json` ‚Äî never from `.claude-plugin/.mcp.json`. The gen plugin had its config in `.claude-plugin/.mcp.json` since its first MCP integration, so **none** of its declared servers ‚Äî `gpt-image`, `graphify`, `3dsvg-export`, `flux-kontext`, `recraft`, `next-devtools`, `chrome-devtools`, `stitch`, `playwright` ‚Äî were ever started. Every pipeline silently fell back to text prompts / HTTP adapters.

Diagnosed empirically: zero gen servers in Claude Code's attempted-MCP list for the project, no `mcp-logs-gpt-image` directory (never attempted), while the `gpt-image` bundle itself probes fine (`serverInfo: gpt-image v4.0.0`) and every *other* installed plugin keeps `.mcp.json` at its root.

## The fix
- **Moved `.mcp.json` to the plugin root** (`.claude-plugin/.mcp.json` ‚Üí `.mcp.json`), matching every working plugin. `sync-mirror` now ships it as `plugins/gen/.mcp.json` (a `FILES_TO_SYNC` root file).
- **Consolidated the bundled servers under root `mcp-servers/`** ‚Äî `3dsvg-export` and `meshy` were stranded in `.claude-plugin/mcp-servers/` while `.mcp.json` pointed at `${CLAUDE_PLUGIN_ROOT}/mcp-servers/‚Ä¶`, so `3dsvg-export` would have failed to spawn once the config loaded. All three (`gpt-image`, `3dsvg-export`, `meshy`) now live at `mcp-servers/`.
- **Hardened optional env interpolation** ‚Äî `${GPT_IMAGE_OUTPUT_DIR:-}` (was no-default); an undefined no-default `${VAR}` can make the loader drop the whole server entry. (The server treats empty as `process.cwd()`, identical to unset.)
- **Hook + docs:** the `session-start` availability summary now reads the root `.mcp.json`; `CLAUDE.md`, `commands/self-audit.md`, the user guide, troubleshooting, and the bundled READMEs all point at the root location.

## Guards
Two new regression tests: `.mcp.json` must live at the plugin root (and not in `.claude-plugin/`, and the mirror must expose it at root), and **every declared `${CLAUDE_PLUGIN_ROOT}/mcp-servers/*` bundle path must resolve to a real file**. `check-mirror-parity` now also verifies the root synced files (`.mcp.json` is load-bearing).

## Verification
Full suite **254/254, clean exit**; `lint:syntax`/`lint:json` clean; self-audit BLOCK=0; mirror parity (incl. root files) green. The `gpt-image` bundle starts and responds to `initialize` standalone. Opus whole-branch review: the relocation is correct and verified (path math, env hardening, mirror, JSON all confirmed); its one blocker ‚Äî the exposed `3dsvg-export` path ‚Äî is fixed and test-guarded.

> **To pick this up:** restart Claude Code (or reinstall the plugin) so the loader reads the root `.mcp.json`. With `OPENAI_API_KEY` set, `mcp__gpt-image__*` tools appear after restart.

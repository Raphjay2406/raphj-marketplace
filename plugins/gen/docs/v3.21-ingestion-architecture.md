# v3.21 Project Ingestion Architecture

## Goal

Bring existing projects (codebase or live URL) built **outside** the Genorah pipeline into it **without losing any detail**. Every byte of the source is traceable; every transformation is logged; the original is always diff-able against the derived form.

## Preservation contract

1. The source is captured verbatim under `source/` (codebase) or `captured/` (URL).
2. Every derived artifact (extracted DNA, mapped component, extracted copy) points back to its origin via `preservation.ledger.ndjson`.
3. No silent drops. If something cannot be mapped, it goes to `GAP-REPORT.md` with reason + user-decision prompt.
4. Archetype inference returns top-3 with confidence; never auto-locks.
5. License / PII gates are explicit; ingest blocks until user acknowledges.

## Two inputs, one artifact graph

```
codebase path ──┐
                ├─→ capture → inventory → extract → match → map → ledger → gap → scaffold
live URL    ──┘
```

Codebase route has higher confidence (explicit tokens, component files, config). URL route has runtime-accurate rendering (what users actually see).

## Stages

| # | Stage | Output | Lossy? |
|---|-------|--------|--------|
| 1 | Capture | `source/` or `captured/` tree | No |
| 2 | Inventory | `manifests/{routes,components,assets,fonts}.json` | No |
| 3 | Extract DNA | `DNA-EXTRACTED.md` (per-token confidence) | Derived, with evidence |
| 4 | Archetype match | `ARCHETYPE-MATCH.md` (top-3) | Derived |
| 5 | Map | `DESIGN-SYSTEM.md` + `CONTENT.md` | Reversible via ledger |
| 6 | Ledger | `preservation.ledger.ndjson` (append-only) | N/A (audit trail) |
| 7 | Gap report | `GAP-REPORT.md` | N/A |
| 8 | Scaffold | `.planning/genorah/` standard files | — |

## Artifact layout

```
.planning/genorah/ingested/<slug>/
├── INGESTION.md                     # high-level summary + confidence scores
├── DNA-EXTRACTED.md                 # reverse-engineered DNA w/ per-token confidence
├── ARCHETYPE-MATCH.md               # top-3 archetypes + evidence per score
├── CONTENT.md                       # extracted copy, section-by-section, with origin selector
├── DESIGN-SYSTEM.md                 # component inventory + block-mapping proposals
├── GAP-REPORT.md                    # ambiguities, user-decision prompts
├── preservation.ledger.ndjson       # immutable append-only audit trail
├── source/                          # (codebase) mirror of user's repo at ingest time
├── captured/                        # (URL) HTML/CSS/JS bundle + robots.txt + sitemap.xml
├── screenshots/                     # 4bp × N routes (URL mode only)
├── assets/                          # sha256-named; origin + license in manifest
├── computed-styles/                 # per-selector CSS JSON (URL mode only)
└── manifests/
    ├── routes.json
    ├── components.json              # file → proposed Genorah block mapping
    ├── assets.json                  # sha256 + origin URL + detected license
    └── fonts.json
```

## Ledger format (NDJSON)

```ndjson
{"ts":"...","kind":"route.captured","route":"/","screenshots":4,"source_lines":2400}
{"ts":"...","kind":"dna.extract","token":"color.primary","value":"#1a1a1a","confidence":0.94,"method":"kmeans-frequency"}
{"ts":"...","kind":"archetype.match","archetype":"brutalist","confidence":0.78}
{"ts":"...","kind":"asset.download","url":"https://...","sha256":"...","bytes":42000,"preserved_at":"assets/xxx.jpg"}
{"ts":"...","kind":"content.extract","selector":"h1.hero-title","text":"Welcome","destination":"CONTENT.md:hero.title"}
{"ts":"...","kind":"gap","reason":"license-unknown","asset":"hero.jpg","needs_user":true}
```

## DNA extraction algorithm (stage 3)

### Color palette
1. Pixel-frequency histogram across all breakpoint screenshots (URL mode) or from CSS-variable declarations (codebase mode).
2. K-means (k=12) with perceptual distance (ΔE2000).
3. Assign to semantic slots (bg, surface, text, border, primary, secondary, accent, muted, glow, tension, highlight, signature) by position + frequency + contrast role.
4. Confidence = inverse cluster variance + coverage %.

### Typography
1. Codebase: read `font-family` declarations from CSS + `@font-face` rules.
2. URL: `getComputedStyle().fontFamily` for every element; group by role.
3. Classify display/body/mono by axis (size usage + weight distribution).

### Spacing
1. Bounding-box gap analysis between siblings.
2. Cluster gaps → infer scale steps (e.g., 4 / 8 / 16 / 24 / 48 px).

### Signature element
1. Detect repeating visual motif (shape, gradient, pattern) via screenshot region clustering.
2. Flag as candidate; user confirms in GAP-REPORT.

## Archetype inference (stage 4)

For each of 33 archetypes in `skills/design-archetypes/testable-markers.json`:
- Count matches (regex / CSS-selector / DOM-pattern markers).
- Score = weighted sum; weights from `seeds/archetype-weights.json` (v3.5.5).
- Return top-3 with per-marker evidence. Never auto-lock. User confirms via `/gen:align`.

## Mapping to Genorah primitives (stage 5)

| Source | Destination | Heuristic |
|--------|-------------|-----------|
| `<Hero>` / `section.hero` / first `<section>` | HOOK beat | Position + height |
| `<Features>` / grids with ≥3 cards | BUILD beat | Repeating children |
| `<CTA>` / centered action section | CLOSE beat | Terminal position |
| Stats / large numbers | PEAK beat | Typographic scale |
| Quote / testimonial | PROOF beat | Quotation mark presence |
| Hero copy | `CONTENT.md > HOOK > headline/subheadline` | Heading depth |

Ambiguous matches → `GAP-REPORT.md`; user decides.

## Command surface

```
/gen:ingest codebase <path>                 # full local ingestion
/gen:ingest url <url> --consent             # full URL ingestion (requires explicit consent)
/gen:ingest route <url> --into <slug>       # incremental single-route append
/gen:ingest diff <slug-a> <slug-b>          # compare two ingestions
/gen:ingest gap <slug>                      # print unresolved items
/gen:ingest verify <slug>                   # preservation invariants check
```

## Legal + ethics gates

- `--consent` flag required for `url` mode; refuses without.
- robots.txt honored; routes disallowed for bots skipped with ledger entry `gap:robots-disallow`.
- `pii-regex-v2026` runs on extracted content before persisting to `CONTENT.md`; matches go to `GAP-REPORT.md`.
- Asset `license` field defaults to `unknown`; all unknowns block scaffold stage until user confirms.

## Preservation invariants (verified by `/gen:ingest verify`)

- Every file in `source/` or `captured/` has a corresponding `capture.*` ledger entry.
- Every entry in `manifests/assets.json` has `capture` + either `license.confirmed` or `gap.license-unknown` ledger entry.
- Every `content.extract` has a valid `destination` pointing into `CONTENT.md`.
- No `dna.extract` event has confidence < 0.5 without a paired `gap` entry.

## Non-goals (v3.21)

- Automatic migration to Genorah component code — user reviews `DESIGN-SYSTEM.md` mapping; actual rewrite happens via `/gen:build` or manual work.
- Behavior replay — scraped interactions are inventoried; replay is a v3.22 concern.
- Dynamic-content reconstruction — if site uses CMS/API, we capture what renders; stitching back to the CMS is user work.

## Future (v3.22+)

- Incremental re-ingest (diff new vs last).

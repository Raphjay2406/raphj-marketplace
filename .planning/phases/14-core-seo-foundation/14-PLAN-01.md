4. **React 19: SEOMeta Wrapper** -- Native `<title>`, `<meta>`, `<link>` hoisting pattern. Include ALL limitations inline as comments/notes:
   - `<title>` not deduplicated (only ONE `<SEOMeta>` per route)
   - `<meta>` not deduplicated (same constraint)
   - Cleanup on unmount not guaranteed (stale tags may persist)
   - Title children must be single string (use template literals)
   - Social crawlers see EMPTY head (they don't execute JS)
   Include the `VITE_SITE_URL` env pattern.

**B. Canonical URL Patterns (SEO-03):**

5. **Canonical URL Rules** -- Brief principle section listing the hard rules:
   - Always dynamic from route, never hardcoded
   - Always absolute URLs (https://...)
   - Self-referencing (each page points to itself)
   - Trailing slash consistency (pick one, enforce everywhere)
   - Protocol and www consistency
   - Canonical must match sitemap URL

6. **Next.js 16: metadataBase + Relative Canonical** -- `metadataBase` in root layout, relative `alternates.canonical` in pages. Note the trailing slash normalization behavior.

7. **Astro: Astro.url.href Canonical** -- Show `Astro.url.href` pattern. Note `trailingSlash` config importance (set to 'always' or 'never', never 'ignore').

8. **Pagination Canonicals** -- Self-referencing canonical per page. Note Google deprecated `rel="prev"/"next"` in March 2019. Do NOT canonicalize all pages to page 1.

9. **Cross-Domain Canonicals** -- Brief note on content syndication (canonical on syndicated copy points to original domain).

**C. Sitemap Patterns (SEO-04, SEO-05):**

10. **Next.js 16: Basic Sitemap** -- `app/sitemap.ts` with static + dynamic pages. Include `images` property for image sitemaps.

11. **Next.js 16: Sitemap Index with generateSitemaps** -- Use the verified v16 pattern with `id: Promise<string>` (MUST await). Show `generateSitemaps()` returning array of `{ id }` and the sitemap function receiving async `id`. Note: max 50,000 URLs per file.

12. **Astro: @astrojs/sitemap** -- Integration config with `filter`, `customPages` (required for SSR dynamic routes), `entryLimit`, `i18n`. Note auto-generates `sitemap-index.xml` + `sitemap-0.xml`.

13. **Sitemap Validation Checklist** -- Quick reference of GSC validation rules. Emphasize: non-canonical URLs in sitemap is now an ERROR (Nov 2025 upgrade).

**D. robots.txt Patterns (SEO-06):**

14. **Next.js 16: robots.ts** -- Type-safe `app/robots.ts` with AI bot taxonomy. Separate rules for: standard crawlers (allow with disallow for /api/, /admin/, /_next/), AI search bots (allow), AI training bots (block). Use `process.env.NEXT_PUBLIC_SITE_URL` for sitemap reference. Reference `appendix-ai-bots.md` for the full taxonomy with rationale.

15. **Astro: Static robots.txt** -- `public/robots.txt` with same AI bot taxonomy. Note: Astro uses static file, not programmatic generation.

16. **React/Vite: Static robots.txt** -- Same static file pattern. Note the SPA caveat: robots.txt works fine for crawlers, but the pages it allows may still have SPA rendering limitations.

**E. hreflang Patterns (SEO-08):**

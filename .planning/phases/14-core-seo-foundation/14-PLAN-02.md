**Layer 4: Anti-Patterns must include (6-8 anti-patterns):**

Source from 14-RESEARCH.md "Common Pitfalls" but rewrite in the skill's Layer 4 format (Name / What goes wrong / Instead).

1. **Legacy Head Component** -- Using `import Head from 'next/head'` in App Router. Silently produces zero metadata. Instead: `metadata` export or `generateMetadata`.

2. **Hardcoded Canonical URLs** -- Strings that become stale when routes change, causing duplicate content. Instead: Dynamic from route (metadataBase + relative path, Astro.url.href).

3. **Synchronous Params in Next.js 16** -- Accessing `params.slug` directly without await. Throws runtime error. Instead: `const { slug } = await params`.

4. **Missing og:image** -- Pages share on social with blank preview. CTR drops significantly. Instead: Every page type has default 1200x630 og:image. Set width/height attributes.

5. **Blanket AI Bot Blocking** -- Copy-paste "block all AI bots" hides site from ChatGPT search, Perplexity, DuckAssist. Instead: Separate rules for search bots (allow) and training bots (block). See appendix-ai-bots.md.

6. **Non-Canonical URLs in Sitemap** -- Trailing slash mismatch, www inconsistency between sitemap and canonical. GSC upgraded to ERROR (Nov 2025). Instead: Single source of truth for URL format, validate sitemap against canonicals.

7. **Multiple Title Tags in React 19** -- Multiple components render `<title>`, all appear in `<head>`. Browser behavior undefined. Instead: Single `<SEOMeta>` component per route owns all metadata.

8. **Static lastmod in Sitemap** -- All entries use `new Date()` (build time). Google loses trust in lastmod signal. Instead: Use actual content modification dates from CMS or git.

**Machine-Readable Constraints table:**

After Layer 4, add the enforceable constraint table. These are the SEO parameters quality-reviewer checks:

| Parameter | Min | Max | Unit | Enforcement |
|-----------|-----|-----|------|-------------|
| title length | 30 | 60 | chars | HARD -- reject if outside range |
| meta description length | 70 | 160 | chars | HARD -- reject if outside range |
| og:image width | 1200 | 1200 | px | HARD -- must be exactly 1200px |
| og:image height | 630 | 630 | px | HARD -- must be exactly 630px |
| og:image alt text | 1 | - | present | HARD -- must have alt text |
| canonical URL format | - | - | absolute URL (https://) | HARD -- reject relative URLs |
| metadataBase | - | - | required in root layout | HARD -- reject if missing (Next.js) |
| sitemap URLs per file | - | 50000 | entries | HARD -- split if exceeded |
| sitemap URL = canonical | - | - | boolean match | HARD -- must match exactly |
| hreflang bidirectional | - | - | all alternates reciprocal | HARD -- reject one-way |
| hreflang x-default | - | - | required if hreflang present | HARD -- reject if missing |
| title uniqueness per page | - | - | no duplicate titles across pages | SOFT -- warn if duplicates detected |
| robots.txt sitemap reference | - | - | present | HARD -- reject if missing |

Note: Some constraints are SOFT (warning) vs HARD (rejection). Only use HARD for things that directly cause SEO damage. Title uniqueness is SOFT because homepages and similar landing pages may legitimately share title patterns.

<objective>
Define the Creative Director agent -- a dedicated vision owner with real authority to approve, reject, and push sections to be bolder.

Purpose: v6.1.0 had no creative oversight agent. Quality review focused on technical compliance (anti-slop scores, accessibility, performance) but nobody checked whether the output was actually bold, distinctive, or emotionally compelling. The CD fills this gap by actively reviewing both plans and output at two checkpoints per wave, with authority to require changes.

Output: One agent markdown file that defines the CD's review protocol, authority model, and integration points with the pipeline.

**creative-director.md** (~300-400 lines):
Frontmatter: name: creative-director, description (owns creative vision, actively reviews plans and output against DNA and archetype personality, pushes boldness, catches drift immediately), tools: Read, Write, Edit, Grep, Glob, model: inherit, maxTurns: 30.

System prompt must define:

**Role & Authority:**
The CD is the creative vision owner for the project. DNA is the floor (minimum compliance), CD pushes the ceiling (opportunities to be bolder within the archetype). CD has real authority: can flag sections as "below creative bar" and require specific improvements before the wave advances. This is NOT advisory -- flagged sections create GAP-FIX.md files that the polisher must address.

**Clear separation from Quality Reviewer:**
- CD owns: Creative vision, boldness, archetype personality, creative tension moments, emotional arc movement, color journey, typography drama, compositional interest, wow moment impact
- CD does NOT own: Anti-slop scoring, accessibility compliance, performance metrics, code quality, Lighthouse scores
- DNA is the floor (QR enforces technical compliance), CD pushes the ceiling (creative excellence)

**Input contract:**
- Reads: DESIGN-DNA.md, BRAINSTORM.md, current wave PLAN.md files, built code for current wave sections
- Also reads: CONTEXT.md (own creative direction notes from previous waves)
- Does NOT read: STATE.md (orchestrator territory), CONTENT.md directly (content is in plans), research files (already distilled into DNA/brainstorm)

**Output contract:**
- Writes: Creative direction notes section in CONTEXT.md
- Creates: `.planning/genorah/sections/{XX-name}/GAP-FIX.md` when section is below creative bar
- Updates: Can suggest modifications to PLAN.md files during pre-build review

**Two-checkpoint review protocol per wave:**

1. Pre-Build Review (LIGHT -- blocking, happens before builders execute):
   - Read current wave's PLAN.md files
   - Check against archetype personality: Does this section FEEL like the archetype?
   - Check creative tension placement: Are tension moments bold enough for assigned beats?
   - Check wow moment ambition: Are wow moments genuinely impressive, not timid fallbacks?
   - Check beat sequence movement: Does the sequence create real emotional movement?
   - Check layout diversity: Are patterns distinct enough between sections?
   - Output: "APPROVED" or specific revision notes for section-planner to address
   - This review should be FAST (~5 min) -- scan plans for creative ambition, not deep analysis

2. Post-Build Review (THOROUGH -- happens after all wave builders complete):
   - Read built code for each section in the wave
   - Evaluate each section against 8 creative dimensions:
     a. Archetype personality: Is the archetype's DNA visible in every design choice?
     b. Creative tension: Were tension moments implemented boldly (not watered down)?
     c. Emotional arc: Does this beat hit the right emotional note?
     d. Color journey: Does color usage create visual progression (not monotone)?
     e. Typography drama: Does type create real hierarchy and visual interest?
     f. Compositional interest: Is the layout compositionally interesting (not default centered)?
     g. Wow moment impact: Would you screenshot this section?
     h. Transition quality: Does this section flow naturally from/to its neighbors?
   - For each section: ACCEPT or FLAG with specific issues and required improvements

**Flag format (creates GAP-FIX.md):**
```
SECTION: [XX-name]
STATUS: BELOW_CREATIVE_BAR
REVIEWER: creative-director
ISSUES:
  - [specific issue with evidence from code]
  - [what's missing and why it matters for the archetype]
REQUIRED_IMPROVEMENTS:
  - [concrete action: "Replace centered layout with asymmetric grid, offset hero image 15% left"]
  - [concrete action: "Increase type scale contrast between headline and body by 2 steps"]
BOLDNESS_OPPORTUNITIES:
  - [specific suggestion to push further: "Add scale-violence tension at 1.3x on the hero stat"]
```

**Creative Direction Notes format (written to CONTEXT.md):**
```
## Creative Direction Notes (Wave [N])
Overall: [1-2 sentence creative assessment of the wave]
Strengths: [What sections did well creatively]
Drift: [Any creative drift detected, with evidence]
Push: [Specific opportunities to be bolder in upcoming sections]
Calibration: [Any adjustments to tension level or boldness target]
```

**Embedded creative review checklist (do NOT reference external skills):**
- Is there at least one "screenshot moment" per wave?
- Are forbidden patterns absent? (Not just DNA tokens -- archetype forbidden patterns)
- Is the signature element present and evolved (not just repeated identically)?
- Do creative tensions actually create tension (not just mild variation)?
- Would this page win a section-by-section comparison against the reference quality targets?

**DNA-as-floor philosophy:**
When reviewing, the CD should actively look for places where the output is "correct but boring." DNA compliance is necessary but not sufficient. The CD asks: "Is this the BOLDEST version of this archetype that still works?" If the answer is "it could be bolder," the CD should push.

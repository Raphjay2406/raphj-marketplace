**3d-specialist.md** (~250-300 lines):
Frontmatter: name: 3d-specialist, description (implements sections requiring Three.js/Spline/WebGL with progressive enhancement and performance handling), tools: Read, Write, Edit, Bash, Grep, Glob, model: inherit, maxTurns: 30.

Inherits everything from section-builder pattern, PLUS embedded domain knowledge:

React Three Fiber Integration:
- Canvas setup with proper Suspense boundaries
- @react-three/drei helpers: Environment, ContactShadows, Float, Text3D, useGLTF
- @react-three/fiber hooks: useFrame, useThree
- Camera and lighting setup per archetype mood
- Proper disposal and cleanup in useEffect

Spline Integration:
- @splinetool/react-spline import pattern
- Loading states with DNA-themed skeleton
- Event handling (onMouseDown, onMouseHover on Spline objects)
- Proper sizing and responsive container

WebGL Shaders:
- Noise displacement (simplex/perlin noise on geometry)
- Liquid distortion (sine wave vertex manipulation)
- Holographic effects (iridescent color shifting)
- Custom ShaderMaterial pattern with uniforms
- useFrame for animation loop

Progressive Enhancement (CRITICAL):
- ALWAYS provide static image fallback for no-JS/no-WebGL
- Mobile detection: check navigator.maxTouchPoints, screen width, deviceMemoryAPI
- Mobile downgrade: reduce geometry complexity (polycount /4), disable post-processing, lower resolution
- Loading strategy: lazy load Three.js bundle, show static image first, swap to 3D when ready
- Bundle splitting: dynamic import() for all Three.js code
- FPS monitoring: if FPS drops below 30 for 2+ seconds, auto-downgrade to static

Performance Rules (3D-specific):
- Max triangle count: 50k for hero, 20k for inline sections
- Texture size: max 2048x2048, prefer 1024
- No uncompressed GLTF (always use .glb or Draco compression)
- requestAnimationFrame cleanup on unmount
- Dispose geometries, materials, textures on unmount

**animation-specialist.md** (~250-300 lines):
Frontmatter: name: animation-specialist, description (implements sections with complex choreographed animations using GSAP/Framer Motion/scroll-driven CSS), tools: Read, Write, Edit, Bash, Grep, Glob, model: inherit, maxTurns: 30.

Inherits everything from section-builder pattern, PLUS embedded domain knowledge:

GSAP ScrollTrigger:
- Import pattern: gsap.registerPlugin(ScrollTrigger) in useEffect
- Timeline construction with labels for choreography
- scrub vs. toggle vs. snap configurations per beat type
- Pin patterns for immersive scroll sections
- Batch animations for staggered lists
- Kill ScrollTrigger instances on unmount (cleanup)

Framer Motion Orchestration:
- Variant chains with staggerChildren
- AnimatePresence for enter/exit choreography
- useScroll + useTransform for scroll-linked values
- useInView for viewport-triggered sequences
- LayoutGroup for shared layout animations
- motion.div with whileInView for scroll reveal

Scroll-Driven CSS Animations (DEFAULT PATH):
- animation-timeline: scroll() and view() syntax
- animation-range for precise trigger points
- @supports (animation-timeline: scroll()) feature detection
- Fallback to Intersection Observer for unsupported browsers
- scroll-snap integration for section-based navigation

Choreography Patterns:
- Stagger reveals: elements enter with 50-100ms delays (not all at once)
- Parallax layers: background moves slower than foreground (parallax ratio per archetype)
- Text splitting: character/word/line animation with stagger
- Counter animations: number counting for stats sections
- Morphing shapes: clip-path and SVG morph for transitions

Performance Rules (animation-specific):
- CSS animations for opacity + transform (GPU-composited)
- JS animations ONLY for complex choreography (will-change for affected properties)
- prefers-reduced-motion: reduce to simple fade, disable parallax, stop auto-playing
- No animation on elements below fold until they enter viewport
- Stagger delays < 200ms per element (avoid slow reveals)
- Total animation duration per section < 3s (entrance complete within 3s of visibility)

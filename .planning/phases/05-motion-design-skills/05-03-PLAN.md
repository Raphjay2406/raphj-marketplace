**Layer 3: Integration Context** (~30-40 lines):
- **-> Cinematic Motion**: Wow moments inherit the archetype's base motion profile (easing, duration). A Kinetic wow moment is snappy; a Luxury wow moment is languid.
- **-> Creative Tension**: Some wow moments ARE tension moments (e.g., physics playground = Interaction Shock). When a wow moment serves as tension, it must follow tension adjacency rules.
- **-> Emotional Arc**: PEAK beat wow moments should be the most impressive on the page. HOOK wow moments set the tone. BREATHE wow moments are ambient-only.
- **-> Design DNA**: All wow moments use DNA color tokens. Gradient mesh uses `--color-primary`, `--color-accent`, `--color-glow`. Particle fields use `--color-accent` at low opacity.
- **-> Performance-Aware Animation**: Tier 3 wow moments (WebGL, Rive, dotLottie) require code-splitting. Never bundle Three.js/R3F on page load for a single wow moment.

**Layer 4: Anti-Patterns** (~30-40 lines):
- **Anti-Pattern: Wow Moment Overload** -- 6 wow moments on a page for a Swiss archetype. Why bad: overwhelms the design, contradicts archetype personality. Instead: respect the archetype intensity modifier table.
- **Anti-Pattern: Wrong Category for Archetype** -- Cursor-responsive wow moments in a Japanese Minimal design. Why bad: too playful for the archetype. Instead: ambient-only for minimal archetypes.
- **Anti-Pattern: All Tier 1** -- Only using simple copy-paste patterns when the archetype demands impressive moments. Why bad: Tier 1 patterns are delightful but not PEAK-worthy alone. Instead: PEAK beats in high-intensity archetypes should use Tier 2-3 patterns.
- **Anti-Pattern: Missing Reduced Motion** -- Implementing a particle field without `prefers-reduced-motion` handling. Why bad: accessibility failure. Instead: every pattern must degrade gracefully (static state, no animation, or simpler animation).
- **Anti-Pattern: Arbitrary Colors** -- Gradient mesh with hardcoded hex values. Why bad: breaks DNA token system. Instead: use CSS custom properties from DNA.

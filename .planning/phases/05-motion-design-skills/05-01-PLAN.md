**Layer 1: Decision Guidance** (~80-100 lines):

1. **CSS-First Motion Decision Tree** -- The primary decision framework. Copy the tree from RESEARCH.md (Pattern 1) and expand it:
   - Is this scroll-linked? -> CSS scroll-driven (DEFAULT) -> Motion useScroll (velocity) -> GSAP ScrollTrigger (complex)
   - Is this entrance/exit? -> Motion variants + whileInView
   - Is this complex choreography (5+ elements)? -> GSAP timeline
   - Is this hover/interaction? -> CSS transitions (simple) -> Motion (complex)
   - Is this ambient/continuous? -> CSS @keyframes
   - Include decision: "If you're unsure, use CSS. Escalate to JS only when CSS literally cannot do it."

2. **Motion Intensity by Archetype** -- Table showing each archetype's motion profile category:
   - HIGH intensity: Kinetic, Neon Noir, Vaporwave, Playful/Startup
   - MEDIUM intensity: Editorial, Neo-Corporate, Retro-Future, Luxury/Fashion, Glassmorphism, AI-Native
   - LOW intensity: Swiss/International, Japanese Minimal, Warm Artisan
   - BOLD intensity: Brutalist, Neubrutalism, Dark Academia
   - VARIABLE: Organic, Ethereal, Data-Dense (context-dependent)

3. **Beat-Dependent Scroll Behavior** -- Table:
   - HOOK: Continuous scroll-linked (parallax, progress-based transforms)
   - PEAK: Continuous scroll-linked (scroll-jacking, progress reveals)
   - BUILD: Entrance-only reveals (staggered children)
   - TEASE: Entrance-only reveals (subtle)
   - REVEAL: Entrance with emphasis (scale + opacity)
   - PROOF: Entrance-only (counters, testimonial cards)
   - BREATHE: Minimal or none (ambient only if any)
   - TENSION: Continuous (tied to tension technique)
   - PIVOT: Entrance with direction change
   - CLOSE: Entrance-only (gentle)

4. **Motion Diversity Enforcement Rules:**
   - No 3 consecutive same-direction animations (e.g., all "rise from bottom")
   - No 3 consecutive same-timing animations (all 0.5s ease-out)
   - Adjacent sections must differ in at least ONE of: direction, timing, stagger pattern
   - The design-lead agent validates this in MASTER-PLAN.md pre-assignment

**Layer 2: Award-Winning Examples** (~250-350 lines):

1. **Hybrid Motion Preset Model** (~40 lines):
   Show the archetype-base + DNA-tweak model from RESEARCH.md (Pattern 4). Include:

2. **CSS Scroll-Driven Animation Patterns** (~60 lines):
   All code uses modern CSS (NOT framer-motion). Include `@supports` progressive enhancement for EVERY pattern.

   - Progress indicator (scroll() timeline + scaleX)
   - Sticky header transform (scroll() timeline with threshold)
   - Browser support note: Chrome 115+, Edge 115+, Safari 26+, Firefox flagged. Always wrap in `@supports (animation-timeline: view())`

3. **Motion Library (ex-Framer Motion) Patterns** (~50 lines):

   - Bundle optimization: `import { LazyMotion, m, domAnimation } from 'motion/react'`
   - Variant-based entrance with whileInView
   - Stagger children pattern
   - useScroll + useTransform for velocity-responsive parallax
   - Note: `framer-motion` package is deprecated, use `motion` package

7. **Reduced Motion Handling** (~15 lines):
   - CSS: `@media (prefers-reduced-motion: reduce)` removes all animation
   - Motion library: `import { useReducedMotion } from 'motion/react'`
   - GSAP: `gsap.matchMedia()` with `(prefers-reduced-motion: reduce)`
   - Every pattern in the skill must document its reduced-motion fallback

**Layer 3: Integration Context** (~40-50 lines):
- **-> Design DNA**: Motion tokens from DNA define per-project tweaks to archetype base presets. CSS custom properties: `--motion-{token}-duration`, `--motion-{token}-easing`
- **-> Emotional Arc**: Beat type determines scroll behavior (continuous vs entrance-only). See beat table in Layer 1.
- **-> Design System Scaffold**: Scaffold generates Tailwind `@theme` motion presets from DNA. Builders use `animate-*` utilities.
- **-> Wow Moments**: Wow moments use motion presets as their foundation. Complex wow moments escalate from CSS to GSAP.
- **-> Page Transitions**: Page-level motion uses the same easing/duration from archetype profile. Cross-skill consistency.
- **-> Creative Tension**: Tension moments may override normal motion patterns (e.g., TENSION beat gets continuous scroll even if beat table says entrance-only).

**Layer 4: Anti-Patterns** (~50-60 lines):
- **Anti-Pattern: All-JS Motion** -- Using `motion` library for simple scroll reveals when CSS `animation-timeline: view()` does it with zero JS on the compositor thread. Why bad: unnecessary bundle weight, main-thread work. Instead: CSS first, escalate to JS only when needed.
- **Anti-Pattern: Generic Fade-In Everywhere** -- Every element does `opacity: 0 -> 1, translateY: 20px -> 0`. Why bad: monotonous, no motion personality. Instead: vary direction, timing, and stagger per the diversity rules.
- **Anti-Pattern: will-change Abuse** -- Applying `will-change: transform` to 20+ elements permanently. Why bad: excessive GPU memory, compositor layer explosion. Instead: apply dynamically via JS before animation, remove after.
- **Anti-Pattern: framer-motion Imports** -- Using `import from 'framer-motion'` instead of `import from 'motion/react'`. Why bad: deprecated package. Instead: always use `motion/react`.
- **Anti-Pattern: GSAP Premium Mentions** -- Referring to SplitText/MorphSVG as "paid plugins." Why bad: outdated since 2024 Webflow acquisition. Instead: all GSAP plugins are free for all use.
- **Anti-Pattern: Safari 18 Assumption** -- Assuming CSS scroll-driven animations work in Safari 18. Why bad: they require Safari 26+. Instead: always use `@supports (animation-timeline: view())` progressive enhancement.
- **Anti-Pattern: Same Motion Every Section** -- Breaking the diversity rule (3+ consecutive same-direction). Why bad: visual monotony, "AI-generated" feel. Instead: pre-assign motion directions in MASTER-PLAN.md.

**Layer 1: Decision Guidance** (~50-70 lines):

1. **When This Skill Runs:**
   - Wave 0 ONLY. This is the first thing built before any section builders run.
   - Triggered by the build-orchestrator agent after DESIGN-DNA.md and MASTER-PLAN.md exist.
   - Input: `.planning/modulo/DESIGN-DNA.md` (color tokens, type scale, spacing, motion language, signature element)
   - Output: Complete set of scaffold files in the target project

2. **What Gets Generated (File Manifest):**
   | File | Purpose | Source |
   | `app/globals.css` | @theme block with all DNA tokens, @keyframes, reduced-motion baseline, @font-face | DNA color, type, spacing, motion tokens |
   | `lib/tokens.ts` | TypeScript types and utilities for DNA tokens (colors, spacing, type scale) | DNA token definitions |
   | `lib/motion.ts` | Motion preset utilities (archetype profile, DNA-tweaked easings/durations) | DNA motion language + archetype profile |
   | `lib/beats.ts` | Beat template definitions (constraints per beat type) | Emotional arc from DNA |
   | `components/ui/section-wrapper.tsx` | Section container with beat-aware defaults | Beat templates |

   Optional (if archetype uses them):
   | `lib/effects.ts` | Signature element utilities, texture patterns | DNA signature element |

3. **Hard Enforcement Philosophy:**
   "Making slop structurally impossible beats catching slop in review."
   - Typed utilities ONLY accept DNA token values. `bg('primary')` works, `bg('#ff0000')` is a TypeScript error.
   - Tailwind @theme maps DNA tokens to utilities. Builders use `bg-primary` not `bg-[#ff0000]`.
   - Beat templates define constraints as typed objects. A HOOK beat template says `minHeight: '90vh'` -- the section-wrapper enforces it.
   - Arbitrary values (`[#hex]`, `[400px]`) should be caught by the quality reviewer, but the scaffold makes them unnecessary.

4. **Extension Mechanism:**
   When a builder needs a token not in the scaffold:
   - Builder adds the token to `lib/tokens.ts` with a `// EXTENDED: reason` comment
   - Builder adds the Tailwind mapping to globals.css @theme block
   - Quality reviewer verifies the extension is justified (not a DNA violation)
   - Design system grows organically while staying typed and intentional

   import { type BeatType, beatTemplates } from '@/lib/beats'

   interface SectionWrapperProps {
     beat: BeatType
     children: React.ReactNode
     className?: string
     id?: string
   }

   export function SectionWrapper({ beat, children, className, id }: SectionWrapperProps) {
     const template = beatTemplates[beat]

     return (
       <section
         id={id}
         className={cn(
           'relative w-full',
           className,
         )}
         style={{ minHeight: template.minHeight }}
         data-beat={beat}
         data-scroll-behavior={template.scrollBehavior}

**Layer 3: Integration Context** (~40-50 lines):
- **-> Design DNA**: The scaffold IS the DNA made real. Every token in DESIGN-DNA.md becomes a CSS variable, Tailwind utility, and TypeScript type.
- **-> Build Orchestrator**: The build-orchestrator spawns a scaffold builder at Wave 0 with the DNA as context. This skill tells that builder exactly what to generate.
- **-> Section Builders**: Every section builder inherits the scaffold. They use `bg('primary')` not `bg-[#ff0000]`. They wrap sections in `<SectionWrapper beat="hook">`. The scaffold constrains them toward quality.
- **-> Cinematic Motion**: Motion presets in the scaffold (`animate-reveal`, `animate-rise`) come from the archetype's motion profile. The scaffold's `lib/motion.ts` provides Motion library variants matching these presets.
- **-> Emotional Arc**: Beat templates in `lib/beats.ts` encode the hard constraints from the Emotional Arc skill. The section-wrapper enforces minHeight and exposes data attributes.
- **-> Anti-Slop Gate**: The gate verifies: are builders using DNA tokens (typed utilities) or arbitrary values? The scaffold makes typed utilities the path of least resistance.

**Layer 4: Anti-Patterns** (~40-50 lines):
- **Anti-Pattern: Tailwind v3 Config** -- Generating `tailwind.config.ts` with `theme.extend.colors`. Why bad: Tailwind v4 uses CSS-first `@theme` directive. JS config is legacy. Instead: all tokens in `@theme` block in globals.css.
- **Anti-Pattern: Arbitrary Hex Values** -- Using `bg-[#ff6600]` or `text-[#333]` in components. Why bad: breaks token system, impossible to theme, anti-slop gate catch. Instead: use `bg-primary`, `text-text`, or typed utility `bg('primary')`.
- **Anti-Pattern: Hardcoded Spacing** -- Using `p-4 mt-8 gap-6` with Tailwind default spacing. Why bad: spacing doesn't follow DNA rhythm. Instead: use `p-spacing-sm gap-spacing-md` from DNA scale.
- **Anti-Pattern: Missing Beat Constraints** -- Section ignoring beat template minHeight or element count. Why bad: breaks emotional arc. Instead: use `<SectionWrapper beat="hook">` which enforces minHeight.
- **Anti-Pattern: System Font Display** -- Using `font-sans` (Inter/system-ui) for display headings. Why bad: instant "generic AI" signal. Instead: use `font-display` which maps to the archetype's chosen display font.
- **Anti-Pattern: Motion Presets Without DNA** -- Hardcoding easing values like `ease: [0.16, 1, 0.3, 1]` in components. Why bad: not DNA-connected, can't adjust globally. Instead: reference `var(--ease-default)` or use `lib/motion.ts` presets.
- **Anti-Pattern: No Extension Comment** -- Adding new tokens without `// EXTENDED: reason` comments. Why bad: quality reviewer can't distinguish intentional extensions from DNA violations. Instead: always annotate extensions.

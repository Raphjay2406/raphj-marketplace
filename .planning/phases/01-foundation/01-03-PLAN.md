**Layer 1: Decision Guidance** (~60-80 lines):

Opening statement: You are a creative director who uses personality archetypes to guarantee distinctive visual identity. Each archetype is a CONSTRAINT SYSTEM -- it defines what you MUST use and what you CANNOT use.

**How Archetypes Work:**
1. During brainstorm (via `/gen:start-project`), present 2-3 relevant archetypes based on project type/industry
2. User selects one or requests custom
3. Archetype constraints populate Design DNA document
4. ALL section builders must follow archetype rules throughout the build
5. Tension override allows ONE intentional rule break per page

**Archetype Selection Guide** -- table format:
| Industry/Project Type | Primary Recommendation | Secondary Options |
Show 10-12 rows covering: SaaS/Tech, E-commerce, Creative Agency, Portfolio, Corporate, Education, Gaming, Finance, Food/Restaurant, Media/Publishing, Healthcare, Developer Tools

**When to Recommend Custom:** User describes something that doesn't map to any archetype, OR wants to hybridize 2+ archetypes.

**Layer 3: Integration Context** (~40-50 lines):
- **Archetype -> DNA**: Locked palette becomes DNA color tokens. Required fonts become DNA font stack. Signature element becomes DNA signature. Tension techniques become DNA tension plan.
- **Archetype -> Emotional Arc**: Each archetype has a default arc template (defined in emotional-arc skill). Some archetypes modify beat behavior (e.g., Japanese Minimal reduces HOOK height).
- **Archetype -> Anti-Slop Gate**: Forbidden patterns are checked during verify. Penalty: -5 per violation.
- **Archetype -> Section Builder**: Builder spawn prompt includes archetype name + constraint summary. Builder references full archetype block for details.

**Tension Override Mechanism** (~20 lines):
Define the escape hatch clearly:
- Builders may intentionally break ONE mandatory rule per page IF:
  1. The break creates deliberate creative tension (not laziness)
  2. The rationale is documented in SUMMARY.md
  3. The specific rule broken is named (not "general override")
  4. It aligns with one of the archetype's 3 tension techniques
- Quality reviewer checks: is the override intentional? Does rationale make sense? Does it IMPROVE the design?
- If override is lazy (no rationale, doesn't improve design), treat as violation

**Custom Archetype Builder** (~30-40 lines):
Two modes:
1. **Structured wizard**: User provides personality keywords, color preferences, font preferences, reference sites. System generates archetype constraint block.
2. **Reference derivation**: User says "I want it to feel like [site URL]". System analyzes the site's visual language and generates matching constraints.

Custom archetype output format matches the standard archetype block structure (same table format, same sections). Custom archetypes are stored in the project's DESIGN-DNA.md, not in the skill file.

**Layer 4: Anti-Patterns** (~30-40 lines):
- **Anti-Pattern: Archetype Mixing Without Intent** -- Applying Brutalist typography with Ethereal colors. Why bad: creates visual confusion, not tension. Instead: pick ONE archetype, use tension override for intentional breaks.
- **Anti-Pattern: Ignoring Forbidden Patterns** -- Using gradient backgrounds in a Brutalist project. Why bad: destroys archetype personality. Instead: check forbidden list before every design decision.
- **Anti-Pattern: Bland Custom Archetype** -- Custom archetype that's just "clean, modern, minimal". Why bad: produces the exact generic output Genorah exists to prevent. Instead: every custom archetype needs at least 3 distinctive techniques and 3 forbidden patterns.
- **Anti-Pattern: Same Archetype Every Project** -- Defaulting to Neo-Corporate for everything. Why bad: Genorah exists for DISTINCTIVE output. Instead: match archetype to project personality, push beyond comfort zone.

**The 19 archetypes (in order):**

1. **Brutalist** -- Raw, exposed, bold. Monospace, no rounded corners, thick borders.
2. **Ethereal** -- Dreamlike, soft, atmospheric. Thin serif, pastels, blur effects.
3. **Kinetic** -- Motion-driven, energetic, alive. Bold sans, vibrant colors, constant animation.
4. **Editorial** -- Magazine precision, content-first. Strong serif, dramatic type scale, grid mastery.
5. **Neo-Corporate** -- Elevated professional. Clean sans, muted palette, subtle polish.
6. **Organic** -- Natural, handcrafted, textured. Rounded forms, earth tones, grain textures.
7. **Retro-Future** -- Nostalgic futurism. Vintage palette, CRT effects, sci-fi typography.
8. **Luxury/Fashion** -- Haute couture digital. Thin serif, dramatic contrast, photography-first.
9. **Playful/Startup** -- Energetic, approachable, fun. Rounded sans, bright colors, bouncy motion.
10. **Data-Dense** -- Information-rich, dashboard-like. Mono + sans, cool palette, tight spacing.
11. **Japanese Minimal** -- Restraint as expression. Generous whitespace, subtle motion, careful typography.
12. **Glassmorphism** -- Frosted layers, depth illusion. Glass panels, blur, translucent surfaces.
13. **Neon Noir** -- Dark, electric, cinematic. Black backgrounds, neon accents, glow effects.
14. **Warm Artisan** -- Handmade, approachable, craft. Warm palette, texture, rounded elements.
15. **Swiss/International** -- Grid perfection, clean logic. Grotesk fonts, strict grid, no decoration.
16. **Vaporwave** -- Digital nostalgia, surreal. Purple-pink palette, retro elements, glitch effects.
17. **Neubrutalism** -- Brutalism's playful evolution. Thick borders, hard shadows, saturated colors, warm off-white backgrounds. (Use RESEARCH.md content -- MEDIUM confidence, verified against multiple sources)

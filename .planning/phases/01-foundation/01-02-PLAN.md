**Layer 1: Decision Guidance** (~60-80 lines):
- When to generate DNA: after brainstorm, before section planning. DNA is a PREREQUISITE for all building.
- DNA is generated by the creative-director agent during `/modulo:start-project`
- How DNA connects to archetypes: archetype selection populates DNA defaults, user customizes from there
- Decision tree for token selection:
  - Start from archetype locked palette
  - User can override any token (within archetype constraints unless using tension override)
  - Font selection from archetype-approved list
  - Signature element chosen from archetype signature + user input
- When to reference DNA: every section builder reads DNA before writing any code
- DNA is immutable during a build session (changes require new session)

**Layer 2: Award-Winning Examples** (~120-160 lines):

**Code Pattern: Complete DESIGN-DNA.md Template**
Show the full markdown template that gets generated in target projects at `.planning/modulo/DESIGN-DNA.md`. This is the core deliverable of this skill. Include:

1. **Color System** section:
   - Semantic Core (8 tokens) table: Token | CSS Variable | Hex Value | Usage | Tailwind Utility
   - Expressive Tokens (4 tokens) table: same columns
   - Tailwind v4 Integration: full `@theme` block showing all 12 tokens mapped to `--color-*` namespace
   - Note: use `--color-*: initial;` first to reset Tailwind defaults, then define project tokens

2. **Typography System** section:
   - Font Stack table: Role (Display/Body/Mono) | Font Family | Weight Range | Google Fonts Import
   - Type Scale table (8 levels): Level Name | Size (clamp) | Weight | Letter-spacing | Line-height | Usage
   - Levels: hero, h1, h2, h3, h4, body-large, body, small
   - All sizes use `clamp()` for responsive scaling (e.g., `clamp(3rem, 8vw, 6rem)` for hero)

3. **Spacing System** section:
   - 5-level spacing scale: xs, sm, md, lg, xl with rem values
   - Section padding pattern (vertical rhythm)

4. **Signature Element** section:
   - Named pattern format: `name: parameter=value, parameter=value`
   - Example: `diagonal-cut: angle=12deg, frequency=every-section, color=accent`
   - Machine-readable: name is lookup key, parameters are enforceable

5. **Motion Language** section:
   - 8 motion tokens table: Token Name | Duration | Easing (cubic-bezier) | Direction | Trigger
   - Tokens: entrance, exit, hover, scroll-reveal, page-transition, micro-interaction, loading, attention
   - CSS custom property format: `--motion-{token}-duration`, `--motion-{token}-easing`, `--motion-{token}-direction`
   - Note: motion tokens are CSS custom properties (NOT in @theme namespace) because they're consumed by JS animation libraries

6. **Forbidden Patterns** section:
   - List format: pattern name + why it's forbidden (per archetype)
   - Example: "gradient-backgrounds: Contradicts Brutalist raw aesthetic"

7. **Tension Plan** section:
   - 3 tension techniques from archetype
   - Assigned to specific beat positions (e.g., tension-1 at PEAK beat)

8. **Emotional Arc Template** section:
   - Default beat sequence from archetype (e.g., HOOK -> TEASE -> BUILD -> PEAK -> BREATHE -> PROOF -> CLOSE)

**Reference Sites:**

  - Stripe.com -- typography scale discipline, signature gradients

**Layer 3: Integration Context** (~60-80 lines):
- **DNA -> Archetype**: DNA defaults come from archetype locked palette/fonts/signature
- **DNA -> Section Builder**: Builders receive DNA snapshot in spawn prompt, reference tokens by CSS variable name
- **DNA -> Anti-Slop Gate**: Gate checks that built output uses DNA tokens (not arbitrary hex values)
- **DNA -> Emotional Arc**: Arc template in DNA defines default beat sequence
- **DNA -> Tailwind v4**: The `@theme` block in DNA IS the Tailwind configuration. Show exact mapping:
  ```
  DNA token "primary: #ff6f3c" -> @theme { --color-primary: #ff6f3c; } -> Tailwind "bg-primary", "text-primary"
  ```
- **DNA -> Motion Libraries**: Motion tokens consumed as CSS custom properties by GSAP/Framer Motion:
  ```
  DNA token "entrance: 400ms ease-out from-bottom" -> CSS "--motion-entrance-duration: 400ms" -> GSAP `gsap.from(el, { duration: 'var(--motion-entrance-duration)' })`
  ```

**Layer 4: Anti-Patterns** (~40-60 lines):
- **Anti-Pattern: Arbitrary Hex Values** -- Using `#ff6600` instead of `var(--color-primary)`. Why bad: breaks token system, impossible to theme. Instead: always reference DNA CSS variables.
- **Anti-Pattern: System Font as Display** -- Using Inter/Roboto/system-ui for headings. Why bad: instant "AI slop" signal. Instead: use archetype-approved display font.
- **Anti-Pattern: Flat Motion** -- Same 300ms ease-in-out for everything. Why bad: no motion personality. Instead: use DNA motion tokens with varied durations and easings per context.
- **Anti-Pattern: Ignoring Expressive Tokens** -- Only using the 8 semantic tokens. Why bad: pages look safe and generic. Instead: use glow, tension, highlight, signature tokens for creative moments.
- **Anti-Pattern: Hardcoded Spacing** -- Using `p-4 mt-8` instead of DNA spacing tokens. Why bad: inconsistent rhythm, can't adjust globally. Instead: reference DNA spacing scale.

**Target length: 400-550 lines.** This is a Core tier skill -- the most important knowledge base in the plugin.

**Layer 1: Decision Guidance** (~50-60 lines):

Opening: You are a narrative designer who treats every page as a story. Each section is a BEAT with a specific emotional role and measurable parameters. Pages that list features are forgettable. Pages with emotional arcs convert.

**When to Use:**
- During section planning (`/modulo:plan-dev`): assign beat types to each section
- During section building: builders read beat constraints and build within parameters
- During verify: quality-reviewer checks beat parameter compliance

**How Arc Planning Works:**
1. Section planner assigns beat types based on content and desired emotional flow
2. Each beat has HARD constraints (must be within range) and SOFT constraints (recommended, flexible)
3. Archetype may override specific beat parameters (e.g., Japanese Minimal reduces HOOK height)
4. Arc template from archetype provides a default beat sequence
5. Planner can deviate from default sequence but must follow arc rules (no invalid sequences)

**Arc Rules (enforced):**
- Every page MUST start with HOOK or TEASE (never BUILD or PROOF first)
- Every page MUST end with CLOSE or PIVOT (never BREATHE or BUILD last)
- No 3+ consecutive high-energy beats (HOOK, PEAK, TENSION) -- must breathe between peaks
- No 3+ consecutive low-energy beats (BREATHE, PROOF) -- needs re-engagement
- PEAK beat can appear at most twice per page
- BREATHE must follow PEAK (mandatory cooldown)

**Beat Energy Levels** (for sequence validation):
| Beat | Energy Level |
|------|-------------|
| HOOK | High |
| TEASE | Medium |
| REVEAL | Medium-High |
| BUILD | Medium |
| PEAK | High |
| BREATHE | Low |
| TENSION | High |
| PROOF | Low |
| PIVOT | Medium |
| CLOSE | Medium-Low |

**The 10 beats with their constraint values:**

1. **HOOK** -- First 2 seconds. Grab attention.
   - Height: 90-100vh, Elements: 3-5, Whitespace: 60-70%, Type: hero, Animation: high (dramatic entrance 800-1500ms)
   - Overrides: Japanese Minimal (50-70vh, 1-2 elements), Data-Dense (skip HOOK -- data IS the hook), Editorial (60-80vh, content-first)

2. **TEASE** -- Hint at depth. Create desire to scroll.
   - Height: 40-60vh, Elements: 3-6, Whitespace: 50-60%, Type: h2-h3, Animation: medium (subtle reveals)
   - Overrides: Kinetic (animation intensity high)

3. **REVEAL** -- The "aha moment." Show the product/concept clearly.
   - Height: 60-80vh, Elements: 4-8, Whitespace: 40-50%, Type: h2, Animation: medium-high (product showcase entrance)
   - Overrides: Luxury (full-viewport product hero), Data-Dense (dashboard reveal with live data)

4. **BUILD** -- Stack value. Layer features/benefits.
   - Height: 80-120vh (can exceed viewport), Elements: 8-12, Whitespace: 30-40%, Type: h3-h4, Animation: low-medium (staggered reveals)
   - Overrides: Editorial (max 6 elements, wider spacing), Japanese Minimal (max 4 elements per build section)

5. **PEAK** -- Maximum intensity. The wow moment.
   - Height: 80-100vh, Elements: 3-5, Whitespace: 40-60%, Type: h1-hero, Animation: high (signature animation, parallax, 3D)
   - Overrides: Swiss (restrained peak -- geometric precision instead of spectacle), Data-Dense (data visualization as peak)
   - NOTE: PEAK is where creative tension moments belong. At least one tension technique should fire here.

6. **BREATHE** -- Pause. Let the user rest.
   - Height: 30-50vh, Elements: 1-3, Whitespace: 70-80%, Type: body-large to h3, Animation: minimal (slow fade or no animation)
   - HARD enforcement on whitespace 70-80%. This is the most critical constraint.
   - Overrides: Brutalist (may use text-only quote), Ethereal (80-90% whitespace)

7. **TENSION** -- Create urgency or conflict. Challenge the user.
   - Height: 40-70vh, Elements: 3-6, Whitespace: 30-50%, Type: h2-h3, Animation: medium-high (dramatic, asymmetric)
   - Often used for: objection handling, comparison, "before/after", challenge statements
   - Overrides: Playful (gamified tension), Neo-Corporate (statistics-driven tension)

8. **PROOF** -- Social proof, testimonials, case studies.
   - Height: 60-100vh, Elements: 5-10, Whitespace: 35-45%, Type: h3-body-large, Animation: low (staggered grid appearance)
   - Overrides: Luxury (full-page case study with photography), Data-Dense (metrics grid)

9. **PIVOT** -- Shift direction. New topic or perspective.
   - Height: 40-60vh, Elements: 2-4, Whitespace: 50-65%, Type: h2, Animation: medium (transition emphasis)
   - Often used for: "But that's not all", topic shifts, secondary offering introduction
   - Overrides: Editorial (dramatic type treatment for pivot statement)

10. **CLOSE** -- Final action. CTA and farewell.
    - Height: 40-60vh, Elements: 3-5, Whitespace: 50-65%, Type: h2-h3, Animation: medium (final attention)
    - MUST contain primary CTA. No ambiguity about next step.
    - Overrides: Japanese Minimal (minimal close, one element + CTA), Luxury (full-page statement close)

**Default Arc Templates by Archetype:**
Show 4-5 archetype-specific arc templates as example sequences:
- **Standard (most archetypes):** HOOK -> TEASE -> BUILD -> PEAK -> BREATHE -> PROOF -> BUILD -> CLOSE
- **Editorial:** HOOK -> REVEAL -> BUILD -> BREATHE -> BUILD -> PEAK -> PROOF -> CLOSE
- **Japanese Minimal:** TEASE -> REVEAL -> BREATHE -> BUILD -> BREATHE -> CLOSE
- **Data-Dense:** BUILD -> REVEAL -> BUILD -> PROOF -> BUILD -> TENSION -> CLOSE
- **Luxury/Fashion:** HOOK -> BREATHE -> REVEAL -> PEAK -> BREATHE -> PROOF -> CLOSE

**6 Transition Techniques Between Beats:**
Define 6 techniques for transitioning between sections with different energy levels:
1. **Hard Cut** -- Immediate energy change, no transition. Best for: BREATHE->PEAK, PROOF->TENSION
2. **Gradient Fade** -- Smooth energy blending via background color shift. Best for: BUILD->BREATHE, REVEAL->BUILD
3. **Spatial Bridge** -- Overlapping elements that connect sections. Best for: TEASE->REVEAL, BUILD->PEAK
4. **Narrative Thread** -- Continuous visual element (line, shape, color) threading through sections. Best for: multi-BUILD sequences
5. **Rhythm Break** -- Intentional disruption of visual rhythm to reset attention. Best for: before PEAK or TENSION
6. **Echo** -- Subtle visual callback to an earlier section. Best for: CLOSE referencing HOOK

**Layer 3: Integration Context** (~40-50 lines):
- **Arc -> DNA**: Default arc template stored in DESIGN-DNA.md. Section planners start from this template.
- **Arc -> Archetypes**: Archetype overrides modify beat defaults. Some archetypes skip certain beats entirely (Data-Dense skips HOOK).
- **Arc -> Section Planner**: Beat assignments are made during `/modulo:plan-dev`. Each section in MASTER-PLAN.md has a beat type.
- **Arc -> Section Builder**: Builder spawn prompt includes beat type + constraints. Builder reads constraint table and builds within parameters.
- **Arc -> Anti-Slop Gate**: Layout diversity check (L1) validates that consecutive sections have different layouts -- this is enabled BY beat variation.
- **Arc -> Creative Tension**: PEAK beats are where tension techniques belong. Tension override should align with PEAK or TENSION beats.

**Layer 4: Anti-Patterns** (~30-40 lines):
- **Anti-Pattern: Feature-List Page** -- Every section is BUILD. Why bad: monotonous pacing, no emotional peaks or valleys. Instead: vary energy with BREATHE, PEAK, TENSION beats.
- **Anti-Pattern: All High Energy** -- HOOK -> PEAK -> TENSION -> PEAK -> CLOSE. Why bad: exhausting, no contrast. Peaks only feel intense BECAUSE of the valleys around them. Instead: follow the 3-consecutive-same-energy rule.
- **Anti-Pattern: Advisory Constraints** -- "BREATHE sections should feel open and airy." Why bad: not enforceable. Instead: "BREATHE whitespace ratio: 70-80%, HARD enforcement."
- **Anti-Pattern: Ignoring Archetype Overrides** -- Using standard HOOK (90-100vh) for Japanese Minimal. Why bad: violates archetype personality. Instead: always check override table before applying defaults.
- **Anti-Pattern: Generic Transitions** -- Every section just fades into the next. Why bad: transitions are pacing tools. Instead: choose transition technique based on energy delta between beats.

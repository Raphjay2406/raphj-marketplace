**ASCII Mockup Format Specification** (~60-80 lines):

Define the standardized symbol vocabulary:
```
+------+     Section boundary (always full width)
|      |     Content area
[TEXT]        Button/CTA (interactive element)
(TEXT)        Decorative/background element
██████        Visual weight/emphasis block
▓▓▓▓▓▓       Image/media placeholder
░░░░░░       Subtle background texture/pattern
───────       Horizontal rule/divider
│             Vertical alignment guide
→             Direction/flow indicator
*text*        Annotation (sizing, effect, treatment)
```

Rules for ASCII mockups:
- Width: 56 characters (fits in markdown without wrapping)
- Every non-obvious element gets an annotation line below it: `*text-8xl, py-40, fade-up on scroll*`
- Show both desktop layout (primary) and mobile note (simplified description, not full ASCII)
- Include section height indication: `*viewport: ~110vh*` or `*viewport: ~60vh*`
- Include at least 1 annotation for motion/animation: `*parallax: -20% on scroll*`
- Include background specification: `*bg: var(--color-bg-primary)*`
- Show element hierarchy through indentation and sizing

Provide 3-5 canonical ASCII mockup templates:
1. **Cinematic Hero** (full-viewport, centered headline, product visual below)
2. **Editorial Hero** (asymmetric, large type left, image right)
3. **Bento Features** (asymmetric grid cards, mixed sizes)
4. **Split Proof** (testimonial left, metric right, or vice versa)
5. **Minimal CTA** (centered, generous whitespace, single action)

Each template: ~15-20 lines of ASCII art with annotations.

**Layer 3: Integration Context** (~50-60 lines):

**Distinctness Validation Matrix:**
After generating 3 directions, validate using this matrix:

| Dimension | Direction A | Direction B | Direction C | All Different? |
|-----------|-------------|-------------|-------------|----------------|
| Archetype | | | | yes/no |
| Color Mood (warm/cool/neutral/dark/vibrant) | | | | yes/no |
| Motion Style (kinetic/subtle/static/cinematic) | | | | yes/no |
| Tension Level (conservative/moderate/bold) | | | | yes/no |
| Typography Voice (serif/sans/mono/mixed) | | | | yes/no |
| Layout Philosophy (cinematic/dense/bento/minimal/immersive) | | | | yes/no |

PASS: At least 3 of 6 dimensions are "yes" (all three directions differ)
FAIL: Regenerate the most similar direction with forced shifts on the 2 weakest dimensions

**Free Mixing Protocol:**
After user reviews all 3 directions:
1. User indicates preferences: "I like A's colors, B's typography, C's motion"
2. Claude identifies the base direction (closest to majority of picks)
3. Claude maps cherry-picked elements onto the base direction's archetype
4. Run coherence check: do the mixed elements conflict? (e.g., Brutalist motion + Ethereal typography = conflict)
5. If coherent: present synthesized direction with updated concept board
6. If conflict: explain the tension and propose a resolution (which element to adjust)
7. Final synthesized direction becomes the Design DNA seed

**How design-brainstorm references this skill:**
The brainstorm protocol produces 3 directions using this format. This skill defines the OUTPUT FORMAT only. The brainstorm protocol defines the PROCESS of arriving at each direction. Separation of concerns: process (brainstorm) vs. presentation (this skill).

**BRAINSTORM.md output format:**
The final artifact saved to `.planning/genorah/BRAINSTORM.md` includes:
- All 3 directions in concept board format
- Distinctness validation matrix (filled)
- User's selection / mixing notes
- Final synthesized direction (if mixing occurred)
- Competitive benchmark summary

**Layer 4: Anti-Patterns** (~40-50 lines):
5 anti-patterns:

1. **Three Shades of Beige**: Directions that differ only on color palette but share the same archetype, layout, and motion. Instead: force different archetypes and validate with the distinctness matrix.

2. **ASCII Art Theater**: Mockups that look pretty but don't communicate layout intent -- too abstract, no annotations, no sizing. Instead: every element annotated with Tailwind sizing, every section with viewport height, every motion with a description.

3. **Motion Afterthought**: Direction includes detailed visual spec but motion identity is a single word ("smooth"). Instead: motion identity must include signature motion, entrance pattern, hover behavior, scroll behavior, and transition pace.

4. **Missing the "Why"**: Directions presented as visual specs without strategic rationale. Instead: every direction ends with "Why This Direction" connecting visual choices to project goals and competitive differentiation.

5. **False Choice**: Two safe directions and one "wild card" that exists to be rejected, making the choice feel predetermined. Instead: all 3 directions must be genuinely viable. The user should struggle to choose because all three are excellent but different.

**Enforcement protocol:**
1. During section planning: section-planner checks PLAN.md copy against both lists
2. During build: builders reference micro-copy skill's banned list (which this extends)
3. During review: quality-reviewer runs final banned-phrase scan
4. Hard-banned: automatic fail, must be replaced
5. Discouraged: warning in review, builder can override with written justification in SUMMARY.md

**Voice Extraction Protocol** (~40-50 lines):
Layer 3 content. This solves Pitfall 6 (voice document being write-once-never-read).

Define how brand voice rules get extracted INTO each section's PLAN.md:
1. During section-planning, the section-planner reads BRAND-VOICE.md
2. For each section, extract:
   - Relevant row from Per-Section Voice Variation table
   - Relevant headline formula from Headline Formula Library
   - Content density rules for this section type
   - 2-3 relevant examples from the content bank matrix (matching beat x section)
   - Any project-specific vocabulary constraints
3. Insert extracted rules into the section PLAN.md under a "## Copy Specification" heading
4. Builders read the Copy Specification in their PLAN.md — they NEVER read BRAND-VOICE.md directly

Format for Copy Specification in PLAN.md:
```markdown
## Copy Specification
**Voice:** [tone position from spectrum] — [key characteristics]
**Headline formula:** [from content bank: formula + 2 examples]
**CTA pattern:** [primary + secondary approach for this section]
**Density:** [max elements, whitespace target]
**Banned in this section:** [any section-specific banned phrases beyond global list]
**Examples from content bank:** [2-3 relevant formulas/examples]
```

This keeps spawn prompts lean (copy spec adds ~15-20 lines per section, well within the 300-line budget).

**Layer 4: Anti-Patterns** (~50-60 lines):
5 anti-patterns:

1. **The Generic Fallback**: When pressed for time, falling back to "Learn More" and "Get Started" instead of using the content bank formulas. Instead: every content bank formula produces a result faster than thinking up "Learn More."

2. **Voice Amnesia**: Brand voice document is generated but forgotten during build — sections revert to Claude's default copywriting tone. Instead: voice extraction protocol ensures every PLAN.md has a Copy Specification section.

3. **One Voice Fits All**: Same tone for hero, features, testimonials, and pricing. Instead: per-section voice variation table explicitly shifts tone, length, formality, and energy per section type.

4. **Archetype-Blind Templates**: Content bank formulas that produce the same output regardless of archetype. Instead: every formula includes archetype modifiers — a Brutalist "HOOK-hero" is fundamentally different from an Ethereal one.

5. **Banned Phrase Whack-a-Mole**: Removing banned phrases one at a time during review instead of preventing them. Instead: content bank formulas are designed to NEVER produce banned phrases — prevention, not detection.

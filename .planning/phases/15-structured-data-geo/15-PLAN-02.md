**Layer 4: Anti-Patterns (6-8 anti-patterns):**

Source from 15-RESEARCH.md "Anti-Patterns to Avoid" and "Common Pitfalls" sections. Rewrite in Layer 4 format (Name / What goes wrong / Instead).

1. **Schema-Content Mismatch** -- What goes wrong: JSON-LD claims data (prices, dates, FAQ answers) not visible on the page. Google issues manual actions, losing rich result eligibility. This is the #1 structured data violation. Especially dangerous after `/gen:iterate` edits when content changes but JSON-LD is not updated. Instead: Run schema audit on every quality-reviewer pass. Every JSON-LD claim must have a visible counterpart. Use the audit protocol from Layer 2.

2. **Fabricated FAQ Schema** -- What goes wrong: FAQPage schema added with questions/answers that do not appear anywhere on the page. Google treats invisible structured data as spam. Instead: Same data array drives both visible FAQ section AND JSON-LD. If there is no visible FAQ section, do not add FAQPage schema. The pattern in Layer 2 enforces this by using a shared `faqs` array.

3. **HowTo Schema Expecting Rich Results** -- What goes wrong: Builder implements HowTo schema for tutorial content and expects step-by-step rich results in Google search. Nothing appears -- rich results are fully deprecated (Aug-Sep 2023). Instead: Use Article schema as the primary schema for tutorial pages. Include HowTo schema alongside Article in the @graph ONLY for AI engine extraction value. Set correct expectations: no Google visual rich results from HowTo.

4. **Over-Optimizing Emotional Beats** -- What goes wrong: FAQ sections, keyword-stuffed headings, and statistic callouts added to BREATHE, PEAK, and other purely emotional beats. The page looks like an SEO farm, Anti-Slop Gate "Creative Courage" score drops, and the emotional arc is destroyed. Instead: Only HOOK, TENSION, PROOF, and CLOSE beats carry required SEO elements. The remaining 6 beats are purely emotional by design. This restraint is what separates award-winning SEO from generic optimization.

5. **Blanket AI Bot Content Blocking** -- What goes wrong: All AI bots blocked indiscriminately in robots.txt. Site becomes invisible in ChatGPT search, Perplexity, and AI answer engines. Even well-structured JSON-LD and GEO patterns are wasted if bots cannot reach the content. Instead: Block training bots (GPTBot, ClaudeBot) to protect content from unauthorized model training. Allow search bots (OAI-SearchBot, PerplexityBot, Claude-SearchBot) for AI search visibility. See `seo-meta` skill's appendix-ai-bots.md for the complete taxonomy.

6. **Duplicate @context in @graph** -- What goes wrong: Each schema object inside `@graph` includes its own `"@context": "https://schema.org"`. Validators warn, some parsers behave unexpectedly. Instead: Declare `@context` exactly once at the root level of the @graph wrapper. Individual entries within `@graph` must NOT include `@context`.

7. **GEO Patterns on Every Page** -- What goes wrong: BLUF formatting, question headings, statistical callouts, and FAQ sections applied to every page regardless of content type. Portfolio pages, landing pages, and minimal-content pages look over-optimized and generic. Instead: Apply GEO patterns based on page type AND archetype intensity tier. Content-heavy pages (blog, article, documentation) get Full/Moderate GEO. Landing pages and portfolios get Minimal/none. The archetype intensity table in Layer 3 prescribes the correct level.

8. **Ignoring Schema Audit After Iteration** -- What goes wrong: JSON-LD is correct after initial `/gen:execute` but becomes stale after `/gen:iterate` changes page content. Prices change, dates shift, FAQ answers are rewritten, but schema still claims old values. Instead: Schema audit runs on EVERY quality-reviewer pass -- after execute AND iterate. The auto-fix protocol updates JSON-LD to match current visible content and notifies the user of changes.

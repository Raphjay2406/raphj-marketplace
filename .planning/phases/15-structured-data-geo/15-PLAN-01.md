**Layer 1: Decision Guidance must include:**

1. **When to Use** -- Every public-facing page that benefits from search visibility. Structured data is always beneficial (even when Google rich results are not guaranteed, AI engines extract it). Specific triggers:
   - Any page with identifiable content type (article, product, event, FAQ, etc.)
   - Any page targeting AI search engine visibility (GEO)
   - Any page where the Emotional Arc assigns TENSION, PROOF, or CLOSE beats (schema integration points)

2. **When NOT to Use** -- Private dashboards behind auth, Tauri/Electron desktop apps. For meta tags, canonical URLs, sitemaps, and robots.txt, use `seo-meta` skill instead. For IndexNow and proactive indexing, use `search-visibility` skill (Phase 16). Structured data and meta tags are complementary -- most pages need BOTH skills.

3. **Schema Status Matrix** -- A decision-critical table showing each schema type's current Google rich result status (Active, Restricted, Deprecated). This prevents agents from making false promises about rich results. Use the verified data from 15-RESEARCH.md:

   | Schema Type | Google Rich Result | Status | AI Engine Value |
   |-------------|-------------------|--------|-----------------|
   | Article/BlogPosting/NewsArticle | Article rich result, Google News | Active | High -- headline, author, dates extracted |
   | FAQPage | FAQ rich result | Restricted (gov/health only since Aug 2023) | Critical -- high correlation with AI Overview appearance |
   | HowTo | NONE | Deprecated (Aug-Sep 2023) | Medium -- AI engines extract step-by-step content |
   | Organization | Knowledge panel | Active | High -- entity recognition |
   | WebSite | NONE (sitelinks search box removed Nov 2024) | Schema valid, no rich result | Medium -- site identity, AI entity recognition |
   | BreadcrumbList | Breadcrumb trail | Active | Medium -- page hierarchy |
   | Product | Product snippet, Popular Products | Active | High -- price, availability, reviews |
   | LocalBusiness | Local business panel | Active | High -- location, hours, contact |
   | Event | Event experience | Active | High -- date, location, tickets |

4. **Per-Page-Type Recipe Table** -- Prescriptive mapping of which schemas to combine per page type. Use the table from 15-RESEARCH.md Pattern 3, preserving the honest notes about deprecated/restricted features. This is the primary reference for section-planners.

5. **Decision Tree** -- Key branching decisions:
   - Single page vs multi-schema: Use @graph when 2+ schemas apply (most pages)
   - FAQ content exists on page: Add FAQPage to @graph (for GEO, not for rich results on most sites)
   - Tutorial/guide content: Use Article schema primarily, add HowTo only for AI extraction (no Google rich results)
   - E-commerce: Product schema requires `name` + `offers` for rich results
   - Physical location: Add LocalBusiness schema alongside Organization
   - Blog/news content: Choose between BlogPosting and NewsArticle (NewsArticle = time-sensitive)
   - Content-heavy page: Apply GEO patterns (BLUF, question headings) -- see Layer 2 GEO section
   - Content-light page (landing page, portfolio): Minimal GEO, focus on schema accuracy

6. **GEO Decision Guidance** -- Brief overview of when GEO patterns apply:
   - GEO = Generative Engine Optimization = structuring content for AI search engines
   - Not all pages benefit equally: content-heavy pages (blog, article, documentation) benefit most
   - Landing pages, portfolios, and product pages benefit less (Emotional Arc drives those)
   - Archetype-aware intensity: GEO patterns adapt per archetype personality (Full/Moderate/Subtle/Minimal -- detailed mapping in Layer 3)
   - Key principle: Same SEO signals, different visual expression

7. **Pipeline Connection:**

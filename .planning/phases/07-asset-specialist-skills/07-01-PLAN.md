**Layer 1: Decision Guidance** (~80-100 lines):

1. **Shape Purpose Decision Tree** -- What are you adding?
   - Section transition? -> Section Dividers (wave, angle, organic, stepped)
   - Background atmosphere? -> Backgrounds & Atmospherics (dot grids, noise, gradient meshes, particle fields)
   - Visual accent/decoration? -> Accents & Decorative (floating shapes, rings, patterns, line art)
   - Hero visual/illustration? -> Hero Illustrations (isometric, abstract compositions, branded shapes)
   - "If shapes are the main content (hero illustration), invest in complexity. If shapes are atmosphere, keep them subtle."

2. **Technique Selection Matrix** -- For each shape category, which technique to use:
   | Need | CSS Only | SVG | Canvas/WebGL |
   | Simple geometric | clip-path, gradients | -- | -- |
   | Organic/flowing | -- | SVG path + simplex-noise | -- |
   | Animated drawing | -- | GSAP DrawSVG (free) | -- |
   | Shape morphing | -- | GSAP MorphSVG (free) | -- |
   | Noise textures | -- | SVG feTurbulence (zero JS) | -- |
   | Particles (many) | -- | -- | Canvas or R3F Points |
   | Procedural organic | -- | simplex-noise -> SVG path | -- |
   | Grid/dot patterns | radial-gradient/linear-gradient | SVG pattern | -- |

3. **Per-Archetype Shape Palette Table** -- All 19 archetypes, compact table:
   | Archetype | Primary Shapes | Secondary | Forbidden | Shape Intensity |
   Key examples:
   - Brutalist: Hard angles, raw geometry, broken grids | Dot matrix, ASCII | Organic curves, blobs | HIGH
   - Ethereal: Organic blobs, gradient meshes, soft waves | Particle fields, aurora | Sharp angles, hard edges | MEDIUM
   - Japanese Minimal: Single circles, thin lines, negative space | Dot grids (sparse) | Complex patterns, many shapes | LOW
   - Kinetic: Dynamic diagonals, motion lines, speed shapes | Particle trails, streaks | Static, heavy patterns | HIGH
   - Neon Noir: Glowing outlines, wireframes, grid lines | Neon rings, scan lines | Organic/natural, warm textures | MEDIUM-HIGH
   Continue for all 19 archetypes (Neubrutalism, Dark Academia, AI-Native included). Builders can mix in related shapes if they fit DNA, but FORBIDDEN shapes are hard constraints.

4. **Beat-Aware Shape Intensity:**
   | Beat | Shape Intensity | Guidance |
   - HOOK: Bold, large, complex | Signature shapes, hero illustrations, animated SVG
   - PEAK: Complex, animated | Morphing shapes, scroll-linked, particle systems
   - BUILD: Moderate, supporting | Background patterns, subtle dividers
   - BREATHE: Minimal, subtle | Negative space, single thin line or nothing
   - TENSION: Intentionally dissonant | Shapes that break the established pattern
   - PROOF: Supporting only | Subtle backgrounds, clean dividers
   - CLOSE: Gentle, echoing | Callback to HOOK shapes at lower intensity

5. **DNA Color Enforcement:**
   - ALL shapes use DNA color tokens: `hsl(var(--color-primary))`, `hsl(var(--color-accent))`, `bg-primary`, etc.
   - NEVER use hardcoded hex values (#8b5cf6) or Tailwind defaults (bg-blue-500)
   - For opacity variations: `hsl(var(--color-primary) / 0.1)` or Tailwind `text-primary/10`
   - For computed colors (gradients between DNA tokens): interpolate between DNA tokens, don't introduce new colors

**Hybrid Generation Approach** (~30 lines):

- Seeded random function for reproducible shapes
- Color token helper: `dnaColor(token, opacity?)` pattern

**Layer 3: Integration Context** (~40-50 lines):
- **-> Design DNA**: All shape colors from DNA tokens. Signature element informs branded shapes.
- **-> Emotional Arc**: Beat type determines shape intensity. See beat-aware mapping.
- **-> Creative Tension**: TENSION beat shapes intentionally break the established shape language.
- **-> Cinematic Motion**: Animated shapes use the archetype's motion presets (easing, duration). Scroll-linked shapes follow beat-dependent scroll behavior rules.
- **-> 3D/WebGL Effects**: For true 3D with lighting/materials/shaders, use the 3D skill. This skill covers 2D shapes and CSS pseudo-3D only.
- **-> Design System Scaffold**: Scaffold may generate base shape utilities from DNA. Builders extend, not replace.

**Layer 4: Anti-Patterns** (~50-60 lines):
- **Anti-Pattern: Hardcoded Hex Colors** -- Using `fill="#8b5cf6"` or `bg-violet-500` in shapes. Why bad: breaks DNA token system, creates visual inconsistency. Instead: always use `hsl(var(--color-primary))` or Tailwind DNA classes.
- **Anti-Pattern: Math.random() for Procedural Shapes** -- Using `Math.random()` produces different shapes on every render and cannot be reproduced. Why bad: inconsistent across sessions, untestable. Instead: use `simplex-noise` with a seed value for reproducible procedural generation.
- **Anti-Pattern: Shape Overload** -- Adding decorative shapes to every section. Why bad: visual noise, dilutes impact. Instead: follow beat-aware intensity mapping. BREATHE beats get minimal or no shapes.
- **Anti-Pattern: CSS-Only Complex Organic Shapes** -- Trying to create organic blobs with pure CSS border-radius hacks. Why bad: limited control, poor animation. Instead: use SVG paths generated with simplex-noise.
- **Anti-Pattern: Wrong Shapes for Archetype** -- Organic blobs in a Brutalist design, or hard geometry in an Ethereal design. Why bad: contradicts archetype personality. Instead: consult the per-archetype shape palette table.
- **Anti-Pattern: Hand-Rolling SVG Morph** -- Manual path point interpolation for shape morphing. Why bad: fails with different point counts, poor easing. Instead: GSAP MorphSVG handles this automatically (now free).
- **Anti-Pattern: Heavy Particle Systems Without Need** -- Canvas/WebGL particle system for a simple dot background. Why bad: unnecessary bundle and GPU cost. Instead: CSS radial-gradient dot grid for simple patterns. Reserve Canvas for 100+ animated particles.

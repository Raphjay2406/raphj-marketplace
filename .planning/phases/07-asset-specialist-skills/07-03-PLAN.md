**Layer 1: Decision Guidance** (~80-100 lines):

1. **When to Use Marketplace Components:**
   - Marketplace components are STARTING POINTS, not final output
   - Best for: animated interactions that would take significant time to build from scratch (animated grids, parallax cards, text effects)
   - NOT for: simple layouts, basic cards, standard sections. Build these from DNA to ensure uniqueness.
   - Hard cap: no more than ~30% of a page's visual elements should originate from marketplace components
   - "If a marketplace component looks impressive but doesn't fit the archetype, don't use it. Build something archetype-appropriate instead."

2. **Marketplace Profiles:**
   | Marketplace | Technology | Strengths | Best For | Installation |
   | Aceternity UI | Framer Motion + Tailwind | Premium animated components, dark UI focus, highly animated | High-intensity archetypes, PEAK/HOOK sections | Copy-paste |
   | Magic UI | shadcn + Tailwind | 50+ landing page components, startup-focused, clean | Landing pages, startup archetypes, BUILD/PROOF sections | Copy-paste, shadcn-based |
   | 21st.dev | shadcn + Tailwind + Community | Open registry, community-contributed, AI-assisted remix | Diverse needs, broad component selection | `npx shadcn add` from registry |
   | Framer Marketplace | Framer-specific | Templates, motion components, plugins | Design REFERENCE and INSPIRATION only | NOT directly portable |

   **Important on Framer:** Framer marketplace components are designed for the Framer platform. They CANNOT be used directly in Next.js/Astro projects. Use them as design reference -- study their motion patterns, layout ideas, and interaction concepts, then implement custom equivalents using R3F, Motion library, or CSS.

3. **Which Marketplace for Which Category:**
   | Category | Aceternity UI | Magic UI | 21st.dev | Framer |
   | Animated grids/cards | Strong | Strong | Community varies | Reference only |
   | Hero sections | Strong | Strong | Growing | Reference only |
   | Scroll effects | Moderate | Limited | Community varies | Reference only |
   | Interactive elements | Strong | Moderate | Community varies | Reference only |
   | Text animations | Moderate | Strong | Community varies | Reference only |
   | Testimonials/social proof | Moderate | Strong | Growing | Reference only |

4. **Import Path Warning:**
   Many Aceternity UI components use `import from 'framer-motion'`. This is DEPRECATED. When sourcing Aceternity components, update ALL imports:
   - `import { motion } from 'framer-motion'` -> `import { motion } from 'motion/react'`
   - `import { AnimatePresence } from 'framer-motion'` -> `import { AnimatePresence } from 'motion/react'`
   - `import { LazyMotion, m, domAnimation } from 'framer-motion'` -> `import { LazyMotion, m, domAnimation } from 'motion/react'`

**Layer 2: Award-Winning Examples** (~200-250 lines):

1. **Category Matrix (Archetype x Beat)** (~80 lines):

   | Beat | Kinetic | Luxury | Japanese Minimal | Brutalist | Ethereal | Neo-Corporate | ... |
   | HOOK | Animated hero grid, particle bg | Slow reveal gallery, slider | Single object focus, zen | Raw text reveal, glitch | Soft particles, floating | Clean hero, subtle motion |
   | PEAK | Interactive 3D viewer, drag demo | Cinematic scroll sequence | Subtle parallax, floating | Scale violence grid | Morphing shapes, aurora | Polished interactive demo |
   | BUILD | Animated card grid, accordion | Elegant feature list | Clean list, micro-animation | Dense info grid | Gentle cards, soft shadows | Feature comparison, tabs |
   | PROOF | Animated counter, testimonial | Logo wall, subtle fade | Minimal counter, clean quote | Raw number display | Floating testimonials | Professional logo wall |
   | CLOSE | Animated CTA, magnetic button | Elegant CTA, subtle | Minimal CTA, space | Bold CTA, contrast | Soft CTA, fade-in | Professional CTA |
   | BREATHE | Ambient gradient only | Negative space, breathing | Pure whitespace | Raw space, contrast | Soft gradient, floating | Clean whitespace |

   Cover all 19 archetypes across 10 beat types. Keep cells concise (2-3 items max).

2. **Full Restyling Protocol** (~80 lines):
   When you source a marketplace component, follow ALL four steps:

   **Step 1: Token Mapping (CSS Variable Swaps)**
   Replace all hardcoded colors with DNA tokens:
   ```tsx
   // BEFORE (marketplace default)
   className="bg-black text-white border-gray-800"
   // AFTER (DNA-styled)
   className="bg-surface text-on-surface border-muted"

   // BEFORE (inline styles)
   style={{ color: '#8b5cf6', background: 'linear-gradient(...)' }}
   // AFTER
   style={{ color: 'hsl(var(--color-primary))', background: `linear-gradient(hsl(var(--color-primary)), hsl(var(--color-accent)))` }}
   ```

   **Step 2: Structural Modifications**
   Adjust layout to match archetype personality:
   - Grid columns, spacing, border-radius
   - Swap fixed sizing for responsive (clamp(), container queries)
   - Add/remove visual elements to match archetype
   - Example: Aceternity bento grid -> widen gaps for Ethereal, tighten for Data-Dense

   **Step 3: Animation Restyling**
   Match DNA motion language:
   ```tsx
   // BEFORE (generic animation)
   transition={{ duration: 0.5, ease: 'easeOut' }}
   // AFTER (DNA motion profile)
   transition={{
     duration: 'var(--motion-duration-medium)',
     ease: [/* archetype easing curve */],
   }}
   ```
   - Replace hardcoded durations with archetype motion profile values
   - Replace generic easings with archetype-specific curves
   - Add/remove animation intensity to match archetype

   **Step 4: Archetype-Specific Customization**
   Final pass for archetype personality:
   - Brutalist: Remove smooth transitions, add raw edges, increase contrast
   - Luxury: Slow everything down, add subtle details, increase whitespace
   - Japanese Minimal: Remove excess, increase negative space, simplify
   - AI-Native: Add data visualization elements, subtle glitch effects

3. **21st.dev Installation Pattern** (~20 lines):
   ```bash
   # Install from 21st.dev registry
   npx shadcn add https://21st.dev/r/{component-slug}

   # This auto-resolves dependencies and adds to your project
   # Components land in your configured components directory
   ```
   - Components are shadcn-based and fully customizable
   - After installation, apply the full restyling protocol

4. **Marketplace Usage Audit Checklist** (~20 lines):
   Before committing marketplace components, verify:
   - [ ] All hardcoded colors replaced with DNA tokens
   - [ ] All `framer-motion` imports updated to `motion/react`
   - [ ] Animation timing matches archetype motion profile
   - [ ] Layout/spacing adjusted for archetype personality
   - [ ] Component is under the 30% marketplace cap for this page
   - [ ] Component genuinely adds value (not just "looks cool")
   - [ ] Reduced-motion fallback implemented or verified
   - [ ] Component doesn't conflict with an archetype's forbidden patterns

**Layer 3: Integration Context** (~30-40 lines):
- **-> Design DNA**: Every marketplace component must be restyled to use DNA tokens. No component ships with default styling.
- **-> Design Archetypes**: The archetype x beat matrix determines which component categories are appropriate. Forbidden patterns from the archetype must be checked against marketplace component styling.
- **-> Cinematic Motion**: Marketplace component animations are restyled to match the archetype's motion profile (easing, duration, stagger).
- **-> Anti-Slop Gate**: The 35-point anti-slop gate will catch unrerestyled marketplace components as "template feel." Proper restyling prevents this.
- **-> Creative Tension**: Marketplace components are generally NOT tension moments. Tension should be custom-built for maximum impact.

**Layer 4: Anti-Patterns** (~40-50 lines):
- **Anti-Pattern: Unstyled Marketplace Drop-In** -- Using an Aceternity component with its default dark styling in a Warm Artisan project. Why bad: clashes with archetype, creates "template" feel, will fail anti-slop gate. Instead: apply full 4-step restyling protocol.
- **Anti-Pattern: Marketplace-Heavy Pages** -- 5 of 7 sections sourced from marketplaces. Why bad: exceeds 30% cap, creates "assembled from parts" feel, no unique identity. Instead: limit to 1-2 marketplace components per page, build the rest from DNA.
- **Anti-Pattern: Specific Component Lock-In** -- Documentation referencing "Aceternity Bento Grid v2.3." Why bad: component names/versions change. Instead: recommend component CATEGORIES (animated grid, card carousel) and link to marketplace for browsing.
- **Anti-Pattern: Deprecated Framer Motion Imports** -- Keeping `import from 'framer-motion'` in sourced components. Why bad: deprecated package. Instead: update to `import from 'motion/react'` immediately on sourcing.
- **Anti-Pattern: Framer Marketplace Direct Use** -- Attempting to use Framer marketplace components directly in Next.js/Astro. Why bad: Framer components are platform-specific, won't work outside Framer. Instead: use as design reference, implement custom equivalents.
- **Anti-Pattern: Skipping the Audit** -- Not running the marketplace usage checklist before committing. Why bad: unstyled components leak through. Instead: every marketplace component goes through the full checklist.

**Layer 1: Decision Guidance** (~80-100 lines):

1. **When to Use AI-Generated Images:**
   - Hero backgrounds where stock photos don't match DNA personality
   - Abstract textures and patterns that need to match DNA color palette
   - Conceptual illustrations that don't exist as stock
   - Placeholder images during prototyping that match final visual direction
   - NOT for: realistic team photos (use real photos), product photos (use actual product), screenshots, logos

2. **When AI Images HURT the Design** -- Per-archetype guidance:
   | Archetype | AI Image Stance | Why | Prefer Instead |
   | Swiss/International | Avoid | Favors real photography, precision, authenticity | Curated stock photography, real photography |
   | Warm Artisan | Avoid | Hand-crafted aesthetic clashes with AI-generated feel | Real textures, hand-drawn, photography |
   | Editorial | Cautious | Editorial demands genuine photography for credibility | Commissioned or curated editorial photography |
   | Japanese Minimal | Cautious | Prefers negative space over imagery; when used, must be photographic | Minimal real photography |
   | Brutalist | Good fit | Raw, abstract, confrontational imagery works | AI excels at surreal/abstract for this archetype |
   | Ethereal | Great fit | Dreamlike, otherworldly imagery is AI's strength | AI-generated dreamy landscapes, abstracts |
   | Neon Noir | Great fit | Cyberpunk, neon-lit scenes match AI capabilities | AI-generated dark cityscapes, neon elements |
   | AI-Native | Great fit | Meta-appropriate; AI imagery is on-brand | Lean into AI aesthetic intentionally |
   | Vaporwave | Great fit | Surreal, retro-digital aesthetic | AI-generated glitchy, retro imagery |
   Continue for all 19 archetypes. Mark each: Great fit / Good fit / Cautious / Avoid.

3. **Tool-Agnostic Approach:**
   - Describe desired output in natural language, not tool-specific syntax
   - User adapts prompt to their preferred tool (Midjourney, DALL-E, Flux, Stable Diffusion)
   - Why: AI image tool landscape changes rapidly. Tool-specific syntax (--v 6, --ar 16:9) becomes outdated.
   - Optional: A "Tool-Specific Appendix" section (clearly marked VOLATILE) maps generic prompt to tool parameters

4. **Prompt Structure Template:**
   ```
   [Subject/Content]: What's in the image
   [Style]: Visual treatment (photographic, illustration, abstract, 3D render)
   [Mood/Atmosphere]: Emotional quality (serene, energetic, mysterious, warm)
   [Color Direction]: DNA palette guidance
   [Composition]: Layout, framing, depth
   [Negative]: What to avoid (derived from DNA forbidden patterns)
   ```

4. **DNA-Derived Negative Prompts** (~40 lines):
   System for automatically generating "what to avoid" from DNA:
   | DNA Constraint | Negative Prompt Generated |
   | Archetype forbids gradients | "no gradients, no color transitions, no rainbow effects" |
   | Archetype forbids organic curves | "no organic shapes, no flowing curves, no rounded elements" |
   | Archetype forbids neon/glow | "no neon colors, no glow effects, no luminous elements" |
   | DNA has HIGH whitespace | "no cluttered composition, no dense details" |
   | DNA has COLD palette | "no warm tones, no earth tones, no orange/brown" |
   | DNA forbids rounded corners | "no soft edges, no rounded corners, no bubble shapes" |
   - Universal negatives (always include): "no watermarks, no text, no logos, no borders, no frames"
   - Quality negatives (always include): "no artifacts, no blurry, no low resolution, no distorted"

5. **Prompt Consistency Guidance** (~30 lines):
   For multi-image projects (multiple hero sections, product series):
   - Use the SAME style/mood/color modifiers across all prompts
   - Reference a "seed image" or describe consistent lighting/angle
   - Maintain consistent aspect ratios within categories
   - Tool-specific consistency notes (marked VOLATILE): seed values, style references, reference images
   - DNA signature element should appear as a subtle motif across images

6. **Tool-Specific Appendix (VOLATILE)** (~30 lines):
   Clearly marked as likely to change. Maps generic prompt to tool syntax:
   | Generic Prompt Element | Midjourney | DALL-E | Flux/SD |
   | Aspect ratio 16:9 | `--ar 16:9` | System prompt or generation settings | Width/height params |
   | High quality | `--q 2` | "high detail" in prompt | Steps/CFG params |
   | Style consistency | `--sref [url]` | System prompt | LoRA/style reference |
   | Avoid elements | `--no [items]` | Include in prompt text | Negative prompt field |
   - **WARNING:** These parameters change frequently. Verify against current tool documentation before using.

**Layer 3: Integration Context** (~30-40 lines):
- **-> Design DNA**: The ENTIRE prompt system is driven by DNA. Color palette, archetype personality, signature element, forbidden patterns all translate directly to prompt attributes and negatives.
- **-> Design Archetypes**: Archetype personality is the primary style driver. The per-archetype modifier table IS the creative direction for prompts.
- **-> Remotion**: AI-generated images can be composed into Remotion video. Generate hero backgrounds for animated video compositions.
- **-> Shape & Asset Generation**: For abstract patterns and textures, consider whether procedural generation (simplex-noise, SVG patterns) would be more DNA-consistent than AI images. Procedural shapes use exact DNA tokens; AI images approximate them.
- **-> Anti-Slop Gate**: AI-generated images that don't match DNA palette will be flagged as visual inconsistency during quality review.

**Layer 4: Anti-Patterns** (~40-50 lines):
- **Anti-Pattern: Tool-Specific Syntax in Main Prompts** -- Using `--v 6 --ar 16:9 --s 750` in the prompt template. Why bad: parameters change with tool versions, creates lock-in. Instead: describe desired output in natural language, use Tool-Specific Appendix only when generating.
- **Anti-Pattern: AI Team Photos** -- Using AI to generate realistic team member photos. Why bad: deceptive, uncanny valley, ethical concern. Instead: always use real photography for actual team members. AI portraits only for stylized/illustrated avatars.
- **Anti-Pattern: Ignoring Archetype Image Stance** -- Generating AI images for a Swiss/International design. Why bad: Swiss archetype favors real photography and precision; AI-generated images undermine authenticity. Instead: check the archetype image stance table before generating.
- **Anti-Pattern: No Negative Prompts** -- Omitting forbidden patterns from negative prompts. Why bad: AI may generate elements that violate DNA constraints (gradients in a no-gradient archetype). Instead: always derive negative prompts from DNA forbidden patterns.
- **Anti-Pattern: Generic Prompts Without DNA** -- "A beautiful abstract background" without DNA modifiers. Why bad: produces generic, non-distinctive imagery that doesn't match project identity. Instead: always include archetype style modifier, DNA color direction, and mood modifier.
- **Anti-Pattern: AI Images for Everything** -- Using AI-generated images for every visual on the page. Why bad: creates an artificial, homogeneous feel. Instead: mix AI-generated images with procedural shapes, real photography, and hand-crafted elements as appropriate for the archetype.
- **Anti-Pattern: Color Mismatch** -- AI-generated image palette doesn't match DNA tokens. Why bad: creates visual disconnect between image and UI. Instead: specify DNA colors precisely in prompt, post-process/filter images to match palette if needed.

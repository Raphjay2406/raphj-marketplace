**Layer 1: Decision Guidance** (~100-120 lines):

1. **When to Use 3D** -- Decision framework:
   - Is the 3D effect the HERO content (product viewer, interactive scene)? -> Full R3F scene
   - Is 3D atmospheric background (floating shapes, particles)? -> R3F with drei helpers (Float, Points)
   - Is it a subtle depth effect (parallax, tilt)? -> CSS 3D transforms (NOT R3F -- see Shape skill)
   - Is it post-processing only (bloom, noise, vignette on 2D content)? -> Consider CSS filters first
   - "If CSS can achieve the effect, use CSS. R3F adds ~150KB gzipped. Reserve it for genuine 3D."
   - Cross-reference: "For CSS pseudo-3D and isometric effects, see Shape & Asset Generation skill."

2. **Three-Tier Responsive 3D** -- Mandatory pattern for ALL 3D content:
   | Tier | Breakpoint | What renders | How |
   | Full | >= 1024px (desktop) | Complete 3D scene | R3F Canvas, full shaders, post-processing |
   | Reduced | >= 768px (tablet) | Simplified scene | Lower geometry detail, fewer particles, no post-processing |
   | Static | < 768px (mobile) | Fallback image | Static screenshot/illustration, lazy loaded |
   - Include the complete pattern from RESEARCH.md (dynamic import with ssr: false, useMediaQuery)
   - Always use `dynamic(() => import('./scene'), { ssr: false })` in Next.js
   - In Astro: `client:only="react"` directive

3. **3D Performance Budgets** (domain-specific, not covered by Phase 5 Performance skill):
   | Tier | Triangle Count | Texture Memory | Draw Calls | Target FPS |
   | Full (desktop) | < 500K | < 128MB | < 100 | 60fps |
   | Reduced (tablet) | < 100K | < 64MB | < 50 | 30fps |
   | Static (mobile) | 0 | 0 | 0 | N/A |
   - Monitor with `r3f-perf` during development
   - Cross-reference: "For general animation performance (code-splitting, lazy loading, reduced-motion), see Performance-Aware Animation skill."

4. **DNA Material Guidance:**
   - Primary material colors: USE DNA tokens (`new THREE.Color('hsl(var(--color-primary))')`)
   - Shader effects: CREATIVE FREEDOM to compute derived colors from DNA base (holographic rainbow from DNA primary, noise-generated gradients from DNA accent)
   - Environment/lighting: Match archetype mood (bright/even for Swiss, moody/dramatic for Neon Noir)
   - Forbidden: arbitrary hardcoded colors unrelated to DNA

5. **Technology Stack (Current 2026):**
   | Package | Version | Purpose |
   | @react-three/fiber | 9.5.x | React renderer for Three.js (stable, React 18+19) |
   | @react-three/drei | 10.7.x | Float, Environment, OrbitControls, useScroll, Html, Detailed |
   | @react-three/postprocessing | 3.x | EffectComposer with Bloom, Noise, Vignette, DepthOfField |
   | three-custom-shader-material | 6.x | Extend standard materials (csm_Position, csm_DiffuseColor) |
   | three | r171+ | Core 3D engine, WebGPU production-ready |

Tier 3 (guidance + reference):

13. Custom Post-Processing -- Approach: extend @react-three/postprocessing Effect class. Use cases: archetype-specific screen effects.

25. WebGPU Status (2026) -- Browser support table from RESEARCH.md (~95% global)

28. R3F v10 Alpha -- Native WebGPU/TSL support, but alpha. Target R3F v9 for production.
29. Warning: Never mix `three` and `three/webgpu` imports in the same codebase.

**Layer 3: Integration Context** (~40-50 lines):
- **-> Design DNA**: Primary colors from tokens. Shaders get creative freedom for derived/computed colors.
- **-> Cinematic Motion**: 3D animations use the archetype's base easing and duration from motion profiles.
- **-> Wow Moments**: 3D Product Viewer and WebGL Shader Background are Tier 3 wow moments. Reference wow moment auto-suggestion matrix.
- **-> Performance-Aware Animation**: General principles (code-splitting, lazy loading, reduced-motion) from Phase 5. 3D-specific budgets (triangles, textures, draw calls) defined HERE in this skill.
- **-> Shape & Asset Generation**: CSS pseudo-3D and isometric effects are in the Shape skill. This skill covers true WebGL/R3F 3D only.
- **-> Spline Integration**: Spline scenes are an alternative to code-built R3F scenes. Use Spline for designer-first workflows, R3F for developer-first workflows.

**Layer 4: Anti-Patterns** (~50-60 lines):
- **Anti-Pattern: Loading R3F on Mobile** -- Importing R3F Canvas without device detection. Why bad: ~150KB+ gzipped, poor mobile GPU performance. Instead: three-tier responsive with static fallback below 768px.
- **Anti-Pattern: Raw Three.js in React** -- Using `useEffect` with `new THREE.Scene()` manually instead of R3F. Why bad: manual lifecycle management, no React integration, verbose. Instead: R3F Canvas is the standard React integration.
- **Anti-Pattern: Full ShaderMaterial From Scratch** -- Writing complete vertex/fragment shaders replacing all standard material features. Why bad: lose lighting, shadows, PBR. Instead: THREE-CustomShaderMaterial extends standard materials with csm_Position, csm_DiffuseColor.
- **Anti-Pattern: No SSR Disabling** -- Rendering R3F Canvas in a server component. Why bad: Three.js requires browser APIs (WebGL context, window). Crashes on server. Instead: `dynamic(() => import('./scene'), { ssr: false })` in Next.js, `client:only="react"` in Astro.
- **Anti-Pattern: Mixing Three.js Import Paths** -- Using both `import from 'three'` and `import from 'three/webgpu'` in the same project. Why bad: Three.js docs explicitly warn against this. Instead: pick one import path for the entire project.
- **Anti-Pattern: Hardcoded Colors in Materials** -- `new THREE.Color('#8b5cf6')` without DNA reference. Why bad: decouples 3D from project identity. Instead: use DNA token values for primary colors.
- **Anti-Pattern: No Post-Processing Merging** -- Multiple separate render passes instead of EffectComposer. Why bad: each pass is a full-screen render. Instead: @react-three/postprocessing auto-merges compatible effects into minimal passes.

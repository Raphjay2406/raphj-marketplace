**Layer 1: Decision Guidance** (~60-80 lines):

1. **When to Use Remotion:**
   - Hero/background animated video that would be too heavy as realtime animation
   - Product showcase video (animated feature walkthrough, before/after)
   - Social media assets (OG images via `<Still>`, video previews for social sharing)
   - Micro-interaction recordings for documentation or portfolio
   - NOT for: realtime interactive animations (use Motion library or GSAP), simple CSS animations, 3D scenes (use R3F)
   - "Remotion is React for video -- if the output is a video file or video player, Remotion is the right tool. If the output is a live interactive element, use realtime animation."

2. **Licensing Model:**
   | Situation | License Required | Cost |
   | Individual developer | Free | $0 |
   | Company with <= 3 employees | Free | $0 |
   | Non-profit organization | Free | $0 |
   | Evaluation / prototyping | Free | $0 |
   | Company with > 3 employees | Company License | Paid |
   - **Important:** Document this clearly. Remotion is NOT fully open source for commercial use by larger companies. The skill should not present Remotion as "free" without this caveat.

3. **Composition vs Rendering:**
   - This skill focuses on COMPOSITION CREATION (React components + Remotion APIs)
   - Rendering (converting compositions to video files) is a deployment concern outside Modulo's scope
   - For in-browser preview: `@remotion/player` embeds compositions as interactive players
   - For video export: separate process using Remotion CLI or @remotion/renderer (mentioned but not detailed)

4. **DNA Integration Points:**
   | DNA Attribute | Remotion Usage |
   | Color palette | Background colors, text colors, accent elements, gradient stops |
   | Typography (display/body fonts) | Font family in composition text elements |
   | Motion language (easing, duration) | spring() config, interpolate easing, transition timing |
   | Archetype personality | Overall composition style (fast/snappy for Kinetic, slow/luxurious for Luxury) |
   | Signature element | Recurring visual motif in intro/outro sequences |

**Layer 2: Award-Winning Examples** (~220-280 lines):

1. **Core API Reference** (~50 lines):
   Quick reference table of essential Remotion APIs:
   | API | Purpose | Example |
   | `useCurrentFrame()` | Get current frame number (0-based) | `const frame = useCurrentFrame()` |
   | `useVideoConfig()` | Get width, height, fps, durationInFrames | `const { fps, width } = useVideoConfig()` |
   | `interpolate(value, inputRange, outputRange, options?)` | Map frame to CSS values | `interpolate(frame, [0, 30], [0, 1])` |
   | `spring({ frame, fps, config? })` | Physics-based 0->1 transition | `spring({ frame, fps, config: { damping: 100 } })` |
   | `<Composition>` | Define a video with component + metadata | `<Composition id="Hero" component={Hero} .../>` |
   | `<Still>` | Define a still image (single frame composition) | `<Still id="OGImage" component={OG} .../>` |
   | `<Sequence from? durationInFrames?>` | Time-shift children (nested sequences accumulate `from`) | `<Sequence from={30}>...</Sequence>` |
   | `<Series>` | Display children sequentially | `<Series><Series.Sequence>...</Series.Sequence></Series>` |
   | `Easing.*` | Easing functions for interpolate | `interpolate(frame, [0, 30], [0, 1], { easing: Easing.bezier(.2,0,0,1) })` |
   | Extrapolation | Clamp/extend output range | `{ extrapolateRight: 'clamp' }` |

2. **DNA-Aware Hero Video Composition** (~50 lines):
   Include the complete composition from RESEARCH.md (HeroVideo component). Expand it with:
   - DNA color props (primary, secondary, accent, surface)
   - DNA font family applied to text elements
   - Archetype-aware timing: spring config varies by archetype personality
   - Signature element animation in intro sequence
   - Example shows `<Composition>` registration in Root.tsx

3. **Product Demo Video Template** (~50 lines):

   - Feature callout sequence (highlight areas, add annotation text)
   - Before/after comparison (split screen, slide reveal)
   - Uses DNA colors for callout annotations, highlight borders

4. **Social Media Asset Generation** (~40 lines):
   - OG Image: `<Still>` composition with DNA colors, headline text, brand signature
   - Video Preview: Short (5-10 second) composition for social sharing
   - Template structure: headline + subtext + visual element + logo/brand mark
   - Size presets: OG (1200x630), Twitter (1200x675), Square (1080x1080), Story (1080x1920)

5. **Brownfield Installation** (~30 lines):
   ```bash

   npm install remotion @remotion/cli @remotion/player

   # Project structure:
   # src/remotion/
   #   Root.tsx          -- Composition registry
   #   compositions/
   #     HeroVideo.tsx   -- Individual compositions
   #     OGImage.tsx
   # remotion.config.ts  -- Remotion config (separate from Next.js)
   ```
   - Remotion runs alongside the web project, shares React components
   - DNA tokens imported as regular constants (not CSS variables -- video is not DOM)
   - For in-browser preview, use `<Player>` component from `@remotion/player`

6. **@remotion/player In-Browser Preview** (~30 lines):
   ```tsx
   'use client'
   import { Player } from '@remotion/player'
   import { HeroVideo } from '@/remotion/compositions/HeroVideo'

   function VideoPreview({ dnaColors }) {
     return (
       <Player
         component={HeroVideo}
         inputProps={{ colors: dnaColors, headline: "..." }}
         durationInFrames={150}
         fps={30}
         compositionWidth={1920}
         compositionHeight={1080}
         controls
         style={{ width: '100%' }}
       />
     )
   }
   ```
   - Player is interactive (play/pause/scrub)
   - Can be embedded in admin/preview pages
   - Does NOT require Remotion CLI for preview

7. **Archetype-Aware Composition Styles** (~30 lines):
   | Archetype Category | Timing | Visual Style | Transitions |
   | HIGH (Kinetic, Neon Noir) | Fast cuts, 15-20 frame transitions | Bold colors, high contrast, motion blur | Hard cuts, glitch transitions |
   | MEDIUM (Editorial, Neo-Corp) | Moderate, 25-30 frame transitions | Clean layouts, professional | Smooth fades, slide transitions |
   | LOW (Japanese Minimal, Swiss) | Slow, 40-50 frame transitions | Minimal elements, whitespace | Gentle fades, minimal movement |
   | BOLD (Brutalist, Neubrutalism) | Abrupt, 5-10 frame transitions | Raw, high contrast, no smoothing | Jump cuts, no easing |
   - spring() config maps to these: `{ damping: 12 }` for fast/bouncy (Kinetic), `{ damping: 200 }` for smooth/gentle (Luxury)

**Layer 3: Integration Context** (~30-40 lines):
- **-> Design DNA**: Composition colors, fonts, and motion timing all derive from DNA. Since video is not DOM, DNA tokens are imported as JS constants (not CSS variables).
- **-> Design Archetypes**: Archetype personality determines composition style (timing, transitions, visual density).
- **-> Cinematic Motion**: Remotion compositions share the archetype's motion philosophy but implement it via interpolate/spring (not CSS/Motion library).
- **-> Image Prompt Generation**: AI-generated images can be composed into Remotion videos. Use Image Prompt skill to generate assets, Remotion to animate them.
- **-> Design System Scaffold**: The scaffold can export DNA tokens as a JS module that Remotion compositions import directly.

**Layer 4: Anti-Patterns** (~40-50 lines):
- **Anti-Pattern: CSS Variables in Compositions** -- Using `hsl(var(--color-primary))` inside Remotion compositions. Why bad: Remotion renders in a headless browser context; CSS variables may not resolve. Instead: pass DNA colors as JS props to composition components.
- **Anti-Pattern: Licensing Ambiguity** -- Presenting Remotion as "free" or "open source" without caveat. Why bad: companies with >3 employees need a paid license. Instead: always document the licensing tiers.
- **Anti-Pattern: Complex Realtime Playback** -- Using Remotion Player for realtime interactive experiences. Why bad: Player is for preview, not production UX. It's heavier than CSS/Motion animation. Instead: use Remotion for video CONTENT, use CSS/Motion for realtime INTERACTION.
- **Anti-Pattern: One Massive Composition** -- Single 300-frame composition with everything. Why bad: hard to maintain, poor reusability. Instead: break into Sequences and separate compositions that can be combined.
- **Anti-Pattern: Ignoring Archetype Timing** -- Using the same spring/interpolation config for Kinetic and Japanese Minimal. Why bad: Kinetic should be fast/snappy, Japanese Minimal should be slow/deliberate. Instead: derive timing from archetype motion profile.
- **Anti-Pattern: Hardcoded Colors** -- Using `'#8b5cf6'` instead of DNA-derived color props. Why bad: composition doesn't match project identity. Instead: pass DNA palette as inputProps.

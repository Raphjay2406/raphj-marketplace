**Layer 1: Decision Guidance** (~50-60 lines):

1. **Spline vs R3F Decision:**
   - Use Spline when: designer creates the 3D scene visually, team has Spline skills, scene is primarily aesthetic/presentational, rapid prototyping of 3D
   - Use R3F when: scene needs custom shaders, complex procedural generation, deep React state integration, full programmatic control
   - Use R3F Bridge (@splinetool/r3f-spline) when: Spline for visual scene design + R3F for custom shaders, post-processing, or advanced animation
   - "Spline is the Figma of 3D. R3F is the code-first approach. Both are valid."

2. **When Spline Is Appropriate:**
   - Hero 3D scenes with interaction (rotate, hover effects)
   - Product showcases with predefined camera angles
   - Animated background elements
   - Interactive 3D elements that respond to user input
   - NOT for: data-driven 3D (use R3F), heavy real-time simulation (use R3F), simple CSS 3D effects (use CSS transforms)

3. **Performance Awareness:**
   - `.splinecode` files can be 5-50MB
   - Always lazy load Spline scenes
   - Never load eagerly on page init
   - Self-host `.splinecode` on CDN with proper cache headers
   - `renderOnDemand={true}` (default) renders only when scene changes -- good for static/hoverable scenes

**Layer 2: Award-Winning Examples** (~150-200 lines):

1. **Basic Embedding** (~30 lines):
   ```tsx
   'use client'
   import { lazy, Suspense } from 'react'

   const Spline = lazy(() => import('@splinetool/react-spline'))

   function SplineScene({ sceneUrl }: { sceneUrl: string }) {
     return (
       <Suspense fallback={
         <div className="aspect-video bg-surface animate-pulse rounded-xl" />
       }>
         <Spline scene={sceneUrl} className="w-full h-full" />
       </Suspense>
     )
   }
   ```
   - Always wrap in Suspense with DNA-styled placeholder
   - Always lazy load the Spline component

2. **Next.js SSR with Blurred Placeholder** (~25 lines):
   ```tsx
   'use client'
   import SplineNext from '@splinetool/react-spline/next'

   function SplineHero({ sceneUrl }: { sceneUrl: string }) {
     return (
       <SplineNext
         scene={sceneUrl}
         className="w-full h-full"
         // Next.js integration provides blurred placeholder automatically
       />
     )
   }
   ```
   - Uses `@splinetool/react-spline/next` import for SSR-optimized loading
   - Provides blurred placeholder while scene loads

3. **DNA Color Mapping via onLoad** (~40 lines):
   Include the pattern from RESEARCH.md (findObjectByName + color manipulation). Expand with:
   - How to find objects by name (naming convention in Spline: prefix with `DNA_` for mappable objects)
   - Color property format: `{ r: 0-255, g: 0-255, b: 0-255 }` (Spline uses 0-255 RGB)
   - DNA token conversion: parse `hsl(var(--color-primary))` to RGB values for Spline
   - Multiple object mapping example (primary shapes, accent elements, background)
   ```tsx
   const onLoad = useCallback((spline: any) => {
     // Map DNA colors to Spline objects
     const primaryShape = spline.findObjectByName('DNA_Primary')
     if (primaryShape) {
       // Convert DNA HSL to RGB for Spline
       primaryShape.color = { r: dnaRgb.r, g: dnaRgb.g, b: dnaRgb.b }
     }

     const accentGlow = spline.findObjectByName('DNA_Accent')
     if (accentGlow) {
       accentGlow.color = { r: accentRgb.r, g: accentRgb.g, b: accentRgb.b }
     }
   }, [dnaRgb, accentRgb])
   ```

4. **Event Handling** (~30 lines):
   - Mouse events: `onSplineMouseDown`, `onSplineMouseUp`, `onSplineMouseHover`
   - Emit events to Spline: `spline.emitEvent('mouseDown', 'ObjectName')`
   - Follow events: `onFollow` for camera follow targets
   ```tsx
   <Spline
     scene={sceneUrl}
     onSplineMouseDown={(e) => {
       // e.target.name identifies which object was clicked
       if (e.target.name === 'CTA_Button') {
         router.push('/signup')
       }
     }}
     onSplineMouseHover={(e) => {
       // Cursor change on hoverable objects
       document.body.style.cursor = e.target.name.startsWith('Hover_')
         ? 'pointer' : 'default'
     }}
   />
   ```

5. **R3F Bridge: Spline Design + R3F Control** (~30 lines):
   ```tsx
   'use client'
   import { Canvas } from '@react-three/fiber'
   import { Environment } from '@react-three/drei'
   import { SplineLoader } from '@splinetool/r3f-spline'
   import { EffectComposer, Bloom } from '@react-three/postprocessing'

   function SplineWithEffects({ sceneUrl }: { sceneUrl: string }) {
     return (
       <Canvas>
         <SplineLoader scene={sceneUrl} />
         <Environment preset="studio" />
         <EffectComposer>
           <Bloom luminanceThreshold={0.8} intensity={0.3} />
         </EffectComposer>
       </Canvas>
     )
   }
   ```
   - Combines Spline's visual scene design with R3F's shader/post-processing ecosystem
   - Use when: designer creates base scene in Spline, developer adds custom effects via R3F

6. **Performance Optimization Checklist** (~25 lines):
   - [ ] Lazy load Spline component (`React.lazy` or `next/dynamic`)
   - [ ] Use Suspense with DNA-styled placeholder
   - [ ] `renderOnDemand={true}` (default -- don't override unless continuous animation needed)
   - [ ] Self-host `.splinecode` files on CDN (not Spline's default hosting)
   - [ ] Set proper cache headers (immutable, 1 year max-age for versioned files)
   - [ ] Add three-tier responsive: skip Spline on mobile (<768px), show static fallback
   - [ ] Monitor `.splinecode` file size -- flag if > 10MB
   - [ ] For Next.js: use `@splinetool/react-spline/next` for blurred placeholder SSR

7. **Scene Creation Guidance** (~30 lines):
   When creating Spline scenes for a Genorah project, follow these DNA guidelines:
   - **Camera angles:** Match archetype mood (low dramatic for Neon Noir, eye-level neutral for Swiss, slightly elevated for Corporate)
   - **Lighting:** Mirror archetype lighting profile (bright/even for Minimal, moody/contrasty for Dark Academia, warm for Warm Artisan)
   - **Materials:** Use DNA color palette for primary objects. Name mappable objects with `DNA_` prefix.
   - **Animation:** Match archetype motion intensity (slow float for Ethereal, energetic spin for Kinetic, minimal for Swiss)
   - **Geometry complexity:** Keep polygon count reasonable. Scene should load in <3s on broadband.
   - **Export:** Export as `.splinecode` (not `.glb`) for Spline's optimized loading pipeline.

**Layer 3: Integration Context** (~25-35 lines):
- **-> Design DNA**: Colors mapped programmatically via onLoad. Naming convention (`DNA_` prefix) enables automatic mapping.
- **-> 3D/WebGL Effects**: Spline for designer-first scenes, R3F for developer-first scenes. R3F Bridge combines both.
- **-> Design Archetypes**: Scene creation guidance maps camera, lighting, and animation to archetype personality.
- **-> Performance-Aware Animation**: Spline scenes follow the same three-tier responsive pattern as R3F. Lazy loading mandatory.
- **-> Wow Moments**: Spline scenes can serve as PEAK or HOOK wow moments. Reference the wow moment auto-suggestion matrix.

**Layer 4: Anti-Patterns** (~30-40 lines):
- **Anti-Pattern: Eager Spline Loading** -- Loading Spline component at page init without lazy/Suspense. Why bad: 5-50MB .splinecode files block page load, destroy CWV. Instead: always lazy load with Suspense and placeholder.
- **Anti-Pattern: Spline Default Hosting in Production** -- Using Spline's default CDN URL in production. Why bad: no cache control, external dependency, potential downtime. Instead: self-host .splinecode on your CDN with proper cache headers.
- **Anti-Pattern: No Mobile Fallback** -- Rendering Spline scene on all devices. Why bad: heavy on mobile, poor touch interaction, battery drain. Instead: three-tier responsive with static image below 768px.
- **Anti-Pattern: Unnamed Objects** -- Not naming Spline objects for programmatic access. Why bad: can't map DNA colors or handle events. Instead: use `DNA_` prefix naming convention for all mappable objects.
- **Anti-Pattern: Continuous renderOnDemand=false** -- Setting `renderOnDemand={false}` for a static/hoverable scene. Why bad: renders every frame even when nothing changes, wasting GPU. Instead: keep default `renderOnDemand={true}` unless scene has continuous animation.
- **Anti-Pattern: Hardcoded Colors in Spline** -- Baking final colors into Spline scene without programmatic mapping. Why bad: can't update when DNA changes, creates maintenance burden. Instead: use neutral colors in Spline, map DNA colors via onLoad.

**Layer 4: Anti-Patterns (5-6 named mistakes with wrong/right approach):**

1. **"The WOFF2 Surprise"** -- Loading a WOFF2 font file into Satori. Satori silently falls back to a system font or throws a parsing error. OG image renders with Times New Roman.
   - Wrong: `readFile('fonts/Display.woff2')` passed to Satori
   - Right: Convert to TTF/OTF first. `readFile('assets/fonts/Display-Bold.ttf')`. Include conversion instruction in project setup
   - Detection: Text rendering looks different from expected, "Failed to decode font" errors

2. **"The Invisible Flexbox"** -- Forgetting `display: 'flex'` on container divs. Satori's Yoga engine needs explicit flex declarations. Layout appears broken with overlapping or collapsed elements.
   - Wrong: `<div style={{ padding: '60px', backgroundColor: '#...' }}>` (no display)
   - Right: `<div style={{ display: 'flex', flexDirection: 'column', padding: '60px', backgroundColor: '#...' }}>`
   - Detection: Elements don't respect alignment or direction properties

3. **"The Grid Trap"** -- Using CSS Grid in Satori templates. Satori only supports flexbox (Yoga engine). `display: 'grid'` silently fails with no error.
   - Wrong: `<div style={{ display: 'grid', gridTemplateColumns: '1fr 1fr' }}>`
   - Right: `<div style={{ display: 'flex', flexDirection: 'row' }}><div style={{ flex: 1 }}>...</div><div style={{ flex: 1 }}>...</div></div>`
   - Detection: Multi-column layouts collapse to single column

4. **"The Metadata Merge Trap"** (Next.js specific) -- Defining `openGraph` in a page's `generateMetadata` causes shallow replacement of the parent's entire openGraph object, losing the `images` field. This issue is covered by seo-meta skill, but especially relevant when mixing opengraph-image.tsx file convention with manual generateMetadata.
   - Wrong: Page sets `openGraph: { title: '...' }` thinking it merges with parent layout's openGraph.images
   - Right: Use `opengraph-image.tsx` file convention per-route (each route generates its own image, no inheritance needed). Or if using generateMetadata, always spread parent openGraph
   - Detection: Social preview shows site default image or no image on specific pages

5. **"The Bloated Canvas"** -- OG images with complex gradients, embedded photos, or uncompressed output exceeding 5MB. Social platforms reject or timeout on large images.
   - Wrong: Embedding a full photograph as base64 in the Satori template
   - Right: Keep OG images simple: text + solid/gradient background + small signature element. Sharp's `png()` compresses well by default. Target < 500KB
   - Detection: Slow social card loading, platforms showing fallback image

6. **"The Generic Preview"** -- Using the same layout and colors for every project regardless of archetype. OG images should reinforce the project's unique visual identity, not look like a generic social card generator.
   - Wrong: One fixed template with background + centered title for all projects
   - Right: Archetype-influenced composition (bold/maximalist vs elegant/minimal vs expressive), DNA token colors, signature element
   - Detection: Social cards from different Genorah projects look identical

**Machine-Readable Constraint Table:**

| Parameter | Min | Max | Unit | Enforcement |
|-----------|-----|-----|------|-------------|
| Image width | 1200 | 1200 | px | HARD |
| Image height | 630 | 630 | px | HARD |
| Image file size | - | 500 | KB | SOFT |
| Image format | PNG | PNG | - | HARD |
| Font format | TTF | OTF | - | HARD (no WOFF2) |
| Title font size | 48 | 72 | px | SOFT |
| Title character limit | - | 80 | chars | SOFT |
| Signature element | required | required | - | HARD |
| Display flex on containers | required | required | - | HARD |
| CSS Grid usage | forbidden | forbidden | - | HARD |

Note on enforcement:
- HARD = violation causes incorrect output (broken image, wrong size, crash)
- SOFT = violation degrades quality (oversized file, truncated title) but image still generates

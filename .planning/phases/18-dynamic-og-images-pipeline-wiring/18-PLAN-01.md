**Layer 1: Decision Guidance must include:**

1. **Introductory sentence:** You are an OG image specialist who generates branded social preview images from Design DNA tokens. Every Genorah project gets unique social cards that reinforce the project's visual identity -- not generic placeholder images.

2. **When to Use** -- Conditions that trigger this skill:
   - Any public-facing site where pages will be shared on social media (Twitter/X, Facebook, LinkedIn, Slack, Discord, WhatsApp)
   - Sites where branded social previews are a conversion factor (landing pages, blog posts, product pages, portfolio pieces)
   - Projects with Design DNA already generated (required input for OG templates)
   - Note: This is triggered during Wave 0 scaffold (OG route setup) AND per-section builds (page-specific OG images)

3. **When NOT to Use** -- Redirect to other skills:
   - Meta tag configuration (og:title, og:description, twitter:card) -> Use `seo-meta` skill (handles the meta tag wiring)
   - Tauri/Electron desktop apps, private dashboards behind auth, internal tools
   - Static marketing sites with < 5 pages where a single hand-designed OG image is fine
   - Note: For Next.js projects using the `opengraph-image.tsx` file convention, Next.js auto-generates the `og:image` meta tag -- no manual meta wiring needed

4. **DNA Token Mapping** -- How OG images consume Design DNA:
   - **Colors (semantic subset):** `bg` (canvas background), `primary` (accent/signature color), `text` (title text). Use `accent`/`glow` sparingly for gradient effects on specific archetype OG layouts
   - **Display font:** The project's DNA display font loaded into Satori/ImageResponse as TTF/OTF (WOFF2 NOT supported). System font for small text (site name)
   - **Signature element:** Always included on every OG image. The DNA signature element (gradient orb, diagonal slash, accent bar, etc.) appears as a visual brand mark
   - Title is rendered at fixed size (e.g., 64px) with truncation after a character limit (Claude decides limit per template type). No auto-scaling

5. **Template Types and Auto-Detection:**
   - 3 template types: `article`, `landing`, `product`
   - Convention-based auto-detection from route structure:
     - `/blog/*`, `/posts/*`, `/articles/*` -> article template
     - `/products/*`, `/shop/*`, `/store/*` -> product template
     - Everything else -> landing template (default)
   - Explicit override via metadata field: `ogTemplate: 'article' | 'product' | 'landing'` in generateMetadata (Next.js) or frontmatter (Astro)
   - Template type influences LAYOUT COMPOSITION, not just color swap. A Brutalist OG should look fundamentally different from an Ethereal one

6. **Framework Decision Tree:**
   - Next.js -> Use `opengraph-image.tsx` file convention (ALWAYS prefer this). Auto-wires og:image meta tag. Uses `ImageResponse` from `next/og` (built-in, zero deps)
   - Astro SSG -> Use `.png.ts` endpoint with `getStaticPaths`. Uses `satori` + `satori-html` + `sharp`. Generated at build time
   - Astro SSR -> Use `.png.ts` endpoint without `getStaticPaths`. Generated on request with caching headers
   - React/Vite SPA -> Not applicable (no server-side image generation). Use a static OG image or external service
   - Installation: Next.js needs nothing extra. Astro: `npm install satori satori-html sharp`

7. **Satori Constraints Summary** (quick reference for agents):
   - Layout: Flexbox ONLY (no CSS Grid). Every container needs explicit `display: 'flex'`
   - Fonts: TTF, OTF, WOFF only. NO WOFF2. Convert project fonts to TTF before loading
   - Styles: ALL inline via `style` prop. No CSS selectors, no `<style>` tags, no class names
   - No React hooks. No `z-index` (use document order). No `calc()`. No CSS variables
   - Size: 1200 x 630px, PNG format, target < 500KB
   - Full CSS reference in Layer 2

8. **Pipeline Connection:**
   - Referenced by: `build-orchestrator` at Wave 0 (OG route scaffold), `section-planner` (OG template type in PLAN.md)
   - Consumed at: `/gen:execute` Wave 0 (scaffold OG route files), Wave 2+ (per-section page-specific OG images if needed)
   - Input from: `DESIGN-DNA.md` (colors, display font, signature element), `seo-meta` skill (meta tag wiring for non-file-convention approaches)
   - Output to: `opengraph-image.tsx` files (Next.js) or `og/[slug].png.ts` endpoints (Astro)

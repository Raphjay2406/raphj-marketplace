**Layer 4: Anti-Patterns must include (8 anti-patterns):**

Source from 17-RESEARCH.md "Anti-Patterns to Avoid" and "Common Pitfalls" sections. Rewrite in the skill's Layer 4 format (Name / What goes wrong / Instead).

1. **Client-Side Secret Exposure**
   **What goes wrong:** Developer puts API keys in `NEXT_PUBLIC_*`, `PUBLIC_*`, or `VITE_*` env vars. Secrets appear in the client JavaScript bundle, visible in browser DevTools. Anyone can extract and abuse the key.
   **Instead:** ALL secrets use unprefixed env var names. Access only in server-side code (`'use server'` files, Route Handlers, Astro API endpoints). If the code runs in the browser, it cannot access the secret. Validation rule: if any env var with public prefix contains "KEY", "SECRET", "TOKEN", or "PASSWORD", it is a critical error.

2. **JSON-Parsing Webhook Body Before Verification**
   **What goes wrong:** Developer calls `request.json()` before verifying the webhook signature. The parsed object is serialized differently than the raw payload, so the HMAC digest does not match. Verification fails on every real webhook. Works in local testing only because test payloads may be simpler.
   **Instead:** Always use `request.text()` first. Verify the signature against the raw string. Parse JSON only AFTER verification succeeds. This applies to Stripe (`stripe.webhooks.constructEvent` requires raw body), GitHub (HMAC-SHA256 against raw payload), and any HMAC-based verification.

3. **String Comparison for Webhook Signatures**
   **What goes wrong:** Developer uses `===` to compare HMAC signatures. This is vulnerable to timing attacks -- an attacker can determine the correct signature byte by byte by measuring response times.
   **Instead:** Always use `crypto.timingSafeEqual(Buffer.from(computed), Buffer.from(received))`. This compares in constant time regardless of where signatures differ. Wrap in try/catch because `timingSafeEqual` throws if buffers have different lengths.

4. **Unbounded Retries Without Backoff**
   **What goes wrong:** Retry loop without delay or max attempts. When the upstream API is down, this hammers it with requests, potentially triggering IP bans or rate limit blocks. Also blocks the user's request indefinitely.
   **Instead:** Cap at 3 retries with exponential backoff (1s, 2s, 4s base delay + jitter). Never retry 4xx errors (except 429). Respect `retry-after` headers. For 429 responses, wait the indicated duration before retrying.

5. **HubSpot Missing objectTypeId**
   **What goes wrong:** HubSpot Forms API v3 submission fails with a 400 validation error because `objectTypeId` is not included in field objects. Many tutorials and older examples omit it.
   **Instead:** Always include `objectTypeId: "0-1"` for Contact properties in the fields array. This is required by the v3 API even though it was not required in v2.

6. **Salesforce Web-to-Lead JSON Parsing**
   **What goes wrong:** Developer expects a JSON response from Salesforce Web-to-Lead and calls `response.json()`. Gets a parse error because Salesforce returns HTML or a redirect, not JSON.
   **Instead:** Set `redirect: 'manual'` in fetch options. Check for 200 or 302 status as success indicators. Do NOT attempt to parse the response body as JSON. Web-to-Lead is a redirect-based form handler from the early 2000s -- it predates JSON APIs.

7. **Turnstile Token Reuse**
   **What goes wrong:** Form retries or double-submissions send the same Turnstile token twice. The second validation fails with `timeout-or-duplicate` error because tokens are single-use.
   **Instead:** After submission (success or failure), reset the Turnstile widget using the ref's `.reset()` method to generate a fresh token. Tokens also expire after 300 seconds (5 minutes), so handle the `onExpire` callback to auto-reset.

8. **Context7 Over-Reliance Without Baseline**
   **What goes wrong:** Skill or agent relies exclusively on Context7 MCP for API documentation, but Context7 may not have the library, may have an outdated version, or the MCP server may be unavailable. Build fails or produces incorrect patterns.
   **Instead:** The skill contains hardcoded, version-pinned baseline patterns for every integration. Context7 is a freshness upgrade, not the primary source. Fallback chain: (1) Try Context7, (2) Use skill baseline, (3) Use official docs URL, (4) Flag to user.

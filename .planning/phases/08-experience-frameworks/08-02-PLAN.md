**Layer 1: Decision Guidance (~80 lines)**

Core philosophy: 375px is the design target. Fluid scaling (clamp, vw units) gracefully handles down to ~320px without horizontal scroll. Design is mobile-first with dramatic recomposition at breakpoints.

Breakpoint system (Tailwind v4 defaults):
- `sm`: 640px (large phones landscape)
- `md`: 768px (tablets)
- `lg`: 1024px (small laptops)
- `xl`: 1280px (desktops)
- `2xl`: 1536px (large screens)

Decision tree:
- Body/paragraph text -> `clamp(rem, rem + vw, rem)` for fluid scaling
- Display/heading text -> Breakpoint steps (`text-4xl md:text-6xl lg:text-8xl`) for dramatic size jumps
- Reusable component layout -> Container queries (`@container` + `@sm:`, `@md:`)
- Page-level layout (columns, sidebar) -> Media queries (`sm:`, `md:`, `lg:`)
- Navigation structure -> Media queries (fundamental structure change between mobile/desktop)
- Touch targets -> 44x44px minimum, enforced on all interactive elements at all sizes
- Images -> `srcset` with responsive sizes, `loading="lazy"`, explicit `width`/`height` for CLS prevention

Typography hybrid rationale: Body text scaling smoothly makes reading comfortable. Display text jumping dramatically at breakpoints matches layout recomposition -- when the layout fundamentally changes at `md:`, the heading size should also jump to match the new hierarchy.

When NOT to use: Never. Responsive design is always active.

**Layer 2: Award-Winning Examples (~180 lines)**

Pattern 1: **Hybrid Typography System**
Show complete CSS custom properties setup:
- Body text: `--text-body: clamp(1rem, 0.875rem + 0.5vw, 1.125rem)` -- fluid from 16px to 18px
- Body large: `--text-body-lg: clamp(1.125rem, 1rem + 0.5vw, 1.25rem)`
- Caption: `--text-caption: clamp(0.75rem, 0.7rem + 0.25vw, 0.875rem)`
- Display text: stepped via breakpoints (2.5rem -> 4.5rem -> 6rem -> 8rem at 375/768/1024/1440)
- Heading 1: stepped (2rem -> 3rem -> 4rem)
- Heading 2: stepped (1.5rem -> 2rem -> 2.5rem)

IMPORTANT: clamp() must use `rem + vw` combination, NEVER pure `vw`. Pure `vw` violates WCAG 1.4.4 Resize Text because it doesn't respond to browser zoom.

Pattern 2: **Dramatic Layout Recomposition**
Show a hero section that fundamentally changes between mobile and desktop:
- Mobile (375px): Stacked layout, full-width image, text below, CTA at bottom
- Tablet (768px): Side-by-side with image taking 40%, text 60%
- Desktop (1024px): Asymmetric grid with text left (45%), visual right (55%), overlapping element
- Large (1440px): Max-width contained, generous whitespace margins

This is NOT just "stacking columns" -- each breakpoint is a different composition.

Pattern 3: **Container Query Component**
Show a card component that responds to its container, not the viewport:
- Narrow container: Vertical stack, small image, compact text
- Medium container: Horizontal layout, image left, text right
- Wide container: Full featured with image, title, description, metadata, actions
- Use Tailwind v4 `@container` syntax: `@container`, `@sm:flex-row`, `@md:grid-cols-2`

Pattern 4: **Mobile Navigation Patterns**
Show 3 navigation approaches with breakpoint-based switching:
- Off-canvas drawer (hamburger -> slide-in panel) for content-heavy sites
- Bottom tab bar for app-like experiences (44px touch targets, 5 items max)
- Collapsing top nav (full links desktop -> hamburger mobile)
Include keyboard accessibility for each: focus trap in drawer, arrow key nav in tab bar.

Pattern 5: **Touch-Optimized Interactive Elements**
Show the 44x44px minimum touch target enforcement:
- Buttons: `min-h-[44px] min-w-[44px] px-4 py-2`
- Links in lists: `py-3` minimum (48px with text)
- Icon buttons: Padding around icon to reach 44px hit area
- Close buttons: Large enough without looking oversized
- Spacing between adjacent touch targets: minimum 8px gap to prevent misclicks

**Layer 3: Integration Context (~60 lines)**

DNA Connection: DNA type scale maps to the hybrid system. DNA spacing scale provides consistent spacing at all breakpoints. DNA breakpoint behavior should be documented in DESIGN-DNA.md.

Archetype Variants: All archetypes use the same responsive system. Archetype personality shows through WHAT recomposes (e.g., Brutalist might use harsh grid shifts, Ethereal might use flowing transitions between layouts), not whether responsive works.

Pipeline Stage: Section builders produce responsive code by default. This is not a review-time check -- responsive is baked into every component pattern.

Related Skills: tailwind-system (container queries, breakpoint classes), accessibility (touch targets, zoom compliance), emotional-arc (beat parameters include viewport height constraints that must be responsive).

**Layer 4: Anti-Patterns (~50 lines)**

Anti-pattern 1: **Pure vw font sizing** -- What: `font-size: 3vw`. Why wrong: Ignores browser zoom (WCAG 1.4.4). Fix: `clamp(rem, rem + vw, rem)`.

Anti-pattern 2: **Desktop-first with mobile afterthought** -- What: Designing at 1440px, then "fixing" mobile. Why wrong: Mobile layout is an afterthought, not a design. Fix: Start at 375px, recompose up.

Anti-pattern 3: **Column-stacking as responsive strategy** -- What: `md:grid-cols-3` and nothing else. Why wrong: Stacking is lazy. Fix: Each breakpoint is a new composition.

Anti-pattern 4: **Tiny touch targets** -- What: 24px icon buttons. Why wrong: Fingers need 44px minimum. Fix: Padding to 44px minimum, 8px gap between targets.

Anti-pattern 5: **Viewport-based component responsiveness** -- What: `sm:flex-row` on a reusable card. Why wrong: Card in sidebar vs main content needs different breakpoints. Fix: Container queries for components.

Anti-pattern 6: **Fixed widths** -- What: `w-[400px]` on content containers. Why wrong: Breaks on smaller screens. Fix: `max-w-[400px] w-full`.

**Machine-Readable Constraint Block:**
| Parameter | Min | Max | Unit | Enforcement |
|---|---|---|---|---|

| Fluid floor | ~320 | - | px | Graceful degradation below 375px |
| Touch target | 44 | - | px | All interactive elements |
| Touch gap | 8 | - | px | Between adjacent targets |
| Body text clamp | rem + vw | - | formula | Never pure vw |
| Display text | breakpoint-stepped | - | - | Dramatic jumps at md/lg/xl |

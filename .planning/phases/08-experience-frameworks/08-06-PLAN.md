**Layer 1: Decision Guidance (~80 lines)**

Framework detection: `next.config.ts` or `next.config.js` -> Next.js. Sub-detection:
- `app/` with `layout.tsx` -> App Router
- `pages/` with `_app.tsx` -> Pages Router
- Both directories -> Hybrid (both routers)
- Only `package.json` dependency -> Ask user

Router choice decision tree:
- New project (2026+) -> App Router (React 19.2, RSC, streaming, Cache Components)
- Existing Pages Router project -> Pages Router patterns (still fully supported in Next.js 16)
- Both routers present -> Document both, prefer App Router for new routes

CRITICAL Breaking Changes (v14/15 -> v16):
| Old | New |
|---|---|
| `middleware.ts` | `proxy.ts` with `export default function proxy()` |
| `params.slug` (sync) | `const { slug } = await params` (async) |
| `cookies()` (sync) | `const cookieStore = await cookies()` (async) |
| `headers()` (sync) | `const headersList = await headers()` (async) |
| `experimental: { ppr: true }` | `cacheComponents: true` |
| Webpack (default) | Turbopack (default) |
| Implicit `default.js` | Explicit `default.js` required for all parallel route slots |

When NOT to use: When project uses Astro, React/Vite, Tauri, or Electron (use those framework skills instead).

Pattern 3: **Async Params and Dynamic Routes (Next.js 16)**
```tsx
// app/blog/[slug]/page.tsx
export default async function BlogPost({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params  // MUST await in Next.js 16
  const post = await getPost(slug)
  return (
    <article className="max-w-prose mx-auto px-4 py-12">
      <h1 className="font-display text-display">{post.title}</h1>
      <div className="prose prose-lg">{post.content}</div>
    </article>
  )
}
```

Pattern 4: **proxy.ts (replaces middleware.ts)**
```tsx
import { NextRequest, NextResponse } from 'next/server'

export default function proxy(request: NextRequest) {
  // Auth check, redirects, rewrites
  if (!request.cookies.get('session')) {
    return NextResponse.redirect(new URL('/login', request.url))
  }
  return NextResponse.next()
}
```

**Layer 1: Decision Guidance (~70 lines)**

Framework detection: `astro.config.mjs` or `astro.config.ts` -> Astro.

Astro's key differentiator: Static-first with Islands of interactivity. Most Genorah output is static HTML with targeted interactive components (scroll animations, theme toggles, mobile nav).

Forward-compatibility rules (work in BOTH Astro 5 and 6):
| Astro 5 | Astro 6 (breaking) | Forward-compatible |
|---|---|---|
| `<ViewTransitions />` works | Removed | Use `<ClientRouter />` (works in both) |
| `Astro.glob()` works | Removed | Use `getCollection()` from Content Layer API |
| Legacy content collections | Removed | Use Content Layer API with loaders |
| Node 18+ | Node 22+ required | Target Node 22+ |
| Zod 3 | Zod 4 required | Use Zod 4 schema syntax |

When to use Astro: Content-heavy sites, marketing pages, blogs, portfolios, documentation. When NOT to use: Highly interactive apps (use Next.js or React/Vite).

Pattern 2: **Base Layout with DNA Integration**
```astro
---
import { ClientRouter } from 'astro:transitions'
---
<html lang="en">
<head>
  <ClientRouter />
  <link rel="stylesheet" href="/styles/globals.css" />
  <script is:inline>
    // FOUC prevention -- runs before rendering
    (function() {
      const theme = localStorage.getItem('theme')
      const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches
      if (theme === 'dark' || (!theme && prefersDark)) {
        document.documentElement.classList.add('dark')
      }
    })()
  </script>
</head>
<body class="bg-bg text-text font-body">
  <a href="#main" class="sr-only focus:not-sr-only ...">Skip to content</a>
  <Navigation transition:persist />
  <main id="main" transition:animate="slide">
    <slot />
  </main>
  <Footer />
</body>
</html>
```

Pattern 3: **Islands Architecture**

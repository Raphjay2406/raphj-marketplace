**Layer 1: Decision Guidance (~80 lines)**

Core philosophy: A multi-page site is not N independent pages sharing a navbar. It is a single design system with page-type-specific content structures, each page having its own emotional arc appropriate to its purpose.

Site-level DNA extensions (beyond single-page DNA):
- Navigation pattern (persistent across all pages)
- Footer design (consistent but page-type-aware)
- Page transition style (between pages, not within pages)
- Breadcrumb / location system
- Shared interactive components (search, theme toggle, mobile menu)
- Consistent sidebar (if applicable, e.g., docs)
- Global announcement/banner pattern

Decision tree:
- Single landing page -> Standard Genorah workflow, no multi-page skill needed
- Marketing site (landing + about + pricing + contact) -> Site DNA + 4 page templates
- Blog/content site -> Site DNA + blog template + content layout + taxonomy pages
- Documentation site -> Site DNA + docs template + sidebar nav + version switching
- Full product site -> Site DNA + all page types, shared component library
- Mixed (marketing + blog + docs) -> Site DNA + multiple templates, consistent chrome

When to use: Any project with 2+ pages.
When NOT to use: Single-page projects.

Pattern 2: **Page-Type Templates with Emotional Arcs**
Define 6 page types with their characteristic emotional arcs:

**Landing Page** (primary conversion):
Arc: HOOK -> TEASE -> BUILD -> PEAK -> PROOF -> BREATHE -> TENSION -> CLOSE
- HOOK: Full-viewport hero with value proposition
- BUILD: Feature showcase, progressive complexity
- PEAK: Wow moment (creative tension, demo, showcase)
- PROOF: Testimonials, logos, case studies
- CLOSE: Final CTA with urgency

**About Page** (trust building):
Arc: BUILD -> PROOF -> BREATHE -> BUILD -> CLOSE
- BUILD: Company story, mission, values
- PROOF: Team, achievements, timeline
- BREATHE: Full-width visual / brand moment
- BUILD: Culture, approach, differentiators
- CLOSE: Join us / contact CTA

**Pricing Page** (decision support):
Arc: HOOK -> BUILD -> TENSION -> PROOF -> CLOSE
- HOOK: Clear value headline
- BUILD: Pricing tiers with feature comparison
- TENSION: "Most popular" / time-limited offer
- PROOF: Testimonials from paying customers
- CLOSE: Start free / contact sales

**Blog/Article Page** (content consumption):
Arc: HOOK -> BUILD -> BREATHE -> BUILD -> CLOSE
- HOOK: Title, meta, hero image
- BUILD: Article content (progressive sections)
- BREATHE: Pull quote or visual break
- BUILD: Continued content
- CLOSE: Related articles, newsletter signup

**Documentation Page** (reference/learning):
Arc: BUILD -> BREATHE -> BUILD
- Minimal arc -- docs are functional, not emotional
- Sidebar navigation is the primary structure
- Content sections with code examples
- Consistent heading hierarchy
- No PEAK/TENSION -- docs should be calm and clear

**Contact Page** (conversion):
Arc: HOOK -> BUILD -> PROOF -> CLOSE
- HOOK: Headline with clear intent
- BUILD: Contact form + alternative methods
- PROOF: Response time guarantee, office location, map
- CLOSE: Submit with confirmation message

Pattern 3: **Shared Component Design**
Navigation component that works across all page types:
- Consistent branding (logo, colors from DNA)
- Page-aware active states (current page indicator)
- Responsive: full nav on desktop, hamburger/drawer on mobile
- Accessibility: skip link, aria-current="page", keyboard nav
- Theme toggle integration (if dark/light mode enabled)

Footer component:
- Consistent across all pages
- Mega footer for marketing sites, minimal for docs
- Responsive: multi-column grid desktop, stacked mobile
- Required links: privacy, terms, sitemap

Pattern 4: **Cross-Page Consistency Rules**
Rules that prevent multi-page sites from feeling disjointed:
- Same DNA tokens everywhere (no page-specific color overrides)
- Same type scale and spacing scale
- Consistent component design (buttons, cards, forms look identical across pages)
- Consistent motion language (same timing, easing, patterns)
- Page-specific content density is OK (docs are denser than landing)
- Page transitions match site personality, not individual page personality

**Layer 3: Integration Context (~60 lines)**

DNA Connection: DESIGN-DNA.md gains a "Site Architecture" section for multi-page projects. This includes navigation style, footer style, page transition style, and shared component declarations. Generated during `/gen:start-design` when multi-page is detected.

Archetype Variants: Archetypes influence navigation style (Brutalist might use stark text nav, Ethereal might use floating minimal nav), footer complexity, and page transition animations. The page-type emotional arcs are archetype-independent (a pricing page needs TENSION regardless of archetype), but the visual execution of each beat varies by archetype.

Pipeline Stage: Section planner uses this skill to create per-page emotional arcs in MASTER-PLAN.md. Build orchestrator ensures shared components are built in Wave 1 before page-specific sections. Quality reviewer checks cross-page consistency.

Related Skills: emotional-arc (beat definitions, parameter constraints), responsive-design (nav responsive patterns, touch targets), accessibility (skip links, aria-current, keyboard nav), design-dna (site-level extensions).

**Layer 4: Anti-Patterns (~40 lines)**

Anti-pattern 1: **Independent page design** -- What: Each page designed from scratch. Why wrong: No cohesion, feels like different sites. Fix: Site DNA + shared components.

Anti-pattern 2: **Landing page arc on every page** -- What: HOOK-BUILD-PEAK-CLOSE on docs page. Why wrong: Not every page needs emotional intensity. Fix: Page-type-appropriate arcs.

Anti-pattern 3: **Copy-paste navigation** -- What: Navigation duplicated in each page file. Why wrong: Drift, maintenance burden. Fix: Shared component, imported everywhere.

Anti-pattern 4: **Page-specific color overrides** -- What: Pricing page uses different brand colors. Why wrong: Breaks DNA consistency. Fix: One palette for entire site.

Anti-pattern 5: **Missing inter-page links** -- What: Pages exist but have no cross-navigation. Why wrong: Users get stuck. Fix: Related content, breadcrumbs, consistent nav.

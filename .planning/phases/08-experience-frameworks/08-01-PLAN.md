**Layer 1: Decision Guidance (~80 lines)**

When to use: Every project. Tailwind v4 is the styling system for all Genorah output.

Decision tree:
- Token configuration: Always use `@theme { }` CSS blocks, never `tailwind.config.ts` for colors/fonts/spacing
- Dark mode: Use `@custom-variant dark (&:where(.dark, .dark *))` in CSS, never `darkMode: 'class'` in config
- Container queries: Built-in `@container`, `@sm:`, `@md:`, `@lg:` -- no plugin needed
- Custom variants: Use `@custom-variant` for any project-specific variants
- Animation presets: Define `@keyframes` inside `@theme { }` block
- PostCSS/Autoprefixer: Built into Tailwind v4, no separate config needed

v3 -> v4 migration table (critical for avoiding outdated patterns):

| v3 (NEVER use) | v4 (ALWAYS use) |
|---|---|
| `tailwind.config.ts` for colors/fonts | `@theme { --color-*: value }` in CSS |
| `@tailwind base; @tailwind components; @tailwind utilities;` | `@import "tailwindcss"` |
| `darkMode: 'class'` in config | `@custom-variant dark (&:where(.dark, .dark *))` in CSS |
| `@tailwindcss/container-queries` plugin | Built-in `@container` |
| `theme.extend.colors` in JS | `--color-*: value` in `@theme` |
| Separate PostCSS + Autoprefixer | Built into Tailwind v4 |
| `@layer base { }` for resets | `@layer base { }` still works but `@theme` preferred for tokens |

When NOT to use: Never. This skill is always active.

**Layer 4: Anti-Patterns (~60 lines)**

Anti-pattern 1: **tailwind.config.ts for tokens** -- What: Using JS config for colors, fonts, spacing. Why wrong: v4 uses CSS-first @theme. Fix: Move all tokens to @theme CSS block.

Anti-pattern 2: **v3 @tailwind directives** -- What: `@tailwind base; @tailwind components; @tailwind utilities;`. Why wrong: Replaced by single `@import "tailwindcss"`. Fix: Single import.

Anti-pattern 3: **Container query plugin** -- What: Installing `@tailwindcss/container-queries`. Why wrong: Built into v4. Fix: Remove plugin, use `@container` directly.

Anti-pattern 4: **Arbitrary hex values** -- What: Using `bg-[#ff6f3c]` instead of `bg-primary`. Why wrong: Breaks DNA token system, impossible to theme. Fix: All colors via DNA tokens in @theme.

Anti-pattern 5: **darkMode config key** -- What: `darkMode: 'class'` in tailwind.config.ts. Why wrong: v4 uses `@custom-variant dark` in CSS. Fix: CSS-first dark mode declaration.

Anti-pattern 6: **Mixing v3 and v4 patterns** -- What: Some tokens in @theme, others in tailwind.config.ts. Why wrong: Confusing, hard to maintain. Fix: All configuration in CSS.

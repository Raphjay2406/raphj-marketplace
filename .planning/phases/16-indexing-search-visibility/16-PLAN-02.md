# Search Visibility: llms.txt Templates & Generation

> Status: Forward-looking convention (~844k adopters, NO confirmed AI platform consumption)
> Spec: llmstxt.org (proposed Sep 2024 by Jeremy Howard)

## What llms.txt Is (and Isn't)

[3-4 sentences: llms.txt is a proposed convention for helping AI systems understand site content. It sits at the site root (/llms.txt) as a markdown file. ~844,000 sites have adopted it (Oct 2025), including Anthropic, Cloudflare, Stripe, Mintlify, and GitBook. HOWEVER: no major AI platform (OpenAI, Google, Anthropic) has confirmed they read these files. Low implementation effort makes it worth including; do not over-invest.]

## Format Specification

### llms.txt (Summary Variant)

Required:
- H1: Site/project name (ONLY mandatory element)

Optional (in order):
- Blockquote: Short project summary
- Paragraphs: Additional context (no headings between H1 and first H2)
- H2 sections: Categorized page lists with markdown links
- `## Optional` section: Secondary content AI systems can skip for shorter context

Link format: `- [Page Title](https://example.com/page): Brief description`

[Include the complete llms.txt template from 16-RESEARCH.md "Pattern 5"]

### llms-full.txt (Detailed Variant)

Same H1/H2 structure as llms.txt but includes full page content inline:
- Each linked page's content reproduced under H3 headings
- Source URLs included for attribution
- Much larger file (acceptable -- AI systems handle long context)

[Template showing the llms-full.txt structure with placeholder content]

## Generation Approaches

### Manual Template (Recommended for most sites)

Create `public/llms.txt` with manually curated content. Best when:
- Site has < 50 key pages
- Content structure is stable
- You want editorial control over what AI systems see

[Copy the template from 16-RESEARCH.md "Pattern 5" with additional annotations]

### Auto-Generation: Next.js Route Handler

For sites with many pages or frequent content changes. Generates llms.txt dynamically from a page registry.

[Use the complete Next.js pattern from 16-RESEARCH.md "llms.txt Auto-Generation from Sitemap" -- the app/llms.txt/route.ts pattern with page definitions and GET handler. Include the Cache-Control header for 24-hour caching.]

### Auto-Generation: Astro Endpoint

[Use the Astro pattern from 16-RESEARCH.md "Astro llms.txt (Static File)" -- the src/pages/llms.txt.ts endpoint pattern. Note: for SSG Astro, this generates at build time. For SSR, it generates per request.]

### Auto-Generation: llms-full.txt

For the full variant, extend the auto-generation pattern:
- Fetch each page's rendered content (or raw markdown source)
- Include under H3 headings with source URLs
- Consider implementing as a build script rather than runtime route (large file, rarely changes)

## What NOT to Do

- Do not spend hours optimizing llms.txt content when basic SEO (meta tags, sitemaps, structured data) is incomplete
- Do not claim llms.txt improves SEO rankings (no evidence)
- Do not include sensitive or private content in llms.txt
- Do not auto-generate from ALL pages -- curate the most important ones
- Do not forget the ## Optional section for secondary content

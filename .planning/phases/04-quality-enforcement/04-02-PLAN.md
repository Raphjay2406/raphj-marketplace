**Layer 1: Decision Guidance (When/Why)**
- WHY structural enforcement: Layout diversity caught in review means sections must be rebuilt. Pre-assignment in MASTER-PLAN.md prevents the problem from occurring.
- WHY 18 patterns: A typical premium landing page has 6-10 sections. Adjacency rules (no same group adjacent) require ceil(N/2)+1 unique groups for N sections. 6 groups with 3 patterns each gives 18 patterns and comfortable margin for any page length.
- WHY visual groups (not just pattern names): Two patterns can have different names but look similar (e.g., "bento-grid" and "asymmetric-grid" are both grid layouts). Grouping by VISUAL character (centered vs. split vs. grid vs. flowing vs. full-bleed vs. layered) prevents subtle repetition.
- DUAL enforcement (LOCKED DECISION): Primary enforcement at planning time (MASTER-PLAN.md pre-assignment). Secondary enforcement at review time (quality-reviewer validates built output matches assigned pattern -- safety net only).
- Background alternation: Part of diversity enforcement. No two adjacent sections share the same background token (bg-primary, bg-secondary, bg-tertiary, bg-accent).

**Layer 2: Award-Winning Examples**

Define the complete 18-pattern taxonomy organized into 6 visual groups. For EACH pattern, include:
- Name (kebab-case identifier)
- Visual description (what it looks like, not how it's coded)
- Compatible beats (which emotional arc beats this pattern works well with)
- Visual weight (light/medium/heavy -- helps with rhythm)
- Example reference (a known site that uses this pattern well)

**Group A: Centered Compositions**
1. `centered-hero` -- Full-width centered content, large type, minimal elements. Compatible: HOOK, CLOSE. Weight: medium.
2. `centered-minimal` -- Centered text block with generous whitespace, single statement. Compatible: BREATHE, TEASE. Weight: light.
3. `centered-stacked` -- Centered heading + vertically stacked content blocks below. Compatible: BUILD, PROOF. Weight: medium.

**Group B: Split Compositions**
4. `split-equal` -- 50/50 side-by-side content and media. Compatible: REVEAL, BUILD. Weight: medium.
5. `split-asymmetric` -- 60/40 or 70/30 with dominant visual side. Compatible: REVEAL, PEAK. Weight: medium-heavy.
6. `split-overlapping` -- Content overlaps media or vice versa, creates depth. Compatible: PEAK, TENSION. Weight: heavy.

**Group C: Grid Compositions**
7. `bento-grid` -- Asymmetric card grid with varied cell sizes (the 2025 dominant pattern). Compatible: BUILD, REVEAL. Weight: heavy.
8. `uniform-grid` -- Equal-sized cards in responsive grid (features, team, pricing). Compatible: BUILD, PROOF. Weight: medium.
9. `masonry` -- Pinterest-style staggered vertical layout. Compatible: PROOF, BUILD. Weight: medium-heavy.

**Group D: Flowing Compositions**
10. `marquee-horizontal` -- Infinite horizontal scroll (logos, testimonials, features). Compatible: TEASE, PROOF. Weight: light.
11. `scroll-storytelling` -- Vertical scroll-driven narrative with pinned/animated elements. Compatible: REVEAL, PEAK. Weight: heavy.
12. `timeline-vertical` -- Chronological vertical flow with alternating sides. Compatible: BUILD, PROOF. Weight: medium.

**Group E: Full-Bleed Compositions**
13. `full-bleed-media` -- Edge-to-edge image/video with overlay text. Compatible: HOOK, PEAK. Weight: heavy.
14. `full-bleed-interactive` -- Full-viewport interactive element (3D, demo, canvas). Compatible: PEAK. Weight: heavy.
15. `parallax-layers` -- Multi-layer depth with parallax scroll effect. Compatible: PEAK, REVEAL. Weight: heavy.

**Group F: Layered Compositions**
16. `card-spotlight` -- Grid of cards with cursor-following spotlight/glow effect. Compatible: BUILD, PEAK. Weight: medium-heavy.
17. `tabbed-showcase` -- Tab navigation revealing different content panels. Compatible: BUILD, REVEAL. Weight: medium.
18. `before-after` -- Side-by-side or slider comparison layout. Compatible: TENSION, REVEAL. Weight: medium.

For each group, include a 2-3 sentence description of the visual CHARACTER that unifies the group (e.g., "Centered compositions draw the eye to a single focal point, creating moments of pause and emphasis").

**Adjacency Rules (explicit, enforceable):**
1. No two adjacent sections use the same pattern (obvious)
2. No two adjacent sections use patterns from the same visual group (A-F) -- stricter rule that prevents subtle repetition
3. Exception: Group C patterns bento-grid and masonry are visually distinct enough to be adjacent (but uniform-grid adjacent to either is not allowed)
4. Background tokens must alternate: no two adjacent sections use the same bg token
5. Visual weight must vary: no 3+ consecutive "heavy" patterns (creates fatigue)
6. A page of N sections must use patterns from at least ceil(N/2) different groups

**MASTER-PLAN.md Layout Assignments Format:**
```markdown
## Layout Assignments (Pre-Assigned, No Adjacent Repeats)

| # | Section | Beat | Layout Pattern | Group | Background | Weight | Adjacent Valid? |
|---|---------|------|---------------|-------|------------|--------|----------------|
| 01 | hero | HOOK | centered-hero | A | bg-primary | medium | -- |
| 02 | logos | TEASE | marquee-horizontal | D | bg-secondary | light | A->D: VALID |
| 03 | features | BUILD | bento-grid | C | bg-primary | heavy | D->C: VALID |
| 04 | product | REVEAL | split-asymmetric | B | bg-tertiary | med-heavy | C->B: VALID |
| 05 | stats | BREATHE | centered-minimal | A | bg-secondary | light | B->A: VALID |
| 06 | demo | PEAK | full-bleed-interactive | E | bg-primary | heavy | A->E: VALID |
| 07 | testimonials | PROOF | masonry | C | bg-secondary | med-heavy | E->C: VALID |
| 08 | cta | CLOSE | centered-hero | A | bg-accent | medium | C->A: VALID |

### Validation Summary
- Groups used: A, D, C, B, A, E, C, A (6 of 6 groups for 8 sections -- PASS)
- Adjacent group repeats: None -- PASS
- Background alternation: primary, secondary, primary, tertiary, secondary, primary, secondary, accent -- PASS
- Weight sequence: med, light, heavy, med-heavy, light, heavy, med-heavy, med -- no 3+ consecutive heavy -- PASS
```

**Layer 3: Integration Context**
- Section-planner uses this skill during MASTER-PLAN.md generation. The planner assigns layout patterns after beat assignment, ensuring compatibility (beat-pattern mapping) and diversity (adjacency rules).
- Quality-reviewer uses this skill as a safety net: after sections are built, verify that the actual implementation matches the assigned layout pattern. If a builder was assigned `split-asymmetric` but built a centered layout, that's a gap.
- Build-orchestrator references the MASTER-PLAN.md assignments when composing spawn prompts for builders -- each builder knows its assigned pattern.
- Pattern names are used in builder spawn prompts: "Build this section using the `bento-grid` layout pattern (asymmetric card grid with varied cell sizes)."

**Layer 4: Anti-Patterns**
- Taxonomy too coarse (5-6 patterns for a 10-section page -- runs out of variety)
- Patterns named by CSS implementation rather than visual character (two different CSS approaches can look identical)
- Diversity enforcement as review-only (catching repetition post-build wastes builder effort)
- Rigid pattern adherence that kills creativity (patterns set the structural approach, not pixel-perfect layout -- builders have creative freedom within the pattern's character)
- Background alternation ignored (two dark sections back-to-back flatten visual rhythm even with different layouts)
- All "heavy" patterns in sequence (creates visual fatigue -- vary the weight)
- Using Group exception (C adjacent to C) too liberally -- it's an exception, not a default

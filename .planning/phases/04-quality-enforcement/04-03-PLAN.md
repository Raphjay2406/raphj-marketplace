**Layer 1: Decision Guidance (When/Why)**
- WHY a dedicated polish pass: Polish applied per-section during build lacks page-level cohesion. The polisher sees the FULL page and can tune stagger timing, shadow consistency, hover state variety, and texture density across all sections.
- WHEN it runs (LOCKED DECISION): Single pass at end of build, after all waves complete. Polisher receives: full built page (all code files), DESIGN-DNA.md, this skill's checklist.
- CREATIVE LICENSE (LOCKED DECISION): Full creative freedom. The polisher can restyle hover states, add unexpected micro-interactions, push visual refinement further. Only constrained by DNA tokens and archetype FORBIDDEN patterns. The checklist is a MINIMUM, not a maximum.
- DUAL-LAYER enforcement (LOCKED DECISION): Pre-assigned in MASTER-PLAN.md during planning AND reviewed post-build. For polish, this means: builders handle "light polish" (basic hover states, responsive) during build. Polisher handles "deep polish" (textures, animation pairing, page-level cohesion, signature element prominence) as end-of-build pass.
- OUTPUT: Updated code files with polish applied + SUMMARY.md with polish report listing what was added/modified.

**Layer 2: Award-Winning Examples**

**Universal Polish Checklist (applies to ALL archetypes):**

Organize into 8 categories. Each item is tagged as MUST HAVE, SHOULD HAVE, or NICE TO HAVE:

**Category 1: Hover & Interactive States (MUST HAVE)**
- Every clickable element has a distinct hover state (not just color change -- scale, shadow shift, glow, or border reveal)
- Hover transitions use appropriate easing and duration (200-300ms, ease-out or custom cubic-bezier, NEVER linear)
- Active/pressed state gives immediate feedback (<100ms)
- Focus states visible for keyboard navigation (consistent focus ring matching accent palette)
- Disabled states prevent interaction AND look disabled (opacity + cursor change)
- Button hover: at least 2 properties change (e.g., scale + shadow, or bg + translateY)
- Card hover: lift effect (translateY(-4px) + shadow increase) or spotlight glow
- Link hover: underline animation (scaleX from 0), color shift, or arrow/icon motion
- Icon hover: subtle rotation, scale (1.1), or color transition

**Category 2: Micro-Textures (ARCHETYPE-DEPENDENT)**
- Noise/grain texture on appropriate surfaces (dark backgrounds: SVG filter noise at 2-5% opacity)
- Gradient borders on key cards/containers (1px gradient from white/10 to white/03 or accent/20 to accent/05)
- Subtle background patterns where archetype permits (grid lines, dot patterns, grain overlays)

**Category 3: Selection & Cursor (SHOULD HAVE)**
- Custom `::selection` color matching accent palette (background: accent-1/20, color: white)
- Custom cursor on special interactive elements where archetype permits (pointer, crosshair, custom SVG)

**Category 4: Micro-Interactions (MUST HAVE)**
- Scroll-triggered entrance animations on every section (not just fade-in -- varied per section)
- Stagger timing on grouped elements (60-150ms between items, creates visual flow)
- Parallax or depth effect on at least one element per page
- Loading state animations where applicable (skeleton, shimmer, or fade)

**Category 5: Typography Polish (MUST HAVE)**
- Gradient text on at least one headline where archetype permits
- `text-wrap: balance` on important headings (prevents widows/orphans)
- Proper truncation with ellipsis where content may overflow
- Smooth font loading: `font-display: swap` + `<link rel="preload">` for display fonts
- Letter-spacing tuned on headings (negative tracking on display sizes, -0.02em to -0.04em)

**Category 6: Depth & Shadow (ARCHETYPE-DEPENDENT)**
- Multi-layer shadows on elevated elements (not just `shadow-lg` -- 2-3 shadow layers with different offsets and colors)
- Colored shadows matching element or accent color where appropriate (e.g., `shadow-[0_8px_32px_-8px_theme(colors.accent-1/25)]`)
- Glass/blur effects on overlays and navigation where archetype permits

**Category 7: Animation Polish (MUST HAVE)**
- All animations respect `prefers-reduced-motion: reduce` (media query wrapping or motion-safe utility)
- Enter animations paired with exit animations (components don't just appear/disappear)
- Scroll animations have appropriate trigger ranges (not too early at 200px below viewport, not too late at 0px)
- No animation plays on page load until fonts and critical images are ready

**Category 8: Responsive Polish (MUST HAVE)**
- No horizontal overflow at any viewport (375-1440px)
- Touch targets minimum 44x44px on mobile
- Font sizes responsive using `clamp()` (not just breakpoint steps)
- Spacing scales proportionally (mobile padding smaller than desktop, not identical)
- Hover effects gracefully degrade on touch (no hover-dependent information)

**Archetype-Specific Polish Addenda:**
For each of the 19 archetypes (16 original + Neubrutalism, AI-Native, Dark Academia), define:
- 3-5 MUST HAVE archetype-specific polish items
- 2-3 SHOULD HAVE items
- 3-5 FORBIDDEN items (things that violate this archetype's personality)

Write FULL addenda for the top 8 archetypes (these are the most used):
1. **Brutalist** -- Hard drop-shadows (4px 4px 0), instant transitions (<100ms), exposed structural elements, rotated elements. FORBIDDEN: gradient borders, glass blur, noise texture, rounded corners >4px, soft shadows.
2. **Ethereal** -- Floating gradient orbs (8-12s cycle), generous padding (py-32+), soft layered shadows (20px+ blur), serif accent text. FORBIDDEN: dark backgrounds, sharp corners, neon accents, heavy font weights, hard shadows.
3. **Kinetic** -- Scroll-triggered animation on EVERY section, parallax depth, horizontal scroll element, aggressive stagger (60-100ms). FORBIDDEN: static sections, slow transitions (>500ms), subtle animations.
4. **Editorial** -- Precise whitespace rhythm, high contrast headlines, pull quotes with serif, rule lines between sections. FORBIDDEN: glass blur, neon glow, 3D transforms, noise textures, playful animations.
5. **Neo-Corporate** -- Bento grids with varied cell sizes, product screenshots with 3D tilt, gradient text on hero headline, colored shadows. FORBIDDEN: system fonts, flat shadows, stock photo aesthetics, generic CTA copy.
6. **Luxury/Fashion** -- Letter-spacing on uppercase labels, generous negative space, art-directed imagery, minimal color palette. FORBIDDEN: bright accent colors, dense layouts, playful animations, rounded buttons.
7. **Neon Noir** -- Glow effects on interactive elements, dark-only palette, cyberpunk-inspired gradients, scan-line or CRT textures. FORBIDDEN: light backgrounds, pastel colors, serif fonts, soft/organic shapes.
8. **Glassmorphism** -- Multi-layer glass cards with blur(12-20px), gradient borders, subtle noise on glass, color-shifting backgrounds. FORBIDDEN: flat backgrounds, hard edges without blur, opaque cards, monochrome palette.

For the remaining 11 archetypes (Japanese Minimal, Swiss/International, Playful/Startup, Retro-Future, Warm Artisan, Vaporwave, Organic, Data-Dense, Neubrutalism, AI-Native, Dark Academia), provide a compact format:
- 3 MUST HAVE items
- 3 FORBIDDEN items

- Quality-reviewer checks that: (a) MUST HAVE universal items are present, (b) archetype-specific MUST HAVE items are present, (c) no FORBIDDEN items exist. Missing MUST HAVE items = WARNING. FORBIDDEN items present = CRITICAL (archetype violation).
- The polisher's input at end-of-build: all built code files + DESIGN-DNA.md + this skill. The polisher does NOT read PLAN.md files or MASTER-PLAN.md -- it works from the code and DNA directly.

**Layer 4: Anti-Patterns**
- Checklist as checkbox exercise: mechanically adding every item to every section regardless of archetype fit
- Polisher limited to checklist only: the checklist is a MINIMUM. The polisher's creative license means they should EXCEED it with unexpected refinements
- Polish during build: builders handle light polish. Deep polish needs the full page context, so it runs end-of-build
- Same polish on every archetype: Brutalist pages with gradient borders, Ethereal pages with neon glow. FORBIDDEN items exist to prevent this
- Over-polishing: adding so many micro-interactions that the page feels busy or slow. Polish enhances, it doesn't overwhelm
- Ignoring `prefers-reduced-motion`: every animation addition must respect this media query. Failing this is a critical accessibility violation, not a polish concern
- Polish that breaks layout: adding hover transforms that cause reflow, glass blur on too many elements causing performance issues. Polish must not degrade performance

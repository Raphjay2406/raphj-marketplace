**5. Insert Steps 6.5, 6.6, 6.7, 6.8 AFTER Step 6 ("Collect Results") and BEFORE Step 7 ("Post-Wave Coherence Checkpoint"):**

**Step 6.5: Post-Wave Quality Review (Parallel)**
- Spawn BOTH CD and QR simultaneously via Task tool
- CD (post-build thorough review): "Checkpoint 2 (post-build thorough review) for Wave [N]" -- provide current wave section names and built file paths; CD reads its contracted files per creative-director.md input contract
- QR (post-wave verification): "Post-wave verification for Wave [N]" -- provide current wave section names; QR reads its contracted files per quality-reviewer.md input contract
- Both run in parallel. Wait for BOTH to complete before Step 6.6
- This is NOT optional -- it runs after EVERY wave

**Step 6.6: Findings Merge + Severity Classification**
- Collect all findings from QR verification report and CD creative assessment
- Deduplicate: if both flag the same section for the same issue, keep the more specific finding
- Classify each finding using the severity system from quality-gate-protocol:
  - CRITICAL: anti-slop < 25, archetype forbidden pattern, missing signature element entirely, build failure
  - WARNING: anti-slop 25-27, CD below-creative-bar, spacing inconsistency, could-be-bolder
  - INFO: score breakdowns, positive observations, strengths noted
- CD findings severity mapping: forbidden pattern = CRITICAL, below-bar = WARNING, could-be-bolder = WARNING, positive = INFO
- Log all findings in wave quality report
- IMPORTANT: Keep this step as COORDINATION logic only. The orchestrator collects and classifies mechanically using predefined tables. It does NOT replicate review criteria. This section should be under 50 lines.

**Step 6.7: GAP-FIX Remediation Loop**
- For each section with a GAP-FIX.md file (from CD or QR):
  - Spawn a SEPARATE polisher instance per section via Task tool (parallel, max 4)
  - Each polisher reads: its GAP-FIX.md + listed code files + DESIGN-DNA.md (per polisher.md contract)
- After all polishers complete: spawn QR to re-score the FULL gate for fixed sections (not partial)
- If still failing: create second GAP-FIX.md, repeat cycle
- Maximum 2 remediation cycles. If still failing after 2 cycles: escalate to user with full evidence (anti-slop breakdown, specific failing items, remediation history)
- If NO GAP-FIX.md files were created: skip this step entirely
- Remediation priority (polisher handles internally): penalty fixes first (-3 to -5 pts), then highest-point category failures, then lowest-point issues

**Step 6.8: Wave Review Gate**
- Any CRITICAL finding exists: BLOCK pipeline. Report to user with evidence. Wait for user decision
- WARNING findings only: Add to running tally in STATE.md. Log real-time status: "Wave [N] complete -- [X] warnings pending" with per-warning detail. Continue to Step 7
- Clean (no CRITICAL, no WARNING): Continue to Step 7

**6. Update Step 10 (CONTEXT.md Rewrite):**

  3. After Layer 3: trigger Layer 4 user checkpoint (mandatory if warnings accumulated, auto-proceed if clean)
- Keep this section under 30 lines -- it is a brief protocol extension, not a full specification. The quality-gate-protocol skill has the full specification.

**CRITICAL CONSTRAINTS:**

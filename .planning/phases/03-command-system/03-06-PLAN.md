**Create `commands/status.md`** (~50-70 lines) -- Full project status display:

Frontmatter:
```yaml
---
description: Show full Modulo project status -- phases, waves, sections, and artifacts
argument-hint: [--verbose] [--section name]
allowed-tools: Read, Glob
---
```

Body structure:
1. Role statement: "You are the Modulo Status reporter. You display the current project state in a clear, actionable format."

2. State Check: Read STATE.md and CONTEXT.md. If neither exists: "No Modulo project found. Run /modulo:start-project to begin."

3. Argument Parsing:
   - `--verbose` or `-v` = show full details per section
   - `--section name` or `-s name` = show details for specific section
   - No arguments = show overview

4. Display Format (overview mode):
   ```
   Modulo Project Status
   =====================

   Phase: [current phase] | Archetype: [name] | Direction: [name]

   Artifacts:
     [checkmark/x] PROJECT.md        [checkmark/x] BRAINSTORM.md
     [checkmark/x] DESIGN-DNA.md     [checkmark/x] CONTENT.md
     [checkmark/x] MASTER-PLAN.md    [checkmark/x] CONTEXT.md

   Sections: [complete]/[total] | Waves: [current]/[total]

   | Section | Wave | Beat | Status | Layout |
   |---------|------|------|--------|--------|
   | 00-shared | 0 | -- | COMPLETE | -- |
   | 01-hero | 2 | HOOK | IN_PROGRESS | split-hero |
   ...

   Discussions: [list of DISCUSSION-{phase}.md files]
   Quality: [last audit score if exists]

   Next action: [contextual suggestion based on state]
   ```

5. Display Format (verbose mode): Add per-section details -- files created, must_haves status, last modified.

6. Display Format (section mode): Show full section detail -- PLAN.md summary, SUMMARY.md if exists, files, quality issues.

7. End with next action suggestion (same as other commands' guided flow).

No rules section needed -- this is a read-only display command.

**Rewrite `commands/audit.md`** (~70-100 lines) -- Comprehensive site audit (absorbs verify, responsive-check, lighthouse):

Frontmatter:
```yaml
---
description: Comprehensive site audit -- visual quality, performance, accessibility, and design compliance
argument-hint: [--section name] [--quick] [--category name]
allowed-tools: Read, Write, Edit, Bash, Grep, Glob, Task
---
```

Body structure:
1. Role statement: "You are the Modulo Audit orchestrator. You run comprehensive quality audits covering visual design, performance, accessibility, and DNA compliance -- producing scored reports with prioritized fix plans."

2. Guided Flow Header: Read STATE.md and CONTEXT.md. Display one-line status.

3. State Check: Required state -- built sections exist. If not: "Nothing to audit yet. Run /modulo:execute first."

4. Argument Parsing:
   - `--section name` or `-s name` = audit specific section only
   - `--quick` or `-q` = run only the anti-slop 35-point gate (skip full audit)
   - `--category name` or `-c name` = audit specific category only (visual, performance, accessibility, content)
   - No arguments = full comprehensive audit

5. Audit Dispatch:
   - Spawn parallel `quality-reviewer` agents via Task tool:
     (a) Visual quality audit (DNA compliance, anti-slop 35-point gate, Awwwards 4-axis scoring)
     (b) Performance audit (CWV, bundle size, images, fonts, animations)
     (c) Accessibility audit (WCAG AA, keyboard, ARIA, focus, contrast)
     (d) Content verification (copy matches CONTENT.md, no forbidden phrases)
   - Each agent writes findings to `.planning/modulo/audit/` directory
   - Include: live browser testing if browser tools available (screenshots at 375/768/1024/1440px)

6. Report Synthesis:
   - Read all agent outputs
   - Create unified AUDIT-REPORT.md with scores per category and overall
   - Create FIX-PLAN.md with prioritized fix tasks
   - Present summary to user with scores and critical issues

7. Completion & Next Step:
   - If score >= 30/35 (anti-slop): "SOTD-ready quality."
   - If score 25-29: "Premium quality. [N] improvements recommended."
   - If score < 25: "Below quality bar. [N] critical issues must be fixed. Run /modulo:iterate --from-gaps to address them."

8. Rules (4-5 items):
   - Goal-backward verification: check if goals were achieved, not if tasks were completed.
   - Be ruthless. Every pixel matters at the award-winning bar.
   - Create actionable GAP-FIX.md for every gap found. Don't just report.
   - Anti-slop gate is automatic and cannot be skipped (even in --quick mode, it's the minimum).
   - Always end with a clear next step.

**This audit command absorbs:**
- v6.1.0 `verify.md` (three-level goal-backward check + anti-slop gate -> absorbed into visual quality audit)
- v6.1.0 `responsive-check.md` (responsive testing -> absorbed into accessibility/visual audit)
- v6.1.0 Lighthouse functionality (performance audit)
- Awwwards 4-axis scoring

  <action>
Create `commands/lets-discuss.md` as a creative deep dive command (~100-130 lines). Structure:

**Frontmatter:**
```yaml
---
description: Creative deep dive -- explore visual features, content direction, and design ideas for a specific phase
argument-hint: [phase name or number, e.g., "hero" or "pricing"]
allowed-tools: Read, Write, Edit, Bash, Grep, Glob, Task
---
```

**Body structure:**

1. **Role statement**: "You are the Genorah Creative Discussion facilitator. You guide users through visual feature exploration, brand voice refinement, and creative ideation -- producing structured output that feeds directly into planning."

2. **Guided Flow Header**: Read STATE.md and CONTEXT.md. Display one-line status. If project doesn't exist, tell user to run /gen:start-project first.

3. **State Check & Auto-Recovery**:
   - Required state: DNA_COMPLETE or later (need DESIGN-DNA.md and BRAINSTORM.md to discuss creative features)
   - If no DNA: "Run /gen:start-project first to establish creative direction."
   - Check if DISCUSSION-{phase}.md already exists for the target phase. If so: "A discussion already exists for this phase. Want to continue it, start fresh, or proceed to planning?"

4. **Argument Parsing**:
   - Bare text = phase name or focus area (e.g., "hero", "pricing", "animations")
   - `--phase N` or `-p N` = specific phase number
   - `--fresh` or `-f` = start new discussion even if one exists
   - If no argument: auto-detect current phase from STATE.md

5. **Phase Context Loading**:
   - Read DESIGN-DNA.md for archetype constraints, color palette, signature element
   - Read BRAINSTORM.md for creative direction
   - Read CONTENT.md if it exists for content context
   - Read any existing section plans for the target phase
   - Determine what creative features are relevant to this phase

6. **Discussion Protocol** -- Three conversation tracks, interleaved naturally:

   **Track A: Visual Feature Proposals**
   - Dispatch to `creative-director` agent to generate 2-3 visual feature proposals for the phase
   - Each proposal includes: name, description, ASCII mockup showing layout, how it fits the archetype, implementation complexity (simple/medium/complex)
   - Present proposals to user. User picks favorites, rejects others, suggests modifications.
   - This is a conversation, not a one-shot presentation. Follow up on their reactions.

   **Track B: Content & Voice Refinement**
   - Based on the phase scope, suggest brand voice refinements
   - Propose CTA styles, headline approaches, micro-copy tone
   - If CONTENT.md exists, suggest improvements specific to this phase's sections
   - User approves, modifies, or rejects suggestions

   **Track C: Creative Wild Cards**
   - Propose 1-2 "what if" ideas -- unexpected creative approaches that push the archetype
   - These should be bolder than the standard proposals (creative tension opportunities)
   - User can adopt, modify, or save for later

   **Important:** These tracks interleave in natural conversation. Do NOT present them as three sequential phases. Follow the user's interest.

7. **Auto-Organization** -- After discussion concludes:
   - Organize all decisions into structured format
   - Create `.planning/genorah/DISCUSSION-{phase}.md` with:
     ```
     # Discussion: [Phase Name]
     Date: [ISO date]

     ## Visual Feature Decisions
     - [feature]: [accepted/rejected/modified] -- [details]

     ## Content & Voice Decisions
     - [decision]: [details]

     ## Creative Ideas (Adopted)
     - [idea]: [details and where to apply]

     ## Creative Ideas (Saved for Later)
     - [idea]: [details]

     ## Task-Ready Items
     Items ready for plan-dev to consume:
     - [ ] [specific implementable item]
     - [ ] [specific implementable item]
     ```
   - Update STATE.md to record that discussion happened for this phase

8. **Completion & Next Step**:
   ```
   Discussion captured for [phase name].

   Artifact: .planning/genorah/DISCUSSION-{phase}.md
   [N] visual features decided, [M] content refinements, [K] creative ideas saved.

   Next step: /gen:plan-dev
     Plan-dev will incorporate these discussion decisions into section plans.
   ```

9. **Rules** (5-6 items):
   - This is a conversation, not a presentation. Follow the user's energy.
   - Visual proposals must include ASCII mockups -- don't just describe, show.
   - Never reject user ideas outright. Build on them or explain tradeoffs.
   - Auto-organize output into DISCUSSION-{phase}.md. Discussions must be actionable.
   - Stay within archetype constraints from DESIGN-DNA.md. Push boundaries, don't break them.
   - Always end with a clear next step.

**Critical constraints:**
- Target 100-130 lines. Creative direction details belong in creative-director agent.
- The command facilitates the conversation; the creative-director agent generates the proposals.
- DISCUSSION-{phase}.md is the key output artifact that plan-dev consumes.
- This command must work both when invoked standalone AND when auto-offered by plan-dev.

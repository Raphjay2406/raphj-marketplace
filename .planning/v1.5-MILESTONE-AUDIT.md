## Gap Details

### GAP-1: SKILL-DIRECTORY.md Registration (CRITICAL)

**Files:** `skills/SKILL-DIRECTORY.md`

`api-patterns` (Phase 17, 1601 lines) and `ssr-dynamic-content` (Phase 19, 1842 lines) are not listed. The directory shows "Domain: 24" but should be 26. Registry version says "Phase 18, Plan 4" but Phase 19 exists.

**Note:** Skills are auto-discovered by the plugin file system, so this does not break runtime. It breaks the reference directory used by agents for skill discovery.

**Fix:** Add both skills to the "Integration" subsection of Domain Skills. Update counts to 51 total (22 Core, 27 Domain, 2 Utility). Bump registry to v2.2.0.

### GAP-2: Build-Orchestrator Wave 0 Incomplete (CRITICAL)

**File:** `agents/pipeline/build-orchestrator.md` lines 72-76

Current Wave 0 scaffold items:
- sitemap.ts (seo-meta)

- metadataBase (seo-meta)

Missing items:

- `llms.txt` (search-visibility)
- `next.config.ts` cache setup (ssr-dynamic-content)
- `proxy.ts` route protection (ssr-dynamic-content, if auth needed)
- Skeleton `loading.tsx` files (ssr-dynamic-content)

**Fix:** Expand Wave 0 scaffold list with search-visibility and ssr-dynamic-content items.

### GAP-3: Agent Pipeline Wiring for api-patterns (MAJOR)

**Files:** `agents/pipeline/section-planner.md`, `agents/pipeline/quality-reviewer.md`, `agents/pipeline/build-orchestrator.md`

When a project needs CRM forms, webhook receivers, or typed API clients, no pipeline agent knows to load `api-patterns`. The researcher has Context7 tools but doesn't reference the skill. The quality-reviewer has no env variable secret checking.

**Fix:** Add api-patterns trigger conditions to section-planner (when section needs form submission, API calls, or webhooks). Add env secret exposure check to quality-reviewer.

### GAP-4: Agent Pipeline Wiring for ssr-dynamic-content (MAJOR)

**Files:** `agents/pipeline/section-planner.md`, `agents/pipeline/quality-reviewer.md`

No agent assigns rendering strategies using ssr-dynamic-content's decision matrix. The quality-reviewer has no constraints for deprecated patterns (middleware.ts, unstable_cache, experimental.ppr).

**Fix:** Add rendering strategy assignment to section-planner. Add SSR anti-pattern checks to quality-reviewer.

### GAP-5: Schema Audit Protocol Not Embedded (MAJOR)

**File:** `agents/pipeline/quality-reviewer.md`

structured-data skill (SDATA-06) defines a 14-item content-schema consistency checklist that must run on every quality-reviewer pass. The quality-reviewer only says "Consult structured-data skill" at line 118 but does not embed the actual checklist items.

**Fix:** Add schema-content consistency check items to quality-reviewer, or add explicit "load structured-data and run SDATA-06 audit" step.
